## 文档概述

本文档定义承兑业务（OnRamp / OffRamp）及跨 SP 法币账户划转的产品属性。按 **SP 能力定义（第1层）** 和 **TP 租户配置（第2层）** 两层拆分。

**本期范围：**

| 产品                    | 本期范围   | 说明                         |
| ----------------------- | ---------- | ---------------------------- |
| OnRamp（买入数币）      | ✅ A1 + B1 | 纯 BB 内部承兑，无跨 SP      |
| OffRamp（卖出数币）     | ✅ A1 + B1 | 纯 BB 内部承兑，无跨 SP      |
| 跨 SP 法币划转          | ✅         | IPL ↔ BB 法币账户间独立划转 |
| 跨 SP 承兑场景（A2/B2） | ❌ Phase 2 | 划转 + 承兑一体化编排        |

**承兑方向：**

| 产品    | 方向         | 本期货币对                |
| ------- | ------------ | ------------------------- |
| OnRamp  | 法币 → 数币 | USD → USDT / USD → USDC |
| OffRamp | 数币 → 法币 | USDT → USD / USDC → USD |

**设计原则：**

- 承兑业务一期由 **BB 单一 SP 承接**（承兑引擎、汇率、风控、数币钱包均走 BB）
- 承兑资金必须已在 BB 承兑交易账户内（OnRamp）或承兑后法币留在 BB 账户内（OffRamp）
- 跨 SP 资金移动 = **独立划转产品**，不属于 OnRamp/OffRamp 范畴
- 计费详见 `productfee.md`，汇率详见 `fx.md`，风控由独立风控系统管理

## 1. OnRamp 买入数币（ON_RAMP）

### 1.1 SP 能力定义（BB 上架）

#### 基本信息

| 字段     | 类型 | 必填 | 说明           | 本期值             |

| 产品名称 | 文本 | —   | 展示名称       | 入金（法币→数币） |

#### 1.1.2 汇率能力

| 字段         | 类型       | 必填 | 说明                   | 本期值                    |
| ------------ | ---------- | ---- | ---------------------- | ------------------------- |

| 字段             | 类型   | 必填 | 说明                       | 本期默认值                     |
| ---------------- | ------ | ---- | -------------------------- | ------------------------------ |

| 容差匹配阈值     | 百分比       | 条件必填 | 入账金额与预约金额的允许偏差 | 5%（或绝对值 ±50 USD，取大者） |

### 1.2 TP 产品配置（租户给商户配置）

#### 1.2.1 承兑配置

| 字段         | 类型 | 说明                        | 约束       | 默认值    |
| ------------ | ---- | --------------------------- | ---------- | --------- |
| 允许的货币对 | 多选 | TP 允许商户使用的承兑货币对 | ⊆ SP 支持 | = SP 全部 |
| 交易模式     | 多选 | 允许即时单 / 预约单         | ⊆ SP 支持 | = SP 全部 |

#### 1.2.2 汇率加点（TP Markup）

> TP 加点 = TP 给商户的承兑汇率加价，是 TP 的利润来源。

| 字段                          | 类型       | 说明                         | 约束 | 默认值         |
| ----------------------------- | ---------- | ---------------------------- | ---- | -------------- |
| 默认买入加点（tp_markup_buy） | 基点（bp） | TP 给所有商户的默认加点      | > 0  | 50bp（0.0005） |
| 商户级买入加点                | 基点（bp） | 为特定商户配置的加点（可选） | > 0  | —（使用默认） |

**汇率计算公式（买入数币）：**

```
商户买入汇率 = 市场汇率 × (1 + TP加点)

示例：USD → USDT
  市场汇率：0.9998
  TP 加点：+0.0005（+50bp）
  商户汇率：0.9998 × (1 + 0.0005) = 0.9998 × 1.0005 = 1.00030
  TP 利润率：50bp
```

#### 1.2.3 滑点配置

| 字段       | 类型   | 说明              | 约束     | 默认值          |
| ---------- | ------ | ----------------- | -------- | --------------- |
| 滑点容忍度 | 百分比 | TP 调整的滑点上限 | ≤ SP 值 | = SP 值（0.5%） |

#### 1.2.4 预约单配置

| 字段         | 类型         | 说明                 | 约束               | 默认值           |
| ------------ | ------------ | -------------------- | ------------------ | ---------------- |
| 预约单有效期 | 数字（小时） | TP 调整的超时时间    | ≤ SP 上限（168h） | = SP 默认（72h） |
| 容差匹配阈值 | 百分比       | TP 调整的入账偏差    | ≤ SP 值           | = SP 值（5%）    |
| 过期自动处理 | 下拉         | 预约单过期后处理方式 | —                 | 自动取消         |

#### 1.2.5 限额配置

| 字段         | 类型 | 说明                  | 约束     | 默认值  |
| ------------ | ---- | --------------------- | -------- | ------- |
| 单笔最小金额 | 金额 | TP 限制的最小承兑金额 | ≥ SP 值 | = SP 值 |
| 单笔最大金额 | 金额 | TP 限制的最大承兑金额 | ≤ SP 值 | = SP 值 |
| 单日最大金额 | 金额 | TP 限制的每日上限     | ≤ SP 值 | = SP 值 |
| 单月最大金额 | 金额 | TP 限制的每月上限     | ≤ SP 值 | = SP 值 |

## 3. 跨 SP 法币账户划转（FIAT_TRANSFER）

> **定位**：独立产品，不属于 OnRamp/OffRamp 范畴。商户在两个 SP 的法币账户间转移资金。
> **本期范围**：仅支持 USD 账户之间的划转（IPL USD ↔ BB USD）。

### 3.1 SP 能力定义

#### 基本信息

| 字段     | 类型 | 必填 | 说明         | 本期值            |
| -------- | ---- | ---- | ------------ | ----------------- |
| 产品代码 | 固定 | —   | 产品唯一标识 | `FIAT_TRANSFER` |
| 产品名称 | 文本 | —   | 展示名称     | 法币账户间划转    |
| 产品状态 | 下拉 | ✅   | 上架状态     | 启用              |

#### 3.1.1 划转路由

| 字段           | 类型 | 必填 | 说明                 | 本期值             |
| -------------- | ---- | ---- | -------------------- | ------------------ |
| 支持的划转方向 | 多选 | ✅   | 允许的 SP 间资金流向 | IPL→BB / BB→IPL  |
| 支持的划转币种 | 多选 | ✅   | 划转的法币币种       | USD                |
| 是否同币种限制 | 开关 | ✅   | 划转出入币种必须相同 | 是（本期仅同币种） |

> 本期仅支持 **USD→USD** 同币种划转。跨币种划转（如 HKD→USD）属于法币换汇产品（`fx.md`），不在此处定义。

#### 3.1.2 划转模式与风控（本期固定，不可配）

| 规则             | 本期值               | 说明                                                       |
| ---------------- | -------------------- | ---------------------------------------------------------- |

| 入款风控       | 强制开启             | 划入方 SP 风控校验，不可关闭                            |
| 同名校验         | 强制                 | 划出划入必须同一商户，不可关闭                          |

> **设计原则**：跨 SP 划转是两笔独立交易单（出款 T001 + 入款 T002），各自有完整的风控/计费/三单模型。本期全部实时执行且强制双重风控。

#### 3.1.3 限额（SP 上限）

| ------------ | ----------- | ---- | ---------------- | -------------- |
| 单笔最小金额 | 金额 + 币种 | ✅   | 最小划转金额     | 100 USD        |
| 单笔最大金额 | 金额 + 币种 | ✅   | 最大划转金额     | 1,000,000 USD  |
| 单日最大金额 | 金额 + 币种 | ✅   | 每日划转累计上限 | 5,000,000 USD  |
| 单月最大金额 | 金额 + 币种 | ✅   | 每月划转累计上限 | 50,000,000 USD |

#### 3.1.4 划转失败处理 & 成功后退回

## 4. 退款属性配置

> 详细退款流程见 `refund.md`。本节仅定义产品层面的退款配置属性。

### 4.1 承兑退款配置（OnRamp + OffRamp 共用）

#### 4.1.1 SP 能力定义

| 字段                 | 类型   | 必填 | 说明                       | 本期值                      |
| -------------------- | ------ | ---- | -------------------------- | --------------------------- |
| 是否支持承兑后退款   | 开关   | ✅   | 承兑成功后是否允许退款     | 是                          |
| 退款汇率模式选项     | 多选   | ✅   | SP 支持的退款汇率模式      | smart / original / realtime |
| 默认退款汇率容忍阈值 | 百分比 | ✅   | smart 模式下 SP 侧默认阈值 | 1%                          |

#### 4.1.2 TP 配置（租户给商户配置）

| 字段                  | 类型   | 说明                 | 约束             | 默认值            |
| --------------------- | ------ | -------------------- | ---------------- | ----------------- |
| refund_rate_mode      | 下拉   | 退款汇率模式         | ⊆ SP 支持的模式 | smart             |
| refund_rate_threshold | 百分比 | smart 模式下容忍阈值 | ≤ SP 默认值     | 1%                |
| refund_fee_policy     | 下拉   | 付款手续费是否退还   | —               | no_refund（不退） |

**退款汇率模式说明（OffRamp 退款：原交易 USDT→USD，付款失败退回）：**

```
smart（智能判断，默认）:
  原汇率 = 原交易时 1 USDT = X USD
  实时汇率 = 退款时 1 USDT = Y USD

  Y ≤ X（USDT贬值，BB不亏）→ 按实时汇率退回数币钱包 ✅
  Y > X 且偏差 ≤ 阈值（BB小亏）→ 按原汇率退回数币钱包 ✅
  Y > X 且偏差 > 阈值（BB大亏）→ 退回到法币账户（不做反向承兑）⚠️

original（始终按原汇率）:
  始终按原交易汇率退回数币钱包

realtime（始终按实时汇率）:
  始终按退款时实时汇率退回数币钱包
```

**通用退款规则（不可配置）：**

| 规则                     | 说明                     |
| ------------------------ | ------------------------ |
| 退回数币时不收承兑手续费 | 反向承兑退回免收承兑费   |
| 默认付款手续费不退       | 特殊情况走清算特殊流程   |
| 退款渠道单是新单         | 关联原渠道单，不修改原单 |
| 商户只看到1笔退款        | 内部多笔交易单不可见     |

### 4.2 划转退款配置

划转产品的退款/退回逻辑见 3.1.5 和 3.1.6，无需额外 TP 配置。

### A. 汇率加点 vs 滑点 概念对比

| 概念    | 英文      | 定义                   | 配置方                 | 作用         |
| ------- | --------- | ---------------------- | ---------------------- | ------------ |
| TP 加点 | TP Markup | TP 给商户的承兑汇率加价 | TP（TP Portal）        | TP 利润来源  |
| 滑点    | Slippage  | 承兑执行时汇率偏差     | SP 定义上限，TP 可收紧 | 汇率波动保护 |

> **SP 返点（SP Spread）** 属于法币换汇产品（`fx.md`），不在承兑业务范围内。

**关系：**

```
市场汇率 ──(TP加点)──→ 商户汇率 ──(滑点保护)──→ 实际执行汇率

利润分配：
  TP 利润 = TP 加点
  商户成本 = TP 加点（已包含在汇率中）
```
