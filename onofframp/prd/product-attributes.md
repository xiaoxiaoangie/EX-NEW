# 数法承兑产品属性定义

## 文档概述

本文档定义 EX 平台"数法承兑"相关产品的完整属性，参照 PPaaS（Payment Platform as a Service）的产品配置模式。

**产品拆分：**

| 产品代码 | 产品名称 | 说明 |
| --- | --- | --- |
| `ON_RAMP` | 入金（法币→数币） | 法币承兑为数币，含即时单和预约单两种模式 |
| `OFF_RAMP` | 出金（数币→法币） | 数币承兑为法币，含直接出款和先承兑再出款 |

> VA 同名充值是 OnRamp 预约单模式下的一种法币入金方式（收款账户类型），不作为独立产品。
> 非同名收款（Collection）是独立产品，不在本文档范围内。

**配置层级（本期2层）：**

```
┌─────────────────────────────────────────────────────────┐
│  第1层：SP 能力定义（SP 上架时填写，不可被 TP/商户修改）    │
│  → 定义 SP 能做什么、能力边界在哪                          │
├─────────────────────────────────────────────────────────┤
│  第2层：TP 产品配置（TP 签约后配置，可对商户做特殊配置）    │
│  → 在 SP 能力范围内做限制、调整、个性化                    │
│  → 本期商户级配置也由 TP 代配                              │
└─────────────────────────────────────────────────────────┘

下期增加：
  第3层：商户自助配置（商户自己在 MP Portal 配置偏好）
```

**关联文档：**

| 文档 | 关联内容 |
| --- | --- |
| `productfee.md` | 计费模型、费率层级、Floor Price |
| `fx.md` | 汇率返点模型、承兑汇率/法币换汇 |
| `productionopening.md` | 商户入网、产品开通、入网推送规则 |
| `onramp-complete.md` | OnRamp 完整业务流程 |

## 1. OnRamp 产品属性

### 1.1 SP 能力定义

> SP 上架 OnRamp 产品时填写，定义 SP 的能力边界。TP/商户不可修改这些属性。

#### 1.1.1 产品基础信息

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 产品代码 | 文本（固定） | 产品唯一标识 | `ON_RAMP` |
| 产品名称 | 文本 | 产品展示名称 | 入金（法币→数币） |
| SP 标识 | 文本 | 提供此产品的 SP | BB / IPL |
| 产品状态 | 下拉 | 上架状态 | 启用 / 停用 |
| 承兑方向 | 固定 | 法币→数币 | Fiat → Crypto |

#### 1.1.2 支持的货币与网络

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 支持的法币币种 | 多选 | SP 支持的法币入金币种 | USD, HKD, EUR, GBP, SGD |
| 支持的数币币种 | 多选 | SP 支持承兑的目标数币 | USDT, USDC |
| 支持的货币对 | 列表 | 允许的法币→数币兑换组合 | USD→USDT, USD→USDC, HKD→USDT |
| 支持的区块链网络 | 多选 | 数币入账支持的链 | TRC-20, ERC-20, BEP-20 |

#### 1.1.3 交易模式能力

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 支持即时单 | 开关 | 是否支持法币余额直接承兑 | 是 / 否 |
| 支持预约单 | 开关 | 是否支持先下单后汇款模式 | 是 / 否 |

#### 1.1.4 法币入金方式（SP 能力）

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 支持的收款账户类型 | 多选 | SP 提供的法币入金通道 | VA同名充值 / 代收付账户 / 法币余额扣款 |
| VA 支持的币种 | 列表 | VA 可收款的币种（按地区） | USD(HK), HKD(HK), EUR(UK) |
| VA 支持的地区 | 多选 | VA 开户所在地区 | HK, SG, UK |
| 代收付账户类型 | 下拉 | 代收付账户模式 | Omnibus（多商户共用） |

#### 1.1.5 限额能力（SP 上限）

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 单笔最小金额 | 金额 | SP 支持的最小交易金额 | 10 USD |
| 单笔最大金额 | 金额 | SP 支持的最大交易金额 | 1,000,000 USD |
| 单日最大金额 | 金额 | SP 支持的单日累计上限 | 5,000,000 USD |
| 单月最大金额 | 金额 | SP 支持的单月累计上限 | 50,000,000 USD |

#### 1.1.6 计费能力（SP 成本价 + Floor Price）

> 详细计费模型见 `productfee.md`，此处仅列出 SP 上架时需定义的计费项。

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 承兑手续费（SP 成本价） | 费率 | SP 给 TP 的结算价 | 0.5% |
| 承兑手续费（SP 商户最低报价） | 费率 | TP 给商户配置的费率下限 | 1.0% |
| 计费方式 | 下拉 | 支持的计费模式 | 固定费 / 比例费 / 混合 / 阶梯 |
| 费用扣取方式 | 下拉 | 内扣还是外扣 | 内扣 / 外扣 |

#### 1.1.7 汇率能力

> 详细汇率模型见 `fx.md`。

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 汇率来源 | 下拉 | SP 提供的汇率来源 | SP 自有报价 / 市场汇率 |
| 汇率锁定时长（即时单） | 数字（秒） | 即时单汇率锁定窗口 | 30s |
| 汇率返点（SP→TP） | 百分比 | SP 给 TP 的汇率返点 | 0.1% |

#### 1.1.8 退款能力（SP 级）

> 详细退款流程见 `refund.md`。

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 是否支持退款 | 开关 | SP 是否支持 OnRamp 退款 | 是 |
| 退款汇率容忍阈值 | 百分比 | 承兑后退款时，汇率波动容忍范围 | 1% |
| 退款汇率策略 | 规则 | 超出阈值时的退款方式 | 阈值内退数币 / 超阈值退法币 |
| 退款手续费策略 | 下拉 | 退款时承兑手续费是否退还 | 不退承兑费 / 全退 |
| 退款时效 | 数字（小时） | 退款处理时效 | 24h |

#### 1.1.9 风控能力（SP 级）

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| SP 风控开关 | 开关 | SP 是否有自己的风控 | 是（EX 不管 SP 内部风控） |
| SP 审核时效 | 文本 | SP 入网审核时效 | 1-3 个工作日 |

#### 1.1.10 预约单能力（SP 级）

| 字段 | 类型 | 说明 | 示例值 |
| --- | --- | --- | --- |
| 预约单默认有效期 | 数字（小时） | 预约单超时时间 | 72h |
| 预约单匹配模式 | 多选 | 支持的匹配方式 | 精确匹配 / 容差匹配 / 人工匹配 |
| 容差匹配阈值 | 百分比 | 入账金额与预约金额的允许偏差 | 5% |
| 同名校验 | 开关 | 是否要求汇款人与商户同名 | 是 |
| Purpose Code 校验 | 开关 | 是否要求转账备注含 Purpose Code | 是 |

### 1.2 TP 产品配置

> TP 签约 OnRamp 产品后，可在 SP 能力范围内为商户做配置。本期商户级配置也由 TP 代配。

#### 1.2.1 入网与开通配置

| 字段 | 类型 | 说明 | 可选值 | 默认值 |
| --- | --- | --- | --- | --- |
| 法币账户模式 | 下拉 | 商户可用的法币账户范围 | 仅BB / 仅IPL / BB+IPL | BB+IPL |
| 入网推送规则 | 规则（固定） | 入网时推送哪些 SP 审核 | BB固定推送 + IPL看商户是否勾选 | — |
| 默认开通的子能力 | 多选（固定） | 开通 OnRamp 时自动开通 | BB数币钱包 + BB法币账户 + 承兑汇率 | 全部默认开通 |

> 详细入网推送规则见 `productionopening.md`。

#### 1.2.2 限额配置（TP 可在 SP 上限内调整）

| 字段 | 类型 | 说明 | 约束 | 默认值 |
| --- | --- | --- | --- | --- |
| 是否有单笔限额 | 开关 | — | — | 是 |
| 单笔最小金额 | 金额 | 单笔交易最小金额 | ≥ SP单笔最小 | = SP值 |
| 单笔最大金额 | 金额 | 单笔交易最大金额 | ≤ SP单笔最大 | = SP值 |
| 是否有单日限额 | 开关 | — | — | 是 |
| 单日限额 | 金额 | 每日交易累计上限 | ≤ SP单日最大 | = SP值 |
| 是否有单周限额 | 开关 | — | — | 否 |
| 单周限额 | 金额 | 每周交易累计上限 | ≤ SP能力 | — |
| 是否有单月限额 | 开关 | — | — | 是 |
| 单月限额 | 金额 | 每月交易累计上限 | ≤ SP单月最大 | = SP值 |
| 是否有单年限额 | 开关 | — | — | 否 |
| 单年限额 | 金额 | 每年交易累计上限 | — | — |

> TP 可以为不同商户配置不同限额（本期由 TP 代配商户级限额）。

#### 1.2.3 计费配置（TP 可在 Floor Price 之上调整）

> 详细计费模型见 `productfee.md`。

| 字段 | 类型 | 说明 | 约束 | 默认值 |
| --- | --- | --- | --- | --- |
| TP 默认费率 | 费率 | TP 旗下商户的默认费率 | ≥ SP 商户最低报价 | = SP 商户最低报价 |
| 商户级费率 | 费率 | TP 为特定商户配置的费率 | ≥ SP 商户最低报价 | = TP 默认费率 |
| 费用扣取方式 | 下拉 | 内扣 / 外扣 | SP 支持范围内 | 外扣 |

#### 1.2.4 汇率配置

> 详细汇率模型见 `fx.md`。

| 字段 | 类型 | 说明 | 约束 | 默认值 |
| --- | --- | --- | --- | --- |
| TP 默认汇率加点 | 百分比 | TP 在 SP 返点基础上的加点 | ≥ 0 | 0 |
| 商户级汇率加点 | 百分比 | TP 为特定商户配置的加点 | ≥ 0 | = TP 默认加点 |

#### 1.2.5 VA 展示与排序配置

> 详细 VA 排序规则见 `vaandpayees.md` 4.2 节。

| 字段 | 类型 | 说明 | 可选值 | 默认值 |
| --- | --- | --- | --- | --- |
| VA 展示排序 | 规则 | 多 SP 的 VA 展示顺序 | BB优先（固定） | BB优先 |
| 重复 VA 能力去重 | 规则 | 同币种+同地区重复时用哪个 SP | BB优先（固定） | BB优先 |
| 商户自定义 VA 排序 | 开关 | 商户是否可自行调整 VA 顺序 | 是 / 否 | 否（本期固定） |

> 本期 VA 排序固定 BB 优先，不可配置。下期开放 TP/商户自定义排序。

#### 1.2.6 预约单配置

| 字段 | 类型 | 说明 | 约束 | 默认值 |
| --- | --- | --- | --- | --- |
| 预约单有效期 | 数字（小时） | 预约单超时时间 | ≤ SP默认有效期 | = SP值（72h） |
| 容差匹配阈值 | 百分比 | 入账金额允许偏差 | ≤ SP容差阈值 | = SP值（5%） |
| 过期自动处理 | 下拉 | 预约单过期后的处理方式 | 自动取消 / 转人工 | 自动取消 |

#### 1.2.7 风控配置（EX 平台层面）

| 字段 | 类型 | 说明 | 可选值 | 默认值 |
| --- | --- | --- | --- | --- |
| EX 风控开关 | 开关 | 是否启用 EX 平台风控 | 是 / 否 | 是 |
| 单笔大额预警阈值 | 金额 | 超过此金额触发人工审核 | — | 100,000 USD |
| 累计异常检测 | 开关 | 是否检测短时间内累计金额异常 | 是 / 否 | 是 |

> SP 有自己的风控（EX 不管），EX 平台层面的风控偏向限额和大额预警。交易频率限制由风控系统独立管理，不在产品配置中。

#### 1.2.8 退款配置

| 字段 | 类型 | 说明 | 约束 | 默认值 |
| --- | --- | --- | --- | --- |
| 退款汇率容忍阈值 | 百分比 | 本期使用 SP 级配置 | = SP值（本期不可改） | 1% |
| 退款手续费策略 | 下拉 | 本期使用 SP 级配置 | = SP值（本期不可改） | 不退承兑费 |

> 本期退款配置统一使用 SP 级定义，TP/商户不可修改。下期考虑开放 TP 级配置。

**配置约束规则：**

```
限额：  商户级限额 ≤ TP 限额 ≤ SP 限额上限
费率：  商户级费率 ≥ TP 默认费率 ≥ SP 商户最低报价（Floor Price）
汇率：  商户级加点 ≥ 0，TP 默认加点 ≥ 0
退款：  本期统一用 SP 级配置，TP/商户不可改
VA排序：本期固定 BB 优先，不可配置
风控：  SP 自有风控（EX 不管）+ EX 平台风控（限额预警，TP 可配阈值）
```
