一、客户背景

1.1 公司概况

传承环球是一家总部位于福建的综合性贸易集团，体系内拥有上市公司，主营耳机产品，以日本为主战场，在亚马逊（Amazon Japan）、雅虎（Yahoo! Shopping）、乐天（Rakuten）等主流电商平台均有布局，具备成熟的跨境电商运营体系。

1.2 支付现状与诉求

维度情况
自有平台已搭建支付聚合平台，后端对接多家支付服务商（Kun、Tazapay、Dogpay、我司等），具备多渠道比较和切换能力
牌照资质已在澳洲获得支付牌照，走自有澳洲渠道非常通畅
业务意向老板最开始有意向做交易所方向，但尚不清楚具体路径和落地方式
核心诉求安全、合规、时效、丝滑度——按此优先级排列

1.3 客户特征分析

  • 地域背景：基于福建，天然具备丰富的跨境贸易和产业资源
  • 合规意识高：上市公司合规要求严格，主营业务以合规业务为主
  • 行业认知深：对 Web3 的理解基本来自对接渠道和我司销售沟通
  • 客户画像：符合目标大客户——独立性强、接受点位报价、注重合规和服务质量、粘性高

二、会议核心议题

2.1 客户选择与市场定位（已明确）

客户类型决策依据
散客❌ 放弃月交易额低于几十万，服务成本高、投入产出比低
比价客户❌ 放弃洽谈周期长、价格竞争激烈，千1～千2费率无利润空间
大客户（如省广类型）✅ 重点开发独立性强、接受点位报价、注重安全合规和时效，粘性高
海外华人客户🔄 长期关注福清/长乐海外餐饮群体业务量大但分散
新兴市场客户🔄 逐步拓展如西班牙游乐场客户，目前量不大但有潜力
结论：专注服务大客户，不做散客，不比价格比服务品质。

2.2 产品与服务问题清单

① 时效性问题 紧急 🔴

  • 客户最关注资金到账时效，T+1 是底线要求
  • 客户从打款开始计算时间，而非从我方处理开始——认知错位
  • 即便其他平台贵千1，客户也会因时效选择竞品

② 汇率波动问题 关注 🟠

  • 小币种汇率波动高达千4～千5，与约定的万6差距巨大
  • 无稳定汇率报价机制

③ 白标后台功能缺失 严重 🔴

  • 无法给客户设置调价等基本操作权限
  • 调手续费需一天且多部门审查
  • 需要让服务商在不超约定红线下可自主配置

④ 通知机制缺失 严重 🔴

  • 美国手机号不能注册等变更无提前通知
  • 客户进件时才知道限制，信任度下降

⑤ 反馈处理机制 问题 🔴

  • 反馈问题未处理是最大毛病
  • 微信群反馈无人回复，需建立"问题不过夜"机制

2.3 竞争环境

竞争对手动态启示
光子易报出千1.5低价抢市场避免价格战
Tazapay接入费降至5000美金，3-4月放接口可评估为潜在合作 SP
Kun数据穿透严重——申请后数据瞬间外流数据安全是红线
Paypal在放接口，产品评价一般品牌大≠产品好

2.4 合规与监管

  • 合规灵活处理：不能一刀切，需根据各国法律灵活操作
  • 中国监管趋紧：预计今年会有"杀鸡儆猴"行动
  • 需加强对中国监管政策的解读和研究，提前应对

2.5 人员与沟通

  • 部分人员专业能力与岗位不匹配
  • 外聘人员权限大但未入职主体公司，风险大
  • 群聊回复不及时，领导进群提高响应优先级

三、关键洞察：Web3 vs Web2 体验落差

3.1 传承环球的核心判断

"你们最大的问题是用 Web2 的要求来做 Web3，但客户正是因为 Web2 不好用才来用 Web3。"

3.2 现实链路分析

客户期望: Web3 充值/提现 → 秒级或分钟级到账 实际链路: Web3 交易 → 合规审核 → 渠道清算（银行 = Web2）→ 到账 ↑ ↑ 合规瓶颈 渠道瓶颈（银行清算速度）

链上部分可以很快，但 OnRamp/OffRamp 最终落地到银行体系，依然是 Web2 的清算速度和合规审核。

3.3 行业性认知 Gap

  • 拓客有教育成本：需在售前管理预期，"链上秒到 ≠ 法币秒到"
  • 谁先缩短 Gap 谁赢：不可能消除银行延迟，但压缩到最短就是壁垒
  • 产品透明度是关键：让客户实时看到资金在哪一步

4.4 更深层的问题：EX 作为 SaaS 的价值是什么？

现状：EX ≈ 渠道对接层

当前：传承环球 ──商务+技术──→ BB/IPL（= EX 团队） 客户感知：EX = BB 的门面 未来：传承环球 ──商务──→ SP（直接谈费率、合规、商务条款） ──技术──→ EX（仅 API 对接） 客户感知：EX = 技术中间件

致命挑战：技术对接层护城河正在消失

能力当前1-2年后（AI 辅助开发成熟）
API 对接需专业团队，数周AI 生成代码，数天甚至数小时
多 SP 聚合有一定门槛标准化工程，门槛大幅降低
前端白标需完整产研团队AI 快速生成，定制成本极低
数据报表开发周期长AI + 低代码可替代
如果 EX 的价值仅停留在"技术对接层"，护城河会在 AI 浪潮中被迅速抹平。 任何有技术能力的租户都可以用 AI 快速搭建自己的聚合层。

关键转折：客户到底在为什么付费？

终极问题的回答

EX 凭什么存在？

不是凭代码——代码 AI 能写。不是凭 SP 数量——客户自己能对接。不是凭数据——数据是结果不是起点。

EX 凭的是合规运营能力——把 BB/IPL 的牌照和合规 know-how 产品化。 牌照可以买到，但怎么做合规运营、怎么审核进件、怎么监控风险、怎么应对监管——这些 know-how 不能 vibe code，不能 AI 生成，不能一夜建成。

合规是 1，清结算和技术是后面的 0。没有合规这个 1，后面多少个 0 都是零。

但必须正视：合规恰恰是当前最被客户诟病的短板。 核心价值主张和当前交付能力之间的 gap，是 EX 最紧迫的战略课题。

六、行动计划

阶段一：止血（已在进行，但无法根治）

#事项责任方现状
1建立"问题不过夜"机制：领导进群，指定响应人，设定 SLA全公司✅ 已在做
2汇率波动临时方案：高波动小币种提供固定汇率窗口或提前告知BB 渠道✅ 已在做
3产品变更提前通知：规则/功能变更提前通知，建立公告机制产研+合规✅ 已在做
但必须正视：这些措施治标不治本。

止血措施解决的是沟通效率和信息透明度问题，但核心痛点——时效——不是靠运营能力能解决的。根本原因是 Web2 银行清算基础设施在支撑 Web3 业务场景。问题是结构性的，不是运营性的。 真正的解法在阶段二。

阶段二：渠道能力建设 — 两条腿并行（1-3个月，最高优先级）

腿1：BB 自身渠道与合规增强

#事项责任方说明
1梳理渠道短板清单BB 渠道明确缺哪些地区、币种、银行通道，量化 gap
2拓展银行通道BB 渠道主力币种至少2条通道，关键地区有冗余
3建立汇率报价机制BB 渠道稳定报价体系，小币种波动控制千1以内
4合规分级处理BB 合规按国家/业务类型差异化，明确进件 SLA
5合规政策透明化BB 合规支持/不支持的主体类型提前公示，变更提前通知

腿2：EX 对接新 SP

#事项责任方说明
1筛选候选 SPEX BD+渠道按标准（互补、数据安全、资质、稳定性、成本）筛选
2评估展会接触机构EX BD如 Tazapay（3-4月放接口、接入费5000美金）等

4联调测试+验证产研+渠道核心链路全流程验证，确认 SLA 达标
关键认知： EX 对接新 SP 的周期（技术联调+合规审核+测试验证）不一定比 BB 自己拓渠道更短。所以必须两条腿同时跑。

7.3 续费策略：坦诚 + 分阶段 + 给利益

原则：不画饼、不承诺做不到的事。用坦诚换信任，用让利换时间，用阶段性成果换长期合作。

① 坦诚沟通能力边界

  • 直接告知哪些能做、哪些暂时不能、预计何时能做到
  • 传承环球是行业老手，画饼适得其反；坦诚反而建立信任
  • 给出明确能力提升时间表（与阶段二渠道建设对齐）

② 续费方案：降低门槛，绑定长期

策略具体方案

八、结论

此次出差最大的收获，不是发现了渠道短板，而是迫使我们想清楚了一个根本问题：EX 到底卖什么？

答案不是技术——代码 AI 能写，客户自己几天就能对接一个 SP。
答案不是渠道聚合——客户自己有聚合系统，传承环球已经同时接了 Kun、Tazapay、Dogpay。
答案不是数据——数据是结果不是起点，鸡生蛋问题短期无解。

答案是合规保护伞。 传承环球走的是代理模式，用 BB 的牌照，BB 承担合规责任。客户买的不是 API 对接，而是"安全感"和"责任转移"。传承环球自己有澳洲牌照，甚至能买更多牌照——但他们不知道怎么做合规运营。牌照是门票，合规 know-how 才是护城河。

这也解释了商业模型：大客户愿意付 SaaS 订阅费 + 交易费（买合规保护），小客户接受交易抽佣（必须走持牌方）。大客户是利润来源，小客户是规模来源。

所以 EX 不是技术对接公司，而是合规运营平台——把 BB/IPL 的牌照和合规 know-how 产品化，变成可配置、可交付、可规模化的服务。合规是 1，清结算和技术是后面的 0。没有合规这个 1，后面多少个 0 都是零。

但讽刺的是：合规恰恰是当前客户诟病最多的短板。 一刀切、不通知、不支持美国主体——核心价值主张和当前交付能力之间的 gap，才是 EX 最紧迫的战略课题。

具体行动：

1. 合规能力产品化（最高优先级）：统一进件审核、合规规则引擎（可配置不一刀切）、风控中台、合规状态透明化
2. 渠道两条腿并行：BB 增强自身渠道 + EX 对接新 SP，谁先跑通谁先上线
3. 传承环球必须留住：坦诚能力边界、费率让利换时间、聚焦合规业务场景、以续费谈判锚定"合规保护伞"价值

每一个因为体验差流失的客户，不仅是客户损失，更是在圈子里形成负面口碑。但比留住客户更重要的，是想清楚自己卖什么。这次我们想清楚了：EX 卖的是合规运营能力，不是代码。接下来要做的，是把这个能力真正建起来。
