# Offramp 商业方案

> 文档状态：待业务、产品、资金、财务、风控合规及管理层评审  
> 方案定位：面向已通过 B 标准入网的企业客户，提供 USDC/USDT 兑换法币并完成对外付款的一体化服务  
> 主体分工：EX 负责产品与系统建设；BB 作为持牌机构负责客户业务承接及后台资金运营  
> 商业目标：扩大客户可服务范围，提高钱包资金转化率，通过报价、专属服务及 BB 资金运营获得可持续收入  
> 关联文档：[Offramp产品BRD.md](./Offramp产品BRD.md)、[Offramp产品PRD.md](./Offramp产品PRD.md)、[MOR模式-运营主体SP最终方案.md](./MOR模式-运营主体SP最终方案.md)

## 一、执行摘要

企业客户持有 USDC/USDT 后，最终通常需要支付供应商、服务商、广告平台、物流商或提现至银行账户。市场上的钱包、兑换和法币付款能力往往由不同服务商提供，客户需要自行处理渠道准入、资金调拨、兑换价格、付款人展示和异常追踪。

EX Offramp 将这些能力整合成统一付款产品：客户只需提交付款目标并比较两份报价，系统在后台完成客户路径判断、钱包选择、兑换执行和法币付款。

客户看到的报价始终保持简单：

- **主路径报价**：系统根据客户服务路径提供标准报价或专属报价；
- **1:1 平价报价**：以 1 USDC/USDT = 1 USD 的名义兑换关系计价，服务费和其他成本独立展示。

客户可以使用已有钱包余额付款，也可以先创建预约付款，再将 USDC/USDT 充值到目标执行钱包。预约付款通过充值前确定路线，减少跨钱包二次数币调拨，使 1:1 报价更具竞争力。

整体业务收益不依赖单一收费项，而来自服务费、兑换点差、专属服务收入、渠道分润及 BB 资金运营收益。EX 负责产品能力建设和业务编排；BB 作为持牌机构负责后台资金运营。所有收入均需建立在全链路成本可见、风险限额明确和客户报价透明的基础上。

## 二、客户是谁

### 2.1 客户共同基线

本产品服务已经通过 B 标准入网的企业客户。A 是否直接服务属于 A 与 B 风险偏好及服务范围差异，不代表客户未通过 B 入网，也不应将专属出款宣传为“高风险客户产品”。

### 2.2 核心客户群体

| 客户群体 | 典型特征 | 主要需求 |
| --- | --- | --- |
|贸易企业 | 收入或资产以 USDC/USDT 持有，需要支付供应商 | 兑换、供应商付款、可解释的付款流水 |
| 广告及数字服务企业 | 高频向广告平台、技术服务商付款 | 到账稳定、付款人展示、时效可预测 |
| 物流、仓储及采购企业 | 多收款人、多国家、多币种 | 统一管理收款人和付款记录 |
|  |

### 2.3 客户选择偏好

客户通常在以下价值之间权衡：

- 更高的预计到账金额；
- 更确定的兑换关系；
- 更快的付款时效；
- 更少的材料和操作步骤；
- 银行流水中的付款人展示；

产品不应让客户理解底层钱包和渠道结构，而应把这些差异转换为清晰报价、时效和服务承诺。

## 三、客户痛点

### 3.1 钱包与付款能力割裂

客户持有 USDC/USDT 的钱包服务商不一定具备法币付款能力。客户需要自行把数币转至承兑或 Offramp 渠道，产生额外操作、网络费、到账等待和转错风险。

### 3.2 兑换和付款流程割裂

客户可能需要先兑换，再将法币转给另一个付款渠道。多个订单和资金流水难以关联，失败或退票时也难以判断由谁处理。

### 3.3 报价不透明

市场常以“低费率”或“1:1”宣传产品，但未完整展示钱包费、链上费、调拨费、兑换价差、付款费和中转行费用。客户无法比较真实到账金额。

### 3.4 渠道服务范围不同

客户已通过 B 入网，但可能无法由 A 直接服务。如果产品只依赖单一标准路径，客户将因渠道风险偏好差异而流失。

### 3.5 1:1 报价受到调拨成本侵蚀

部分 1:1 渠道要求同时使用其钱包。客户资金已经进入独立钱包后，再调拨至 1:1 渠道会增加成本，使名义上有吸引力的 1:1 报价失去竞争力。

### 3.6 付款人和业务关系管理复杂

不同场景需要 POBO 或非 POBO，且各渠道对付款人、收款人和交易材料的要求不同。客户难以理解和维护多套流程。

### 3.7 异常缺乏统一处理

数币调拨、兑换、付款、退票和退款由不同机构完成时，客户缺少统一订单和责任入口，容易产生长时间结果未知和多方沟通。

## 四、我们的解决方案

### 4.1 统一 Offramp 付款入口

EX 将钱包、兑换、法币付款、收款人、POBO/非 POBO、订单、对账和异常处理整合为统一产品。客户不需要分别操作钱包、兑换和付款渠道。

### 4.2 系统先确定服务路径

标准出款和专属出款不交给客户按价格选择：

| 系统判断 | 主路径报价 |
| --- | --- |
| A 可直接服务 | 标准报价 |
| A 不直接服务、其他SP服务 | 专属报价 |
| A 不直接服务、专属路径不可用 | 不提供主路径报价，可单独判断 1:1 是否可用 |



### 4.3 客户比较两份报价

系统完成路径筛选后，客户比较：

1. **标准或专属报价**：使用实时/市场化计价机制；
2. **1:1 平价报价**：名义兑换关系为 1:1，费用单独展示。

每份报价至少展示：

- 兑换机制；
- 服务费；
- 调拨费；
- 其他已知费用；
- 预计到账金额；
- 预计付款时效；
- 报价有效期和重要限制。

#### 4.3.1 客户什么时候选择标准报价

标准报价适合以下场景：

| 场景 | 客户选择原因 |
| --- | --- |
| USDT/USDC 相对 USD 处于溢价 | 市场化兑换可能获得高于 1:1 的毛兑换金额 |
| 标准报价扣费后到账更多 | 即使汇率接近，标准路径的服务费、调拨费可能更低 |
| 客户更关注综合到账金额 | 客户愿意接受成交时点汇率变化，以争取更优价格 |
| 已有钱包余额可直接执行 | 无需跨钱包调拨时，标准报价的全链路成本更低 |
| 大额交易可以获得更优市场报价 | BB 资金运营或外部渠道可根据交易规模提供更好的点差 |

标准报价的主要价值是“按当前市场条件争取更优的综合到账金额”，而不是保证固定兑换关系。

#### 4.3.2 客户什么时候选择 1:1 平价报价

1:1 平价报价适合以下场景：

| 场景 | 客户选择原因 |
| --- | --- |
| 标准实际净报价明显低于 1:1 | 无论原因是 USDT/USDC 价格偏离、市场买卖价差扩大，还是渠道报价成本上升，只要 1:1 扣除全部费用后的预计到账更高，1:1 就具有价格优势 |
| 财务需要提前计算充值金额 | 预约付款下可在充值前确定兑换逻辑和费用，便于资金安排 |
| 客户不希望承担短时汇率变化 | 即使费用略高，也愿意为确定性付费 |
| 预约付款可以直达 1:1 渠道钱包 | 避免二次数币调拨后，1:1 的综合报价更有竞争力 |

1:1 的主要价值是“金额确定性 + 特定市场条件下的价格优势”，但不代表零费用，也不代表在所有市场条件下都优于标准报价。

#### 4.3.3 近期真实数据与业务报价示例

需要区分“公开市场价格偏离”和“客户实际可成交报价”：

- OKX 历史数据记录，2026-06-30 USDT/USD 日内低点约为 **0.9982**；
- CoinGecko 历史数据记录，2026-06-22 USDT 价格约为 **0.998884 USD**；
- 上述公开数据属于千分位偏离，更接近稳定币正常波动，不宜表述为“大幅脱锚”。

数据来源：[OKX Tether price history](https://www.okx.com/en-eu/price-history/tether-usdt)、[CoinGecko Tether historical data](https://www.coingecko.com/en/coins/tether/historical_data)。

以 OKX 的 0.9982 为例，客户卖出 100,000 USDT：

| 项目 | 标准市场毛结果 | 1:1 毛结果 |
| --- | ---: | ---: |
| 客户卖出 | 100,000 USDT | 100,000 USDT |
| 兑换率 | 0.9982 | 1.0000 |
| 毛兑换金额 | 99,820 USD | 100,000 USD |
| 1:1 毛优势 | — | 180 USD |

最终是否选择 1:1，还要扣除两边各自的服务费、调拨费和付款费。如果 1:1 比标准报价多收的费用超过 180 USD，标准报价仍可能更优。

如果 BB 或渠道当时给客户的实际标准报价明显低于 0.9982，例如受到渠道买卖价差、流动性成本或费用影响，那么 1:1 的真实优势会更大。但这应描述为“标准实际净报价明显低于 1”，不能仅凭渠道报价认定 USDT 在公开市场发生了大幅脱锚。正式商业案例应补充 BB 当日真实报价、渠道报价时间和费用快照。

#### 4.3.4 前端推荐原则

系统不应仅用“费率较低”推荐产品，而应比较两份报价的最终结果：

```text
标准预计到账
= 数币数量 × 标准对客汇率 − 标准全部费用

1:1预计到账
= 数币数量 × 1.0000 − 1:1服务费 − 调拨费 − 其他费用
```

- 标准预计到账更高：标记“预计到账更多”；
- 1:1 预计到账更高：标记“当前更划算”；
- 两者接近：同时展示“市场化价格”和“金额更确定”的差异，由客户选择；
- 钱包付款的 1:1 因调拨成本不具优势：提示客户改用预约付款重新获取报价；
- 推荐标签只解释当前订单的经济结果，不承诺未来汇率，也不替客户自动选择。

### 4.4 钱包付款

客户直接使用现有钱包余额发起付款。

优势：操作快、充分转化现有钱包资金。

限制：如果选择的执行渠道与现有钱包不同，需要进行数币调拨。相关钱包费和链上成本必须进入报价，不能由平台隐性承担。

### 4.5 预约付款

客户先提交付款目标并选择报价，系统再为该订单分配目标执行钱包的充值地址：

```mermaid
flowchart LR
    A["创建预约单"] --> B["选择主路径或1:1报价"]
    B --> C["生成目标渠道充值地址"]
    C --> D["充值USDC/USDT"]
    D --> E["Convert"]
    E --> F["Payout"]
```

预约付款的核心商业价值是充值前完成路由，使资金直接进入最终执行钱包，减少二次调拨成本。它不是第三种兑换报价，而是一种资金进入和付款发起方式。

### 4.6 后台执行

客户不感知实际由谁完成兑换。后台可以根据成本、头寸、限额和时效选择：

- BB 资金运营团队提供兑换；
- 外部渠道完成兑换；
- 钱包+Convert+Payout 一体化渠道；
- 独立钱包调拨至 Offramp 渠道；
- 专属路径完成交易。

报价确认后，后台执行变化不得降低客户到账金额或改变付款模式；经济结果变化时必须重新报价。

## 五、产品价值

### 5.1 对客户的价值

- 一个入口完成充值、兑换和付款；
- 只比较两份清晰报价，无需理解底层渠道；
- 可在市场化报价和 1:1 确定性之间选择；
- 既可以使用已有余额，也可以预约后充值；
- 通过预计到账金额比较真实成本；
- 统一追踪付款、退票、退款和调单结果；
- A 不直接服务时，仍可能获得专属出款能力。

### 5.2 对 B/租户的价值

- 将客户钱包余额转化为交易收入；
- 扩大 B 已入网客户的 Offramp 可服务范围；
- 增加客户留存和钱包资金沉淀；
- 减少销售逐个解释渠道和价格的成本；
- 可通过白牌或 API 向更多企业客户输出能力。

### 5.3 对 EX 的价值

- 从渠道接口集成方升级为钱包、兑换、付款和成本路由平台；
- 不依赖单一渠道，增强议价能力和业务稳定性；
- 新增渠道时不需要新增对客产品；
- 形成全链路成本、报价、渠道质量和客户行为数据；
- 为 BB 的 Onramp、Offramp 和资金运营协同建立产品基础。

### 5.4 对 BB 资金运营团队的价值

- 获得稳定且可预测的客户流量；
- 使用订单流进行数币和法币头寸管理；
- 在风险限额内获取兑换收益；
- 通过 Onramp/Offramp 净额降低外部兑换频次；
- 比较自营和渠道执行的真实收益贡献。

## 六、收费方案

### 6.1 收费原则

1. 客户看到全部已知费用和预计到账金额。
2. 1:1 不等于免费，服务成本通过明确费用体现。
3. 专属报价的额外收费对应额外服务和执行成本，不是风险通行费。
4. 钱包调拨成本由实际资金路径决定，不应使用统一虚高费率覆盖。
5. 具体费率按客户、金额、币种、渠道和商业协议配置，不在方案未批准前写死。

### 6.2 建议收费结构

| 收费项 | 适用场景 | 计费方式建议 |
| --- | --- | --- |
| 基础服务费 | 全部成交订单 | 交易金额比例、固定费或阶梯费率 |
| 兑换点差 | 实时/市场化报价 | 对基准汇率增加经批准点差 |
| 1:1 服务费 | 1:1 平价报价 | 交易金额比例，可设置最低收费 |
| 专属服务费 | 专属报价 | 综合费率或单独服务费 |
| 数币调拨费 | 钱包付款且需要跨钱包调拨 | 按实际网络和钱包成本计费 |
| 法币付款费 | 银行、本地或跨境付款 | 按币种、国家和付款方式计费 |
| POBO 服务费 | 渠道支持且客户选择时 | 按单或按金额计费，是否单列待确认 |
| 异常处理费 | 客户原因导致的错币、错链、退票等 | 按实际处理成本及合同约定 |

### 6.3 对客报价公式

```text
预计到账金额
= 数币数量 × 对客兑换率
− 基础服务费
− 专属或 1:1 服务费
− 数币调拨费
− 已知付款及其他费用
```

如果预约付款以“目标到账金额”下单，则反向计算客户应充值数币数量，并明确报价有效期和少充、超额、迟到处理规则。

### 6.4 客户分层定价

可根据以下因素设置费率档位：

- 月交易量和单笔金额；
- 钱包付款或预约付款；
- 标准、专属或 1:1 报价；
- 币种、网络和付款国家；
- 客户材料和运营复杂度；
- 渠道实际成本及稳定性；
- 合作期限和承诺交易量。

客户分层定价不得突破准入和渠道规则。

## 七、收入来源

### 7.1 交易服务收入

通过每笔 Offramp 的基础服务费、付款费或综合费率获得收入，是最稳定的收入来源。

### 7.2 兑换收入

- 外部渠道报价与对客报价之间的经批准点差；
- BB 资金运营团队完成兑换产生的点差收益；
- 1:1 报价对应的服务费。

### 7.3 专属服务收入

专属报价包含额外的路径、运营和服务成本，可通过更高综合费率或单独服务费获得收入。内部成本和责任以 MOR 最终方案及商业协议为准。

### 7.4 渠道及生态分润

根据与钱包、兑换、付款和租户合作方的协议，可能形成渠道返佣、租户分成或服务分润。

### 7.5 BB 资金运营收益

BB 资金运营团队通过流量净额、头寸管理和执行优化获得收益。该收益属于 BB 的后台执行结果，不作为客户产品名称或无约束利润来源。

## 八、成本结构

### 8.1 单笔直接成本

- 钱包服务费；
- 链上网络费；
- 跨钱包调拨费；
- 外部兑换成本或 BB 自营资金成本；
- 法币付款渠道费；
- 专属服务执行成本；
- POBO 或其他增值能力成本；
- 中转行、退票和退款成本。

### 8.2 运营成本

- 客户和渠道运营；
- 报价、订单和对账系统；
- 调单、补件和异常处理；
- 财务、资金和审计；
- 法务、合规及渠道管理；
- 客户服务和销售支持。

### 8.3 风险成本

- 渠道冻结和资金占用；
- 自营汇率敞口；
- 报价与成交时点差异；
- 数币错链、调拨失败和结果未知；
- 法币退票、退款和无法追回费用；
- 渠道关户或退出造成的迁移成本。

## 九、盈利模型

### 9.1 单笔贡献利润

```text
单笔收入
= 服务费收入
+ 兑换点差收入
+ 专属/1:1增值收入
+ 渠道分润

单笔贡献利润
= 单笔收入
− 钱包成本
− 链上及调拨成本
− 兑换执行成本
− 法币付款成本
− 专属服务成本
− 异常及资金占用成本
```

### 9.2 盈利的主要驱动因素

- 交易规模和复购频率；
- 预约付款占比及调拨成本下降；
- BB 自营资金与渠道兑换的择优能力；
- Onramp/Offramp 流量净额比例；
- 钱包和渠道议价能力；
- 专属路径的服务成本和费率匹配；
- 自动化水平和人工处理率；
- 付款成功率、退票率和异常损失。

### 9.3 盈利场景判断

| 场景 | 盈利特征 |
| --- | --- |
| 标准报价 + BB 自营兑换 | 有机会获得较高点差收益，但 BB 承担头寸和市场风险 |
| 标准报价 + 渠道兑换 | 收益较稳定，依赖渠道采购价和付款成本 |
| 专属报价 | 客单价较高，但材料、运营和执行成本也较高 |
| 钱包付款 + 1:1 | 调拨成本可能侵蚀利润，需要按单计价或设置阈值 |
| 预约付款 + 1:1 | 减少二次调拨，通常比钱包付款更有报价和利润优势 |
| 钱包+Off 一体化渠道 | 路径短、成本清晰，但需关注渠道集中和冻结风险 |

### 9.4 盈亏平衡分析框架

具体数字待商务、资金和财务输入后计算：

```text
月度贡献利润
= 月成交金额 × 综合净收益率
− 月固定运营成本
− 技术及合规摊销

盈亏平衡交易量
= 月固定成本 ÷ 单位交易金额贡献利润率
```

评审时至少需要补充：预期月交易额、平均客单价、各报价选择率、平均收费率、渠道成本、自营成本、异常率和固定投入。

## 十、成本优化与 BB 资金运营

### 10.1 充值前路由

预约付款先确定报价和目标钱包，再让客户充值，是降低 1:1 调拨成本的首要策略。

### 10.2 全链路成本择优

系统不能只比较兑换费率，而应比较：

> 钱包费 + 链上费 + 调拨费 + 兑换成本 + 付款费 + 专属服务成本 + 资金占用 + 预期异常损失

满足客户路径和风险约束后，再综合优化客户到账金额、平台毛利、成功率和时效。

### 10.3 BB 自营与渠道竞争

主路径报价可以比较 BB 自营资金和外部渠道的内部成本。客户看到的仍是一份标准或专属报价，不感知后台流动性来源。

### 10.4 流量净额

结合 Onramp 和 Offramp 订单，在合规、会计和资金隔离允许的前提下，对数币和法币需求进行净额管理，只对净差额进行外部兑换。

### 10.5 批量调拨

对已有钱包余额产生的跨渠道需求，可按币种和渠道汇总调拨，摊薄网络费。批量调拨不得导致超出对客 SLA，也必须保持客户资金可追溯。

### 10.6 调拨成本阈值

当钱包付款的 1:1 调拨成本超过可接受比例时：

- 不提供不可持续的成交报价；
- 提示客户使用预约付款；
- 或要求达到最低交易金额。

### 10.7 渠道运营头寸

成熟阶段可以在高频渠道保留有限运营头寸，减少逐笔调拨，但必须设置额度、集中度、冻结预警、对账和退出机制。

## 十一、竞争优势

### 11.1 产品简单，后台灵活

客户只比较两份报价；EX 可以在后台组合不同钱包、兑换、付款和资金来源。新增渠道不会导致前台产品数量持续增加。

### 11.2 服务覆盖更广

标准路径、专属路径和 1:1 能力共同覆盖不同服务场景，降低单一 A 或单一渠道风险偏好造成的客户流失。

### 11.3 预约付款降低结构性成本

市场上常见方案在客户入账后才寻找 Offramp 渠道。EX 通过预约付款在充值前确定路线，能够从资金路径上减少调拨，而不仅依赖渠道降价。

### 11.4 BB 自营资金能力

BB 拥有资金运营团队，可以在渠道报价之外进行头寸、净额和执行优化；EX 将该能力产品化并用于报价与订单编排，从而提高报价竞争力及利润空间。

### 11.5 全链路数据与对账

统一订单关联 Deposit、Convert 和 Payout，可形成渠道成功率、真实成本、客户行为和异常损失数据，持续优化路由和定价。

## 十二、销售与客户沟通策略

### 12.1 对客价值表达

- 一个入口完成数币兑换和全球付款；
- 两份报价，费用和预计到账金额清晰；
- 已有余额可以直接付；
- 提前预约可以减少资金调拨成本；
- 支持不同付款人展示和收款场景；
- 全流程订单及异常可追踪。

### 12.2 禁止使用的销售话术

- “专属出款可以绕过审核”；
- “支付更高费用就一定能付款”；
- “1:1 等于零费用”；
- “保证到账、不会冻结、不会调单”；
- “所有客户、国家和行业都支持”。

### 12.3 客户推荐逻辑

- 已有余额且重视速度：优先钱包付款；
- 尚未充值且有明确付款目标：优先预约付款；
- 重视综合到账金额：比较主路径报价；
- 重视金额确定性：比较 1:1 平价报价；
- 钱包付款走 1:1 成本较高：建议改用预约付款。

## 十四、主要风险与应对

| 风险 | 影响 | 应对方向 |
| --- | --- | --- |
| 渠道协议或责任不清 | 资金异常时难以追责 | 上线前完成签约、责任和赔付确认 |
| 1:1 调拨成本过高 | 报价无竞争力或亏损 | 预约付款、成本阈值、批量调拨 |
| 自营汇率敞口 | 市场波动导致损失 | 限额、实时监控、暂停和对冲机制 |
| 渠道冻结或关户 | 资金占用和服务中断 | 多渠道储备、集中度限制、退出预案 |
| 专属服务成本失控 | 高收入但低利润 | 独立核算材料、运营、调单和异常成本 |
| 报价与成交不一致 | 客诉或平台损失 | 明确报价有效期、成交时点和滑点规则 |
| 错币、错链或迟到 | 资金恢复成本高 | 专属地址、明确提示、异常 SOP |
| 退票和退款 | 额外费用及汇率损失 | 明确返还方式、费用和责任方 |
| 产品表达误导 | 合规和声誉风险 | 统一销售话术和费用披露 |

## 十五、分阶段商业落地

### 阶段 1：验证产品与价格

- 上线标准/专属主路径与 1:1 两份报价；
- 钱包付款计入真实调拨成本；
- 预约付款先支持 USDC/USDT → USD；
- 专属出款引用 MOR 最终方案；
- 建立按产品和渠道的基础收入成本账。

验证重点：客户是否理解两份报价、1:1 选择率、预约付款转化率、真实调拨成本和单笔毛利。

### 阶段 2：扩大渠道与自动路由

- 接入更多钱包和 Offramp 渠道；
- 建立统一渠道能力和成本中心；
- 自动比较自营和渠道执行成本；
- 上线成功率、时效、毛利和异常监控；
- 建立客户分层定价。

### 阶段 3：规模化 BB 资金运营

- Onramp/Offramp 流量净额；
- 批量调拨；
- 有限渠道运营头寸；
- 多币种和多国家扩展；
- 基于成本、成功率、时效和风险的动态执行优化。
