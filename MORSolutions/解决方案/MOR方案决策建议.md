# MOR 方案三方案对比

> **文档定位**：为服务 **分类 2 客户**（贸易实质合规、但不符合 A 当前贸易政策）确定承载/展业模式。目前有三套方案，本文做 **横向对比、风险权衡**，并 **专门分析方案三中 MOR 运营主体的角色（租户还是 SP）**。
>
> 三方案详见：
>
> - **方案一**：`B-A-OFFRAMP.md`（B-A 单渠道 + 独立 MOR 系统包装）
> - **方案二**：`MOR模式-受控租户方案.md`（受控租户展业，MOR 为客户角色内置租户）
> - **方案三**：`MOR作为B渠道方案.md`（把 MOR 运营主体抽象为 B 的一个非持牌渠道，背后聚合多渠道）

---

## 〇、术语（先统一名词）

| 名词                    | 含义                                                                                                           |
| ----------------------- | -------------------------------------------------------------------------------------------------------------- |
| **MOR（壳主体）** | 帮**真实客户** 提供资料（贸易背景/合同/invoice）的 **壳公司主体**；一个体系可有多个 MOR 壳         |
| **MOR 运营主体**  | 运营/管理**多个 MOR 壳** 的主体；**要么做租户（方案二）、要么做渠道（方案三），二者互斥**          |
| **租户**          | 展业主体（客户登录操作）；方案二里是 MOR 运营主体；**方案三里是任意非 MOR 主体的租户（BB/其他方）**      |
| **渠道**          | B 路由层的承载通道。**方案三中「渠道」= 一个 MOR 运营主体**（内含多个 MOR 壳），背后再聚合 A/Cregis/宝付 |

> 下文方案三的「**MOR 渠道**」均指「**MOR 运营主体作为 B 的一个渠道**」。

## 一、三方案一句话概述

| 方案             | 一句话                                                                                                                                                                                                                                                                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **方案一** | BB 直客展业；分类 2 客户报备 A 前先经**独立部署的 MOR 系统** 做关系与贸易材料包装；调单按是否 MOR 分流                                                                                                                                                                                                                                |
| **方案二** | BB 不直接展业，改由**少数受控租户** 展业；**MOR 退化为「客户为他人提供贸易材料」的角色**，能力内置租户后台；资金/报备主体 = 客户本身                                                                                                                                                                                            |
| **方案三** | 把**MOR 运营主体建成 B 路由层的一个「收数币→做 OffRamp」的非持牌渠道**（**只做渠道、不做租户**，与其他直连渠道同级，内含多个 MOR 壳）；背后聚合 **出款渠道（A/Cregis/宝付/其他）+ 钱包服务商渠道（BB 钱包）**；**B 只把 U 划到该渠道钱包（MOR 在 BB 的钱包），MOR 按自身路由选背后渠道→A2A 转账→以客户名义提现** |

## 三、方案三中 MOR（运营主体）是租户还是 SP？（专项分析）

> 先分清 **两个层次**：**MOR 壳** 是提供资料的壳公司；**MOR 运营主体** 是管理这些壳、并对外承担角色的主体。「是租户还是 SP」问的是 **MOR 运营主体**，而不是壳。

### 3.1 MOR 壳：既不是租户也不是 SP

- MOR 壳只负责 **给真实客户提供贸易材料**（合同/invoice），是 **材料提供者**；
- 它本身不展业、不承兑、不路由，**不构成租户或 SP**——只是被运营主体（渠道）内部使用的一个个壳。

### 3.2 方案二做「租户」、方案三只做「渠道SP（b的渠道）」——二者互斥

用定义对照：

- **租户 = 展业主体**（客户在其端登录操作）；
- **SP = KYC 审核 + 资金承接（承兑）+ 路由**（承接数币→法币、对接渠道）。

同一个 MOR 运营主体，摆到不同位置就是不同方案：

- **方案二 · MOR 运营主体做「租户」**：对客户展业、客户登录它（受控租户）；
- **方案三 · MOR 运营主体只做「渠道」**：**不展业、客户不登录它**；它只 **承接数币（B 划 U 过来）、聚合背后渠道并做 OffRamp（A2A 转账 + 以客户名义提现）**，因此实质是 **B 之下的 SP**；**展业的租户是另一个非 MOR 主体的租户（BB/其他方）**。

> **二者互斥**：一个 MOR 运营主体 **要么做租户（方案二）、要么做渠道（方案三）**，不同时充当两个角色。方案三的硬约束：**展业租户不能是 MOR 主体**。

### 3.3 MOR 运营主体作渠道时更像「SP」

> **前提澄清**：**SP 本身就有「持牌」与「非持牌」之分**，持牌与否并不改变「它是不是 SP」——它只是 SP 的一种 **资质属性**。因此方案三中的 MOR，本质是 **B 之下的一个 SP（通道）**，至于它是持牌还是非持牌，取决于其自身资质与法务安排。

MOR 在方案三里承担了一系列 **SP 式职能**：

- **资金承接**：**B 只把 U 划到 MOR 渠道钱包（MOR 在 BB 的钱包）**，MOR 收 U；
- **自主路由 + 出款**：MOR 按 **自身路由规则** 选背后渠道（A/Cregis/宝付），在渠道内做 **A2A（账户到账户）转账**；
- **以客户名义提现**：MOR **以客户名义在背后渠道发起提现**，法币出金到客户账户、附 KYB 材料。

它与最上层 SP（B）的关系：

- **位于 B 之下**：从 B 视角，MOR 是 **B 路由层里的一个渠道**（与其他直连渠道同级），B 仍是最上层的路由 + 承兑 SP；
- **资质属性待定**：MOR 作为 SP，**是持牌 SP 还是非持牌通道**是其属性问题——若非持牌，则实际持牌出款/合规能力落在 **背后渠道（A 等）**。

**结论**：**方案三中的 MOR ≈ 一个「SP / 嵌套通道」**——

- **对上**：是 B 的 **一个渠道**（被 B 路由选择）；
- **对下**：是 **聚合多个真实渠道（含钱包服务商渠道）的「收 U / 路由 / A2A / 以客户名义提现」主体**；
- 它 **偏 SP（通道）性质，而非租户**；**持牌/非持牌是其资质属性，需法务确定并据此签约**。

### 3.4 由此带来的关注点

- **资金/报备主体上移到 MOR/背后渠道**（区别于方案一/二的「客户本身」）——需确认合规上 **谁是报备与「以客户名义提现」责任主体**；
- **SP 的持牌/非持牌定性** 与背后渠道的责任边界（谁对 A 负责、KYB 由谁背书）；
- **BB 钱包作为「钱包服务商渠道」** 的资金隔离与对账（MOR 专用子钱包？）；
- **【待办·法务】SP 的协议如何签订**：MOR 作为 SP，需法务明确——
  - 其 **持牌 / 非持牌资质定性**，以及对应的 **展业与「收 U / 路由 / A2A / 以客户名义提现」合规边界**；
  - **协议主体与层级**：B ↔ MOR（SP）如何签、MOR ↔ 背后渠道（A/Cregis/钱包服务商）如何签、是否需要客户侧授权/委托；
  - **责任与权属划分**：承兑、报备、A2A 转账 / 以客户名义提现、KYB 背书、调单配合各自的责任主体与追偿；
  - 非持牌情形下 **借背后持牌渠道落地** 的合规合法性与协议兜底条款。

## 四、风险与权衡

- **方案一**：能跑流程，但直客展业存在 **结构性矛盾**——既依赖销售/运营，又必须对其隐瞒 MOR，靠系统特殊处理维持，**漏处理即暴露**；且需新建独立系统，责任高度集中于 BB。
- **方案二**：从展业结构上 **消除该矛盾**（BB 不触达客户、租户天然隔离），复用租户既有能力、投入小、可灰度；资金/报备回归客户，权责清晰。
- **方案三**：把包装/调单收敛进 **一个渠道**，客户零感知、可聚合多背后渠道、便于扩展；但 **MOR 成了 B 之下的 SP**，**资金/报备主体上移**，对 **合规定性与责任划分、钱包资金隔离、SP 协议签约** 要求更高。

## 五、需要拍板 / 确认

- 方案三中 **MOR 作为 SP** 与 **资金/报备责任主体**（MOR/背后渠道 vs 客户）；
- **【法务】SP 的协议如何签订**：B↔MOR、MOR↔背后渠道（A/Cregis/钱包服务商）的协议主体、层级与责任划分；
- 一期 **灰度范围与规模上限**（受控租户、MOR 数量、交易量阈值）；
- 与 A 合规确认：**反洗钱验证账户开立方式与成本**、**调单异常处置口径**。
