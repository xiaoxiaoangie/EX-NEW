MP / Dashboard

Dashboard（工作台）

菜单设计原则：
1. Assets（资产）= 同名充提 + 兑换（自己的钱左手倒右手，纯钱包内操作）；承兑后要付出去的走Transfer Out
2. Transfer In（转账收款）= 收款工具管理（Collection Tools）+ 汇款人维护 + Payins（所有入金）+ Onramp（收法币→自动换U）
3. Checkout（收单）= 商户管理收款订单，底层收款方式（VA/地址/在线支付）由系统自动选择
4. Transfer Out（转账付款）= Beneficiary（收款人）+ Payouts（所有出金：法币付款+链上付币+同名提现+同名提币）+ Remittance Orders（收→换→付一单到底，编排层）
5. Cards（发卡）= 通过卡消费方式付款
6. Transfer In vs Checkout：Transfer In 管收款工具（商户主动申请的VA/地址），Checkout 管收款订单（底层工具由系统分配，对商户透明）
7. Payins vs Payouts 对称：Payins = 所有入金，Payouts = 所有出金；Onramp（收法币→换U）和 Remittance（收→换→付）是编排层
SP透明 & 账户结构规则：
1. 底层账户结构：每个SP给商户开独立账户（IPL有USD账户，BB也有USD账户）
2. 商户端不展示SP：商户只看到按币种合计的总余额，SP明细只在租户端展示
3. 充值/提现/付款：商户不选SP，系统根据默认配置和优先级自动选择扣款/入账的SP账户
4. 付款余额不足时：系统自动从多个SP账户扣款（如IPL $5000 + BB $3000 = 总余额 $8000）
5. SP间互转：打通的SP之间（如IPL⇄BB）可互转，在Exchange中操作，对商户透明
6. 交易记录：所有SP的记录混合展示，不区分SP来源

Assets / Fiat Accounts

Fiat Accounts（法币账户）

管理法币账户余额，同名充值与提现 同名操作

职责定义

本菜单只管"自己的法币"：查看余额、同名充值（自己银行→平台）、同名提现（平台→自己银行）。
不管：收别人的钱（→ Transfer In / Checkout）、付给别人（→ Transfer Out）、兑换（→ Exchange）

页面功能

账户余额总览
按币种展示总余额（多个SP账户合计），如 USD $8,000 / EUR €3,000。商户不感知SP，底层可能分属多个SP账户
同名充值（Deposit）
从自己的银行账户转入法币到平台。商户只选币种和金额，系统自动选择入账的SP账户（有默认配置）
同名提现（Withdraw）
从平台提现到自己的银行账户。商户只选币种和金额，系统自动选择扣款的SP账户
交易记录
所有法币变动记录（充值/提现/收款入账/付款扣款等），所有SP混合展示，不区分SP来源

边界说明

账户结构（商户不可见）：
• 底层每个SP独立开户：IPL-USD $5,000 + BB-USD $3,000 = 商户看到 USD $8,000
• 付款时系统根据优先级自动从多个SP扣款，商户只看到总余额
• SP明细仅在租户端（TP）展示
当前阶段（仅 IPL + BB）：
• 法币余额 = 1个合并余额（IPL + BB 合计），商户看到按币种的总余额
• 后续增加其他SP时再评估是否需要调整展示逻辑
Assets / Crypto Wallet

Crypto Wallet（加密钱包）

管理加密资产，同名充币与提币 同名操作

职责定义

本菜单只管"自己的加密资产"：查看余额、同名充币（自己外部钱包→平台托管地址）、同名提币（平台→自己外部钱包）。
不管：收别人的加密转账（→ Transfer In / ）、转给别人（→ Transfer Out ）、法币⇄加密兑换（→ Exchange）

页面功能

钱包余额总览
按币种+链展示总余额（多个SP托管地址合计），如 USDT-TRC20 10,000 / BTC 0.5。商户不感知SP
同名充币（Deposit）
从自己的外部钱包转入加密资产到平台托管地址。系统自动选择SP的托管地址，显示充值地址和二维码
同名提币（Withdraw）
从平台托管地址提币到自己的外部钱包。系统自动选择扣款的SP托管地址
交易记录
所有加密变动记录（充币/提币/收款入账/转出扣款等），所有SP混合展示，含链上TxHash

边界说明

当前阶段（仅 IPL + BB）：
• 数币余额 = 1个余额（仅BB提供加密托管能力），商户看到按币种+链的余额
• 后续增加其他有加密托管能力的SP时再评估合并展示逻辑
Assets / Exchange

Exchange（兑换/承兑）

自己账户间的资产形态转换，钱不出服务商体系 同名操作

职责定义

本菜单只管"纯钱包内兑换"：换汇、On-Ramp、Off-Ramp、SP间互转 — 钱始终在自己账户体系内，不出去。
不管：承兑后要付出去的场景（→ Remittance，一单到底自动编排）

法币换汇（FX）

即时换汇
自己法币账户间换汇，如 USD → EUR。系统自动选择SP，商户不感知
换汇记录
历史换汇记录、汇率、手续费详情

法转数（On-Ramp）

法币 → 加密
从自己法币账户余额兑换为加密资产，存入自己的加密钱包。钱没出去，只是换了形态
On-Ramp 记录
法转数历史记录、兑换汇率、到账数量

数转法（Off-Ramp）

加密 → 法币
从自己加密钱包余额兑换为法币，存入自己的法币账户。钱没出去，只是换了形态
Off-Ramp 记录
数转法历史记录、兑换汇率、到账金额

边界说明

Exchange vs Remittance：
• Exchange = 纯兑换，钱留在自己钱包里（适合炒汇率、资产配置）
• Remittance = 承兑+付款一单到底，钱最终要出去（如BB Off-Ramp后直接到IPL法币账户再付出）
• SP间互转（如IPL⇄BB）也在Exchange中完成，对商户透明
• Remittance 的子步骤中如果涉及兑换，Exchange 也会产生对应记录，通过关联订单号串联
Transfer In / Collection Tools

Collection Tools（收款工具）

管理所有收款工具：Global Account（VA）和加密收款地址 他人→我（转账）

职责定义

本菜单统一管理"商户主动申请的收款工具"：包括法币收款的 Global Account（VA）和加密收款的链上地址。两者都是商户长期持有的固定收款入口，供客户/买家反复转入。
不管：Checkout中系统自动分配给订单的临时VA/地址（对商户透明，不在此管理）

收款地址（Onramp Address）加密收款

收款地址管理

地址详情

边界说明

收款工具 vs Payins：
• Collection Tools = 管理收款工具本身（VA账户、加密地址的申请/查看/启停）
• Payins = 查看所有入金交易记录（通过VA收到的法币、通过地址收到的加密、同名充值、充币等）
• 收款工具是"入口"，Payins是"流水"
Transfer In / Remitter

Remitter（汇款人维护）

管理向您汇款的汇款人信息，支持银行账户和链上地址 他人→我

职责定义

本菜单管"谁给我打钱"：维护汇款人（Remitter/Sender）信息，记录其汇款方式（银行账户或链上地址）。
用于入金时自动匹配汇款来源、加速入账确认、满足合规要求（KYC/AML需要知道资金来源方）。
与 Beneficiary 的对称关系：Beneficiary = 我付给谁（Transfer Out），Remitter = 谁付给我（Transfer In）

页面功能

汇款人列表
查看所有已登记的汇款人，支持按类型（个人/企业）、汇款方式（银行/链上）筛选
新增汇款人
登记汇款人基本信息：名称、类型（个人/企业）、国家/地区、证件信息
汇款方式 — 银行账户
为汇款人添加银行账户信息：银行名称、账户名、账号、SWIFT/IBAN、银行地址等
汇款方式 — 链上地址
为汇款人添加链上地址：钱包地址、链、币种。用于匹配链上入金来源
编辑/删除汇款人
修改汇款人信息、管理其汇款方式、删除/恢复
汇款人详情
查看汇款人完整信息、所有汇款方式、关联的入金记录历史

边界说明

汇款人 vs 收款人（对称设计）：
• Remitter（汇款人）在 Transfer In — 记录"谁给我打钱"，一个汇款人可有多种汇款方式（多个银行账户 + 多个链上地址）
• Beneficiary（收款人）在 Transfer Out — 记录"我付给谁"，一个收款人可有多种收款方式
• 入金时系统可自动匹配已登记的汇款人，加速入账确认
• 合规要求：KYC/AML需要明确资金来源方身份
Transfer In / Payins

Payins（所有入金交易）

所有资金流入的交易记录，统一查看 他人→我 + 同名充值/充币

职责定义

本菜单管"所有入金交易记录"：统一展示所有资金流入记录，包括：
1. 他人通过VA银行转账入金
2. 他人通过链上地址转入加密资产
3. 同名充值（自己银行→平台法币账户）
4. 同名充币（自己外部钱包→平台托管地址）
不管：Checkout收单产生的收款（→ Checkout / Online Payment 等）

页面功能

入金交易列表
所有入金记录混合展示，支持按类型（VA收款/链上收款/同名充值/同名充币）、币种/时间/状态/汇款人筛选。不展示SP
交易详情
入金金额、来源类型、汇款人/来源信息、入账时间、关联收款工具（VA/地址）、链上TxHash（如适用）
退款
对已入账的收款发起退款申请
交易类型标签：
• VA收款 他人通过银行转账到VA
• 链上收款 他人通过链上转账到收款地址
• 同名充值 自己银行账户充值到平台
• 同名充币 自己外部钱包充币到平台
筛选条件：交易类型、币种、金额范围、时间范围、订单状态、汇款人名称
Transfer In / Onramp

Onramp（收款换U）

通过VA收款后自动兑换为USDT，一单到底 入金+换U

职责定义

本菜单管"法币收款→自动换U"的组合流程：他人通过VA银行转账法币入金，系统自动将收到的法币兑换为USDT（或其他稳定币），最终入账到商户的加密钱包。整个流程一单到底，商户无需手动操作Exchange。
底层编排：VA收款（Payins）→ On-Ramp/法转数（Exchange）→ 入账USDT到Crypto Wallet
与 Payins 的区别：Payins = 纯入金记录（法币到法币账户），Onramp = 入金+自动换U（法币→USDT到加密钱包）
与 Remittance 的对称关系：Remittance = 收→换→付（钱出去），Onramp = 收→换→入账（钱进来换形态）

页面功能

收款换U订单列表
所有Onramp订单，展示：法币收款金额、兑换汇率、USDT到账数量、订单状态。支持按时间/币种/状态筛选
订单详情 — 全链路视图
展示完整资金链路：VA收款（法币入账）→ 兑换（法币→USDT）→ 入账（USDT到加密钱包），每步状态实时更新
子步骤追踪
每个子步骤的独立状态、时间、金额、汇率、手续费明细
配置规则
设置自动换U规则：指定VA收款后自动触发兑换、目标币种（USDT/USDC等）、汇率策略（实时/锁定）

典型场景

场景：跨境电商收款换U
海外买家通过银行转账USD到商户VA → 系统自动将USD兑换为USDT → USDT入账到商户加密钱包。商户全程无需手动操作
与其他菜单的关系：
• Onramp 是"编排层"，底层复用 Payins（VA收款）和 Exchange（On-Ramp/法转数）的能力
• 商户在 Onramp 看到完整链路，在 Payins 和 Exchange 也能看到对应的子订单记录
• 通过关联订单号串联，各菜单都能追溯

Transfer Out / Payouts

Payouts（所有出金交易）

所有资金流出的交易记录，统一查看 出金

职责定义

本菜单管"所有出金交易记录"：统一展示所有资金流出记录，包括：
1. 法币付款（银行转账付给他人收款人）
2. 链上付币（加密资产转给他人链上地址）
3. 同名提现（平台法币账户→自己银行账户）
4. 同名提币（平台托管地址→自己外部钱包）
不管：需要先承兑/换汇再付款的组合流程（→ Remittance = 收→换→付一单到底）；Checkout退款产生的出金（→ Checkout）
与 Payins 的对称关系：
• Payins = 所有入金（VA收款 + 链上收款 + 同名充值 + 同名充币）
• Payouts = 所有出金（法币付款 + 链上付币 + 同名提现 + 同名提币）
• 两者都是"纯交易记录"，不含编排层（Onramp / Remittance 是编排层）

创建付款

出金交易列表
所有出金记录混合展示，支持按类型（法币付款/链上付币/同名提现/同名提币）、币种/时间/状态/收款人筛选。不展示SP
交易详情
出金金额、目标类型、收款人/目标信息、到账时间、链上TxHash（如适用）
交易类型标签：
• 法币付款 银行转账付给他人收款人
• 链上付币 加密资产转给他人链上地址
• 同名提现 平台法币账户提现到自己银行
• 同名提币 平台托管地址提币到自己外部钱包

Trade Documents（贸易单据）

贸易单据管理与合规文档

职责定义

本菜单管"贸易背景材料"：管理与收付款相关的贸易单据，用于证明交易的真实贸易背景，满足合规审核要求。
典型单据：商业发票（Invoice）、采购合同（Contract）、提单（Bill of Lading）、装箱单（Packing List）、报关单等

页面功能

单据列表
查看所有贸易单据，按类型（发票/合同/提单/报关单）、状态（待审核/已通过/已拒绝）筛选
上传单据
上传发票、合同、提单等贸易文档，支持批量上传，自动OCR识别关键信息
单据关联
将单据关联到对应的收付款订单、汇款订单，作为贸易背景证明
审核状态
查看单据的合规审核状态，支持补充材料（RFI）

Trade Documents / Shop Management

Shop Management（店铺管理）

管理电商店铺，同步订单数据作为贸易背景

职责定义

本菜单管"电商店铺绑定与订单同步"：商户绑定其电商平台店铺（如Amazon、Shopify、eBay等），系统自动同步店铺订单数据，作为收付款的贸易背景证明。
用途：合规审核时自动匹配订单与收付款记录，减少人工上传单据的工作量

页面功能

店铺列表
查看所有已绑定的电商店铺，含平台类型、店铺名称、绑定状态、最近同步时间
绑定新店铺
选择电商平台 → 授权绑定 → 配置同步规则（同步频率、订单范围等）
订单同步记录
查看从店铺同步的订单数据，含订单号、金额、买家信息、物流状态
店铺设置
管理店铺授权、同步配置、解绑店铺
