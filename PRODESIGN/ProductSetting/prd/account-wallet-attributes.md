## 核心设计：账户币种 Attributes

> **SP 上架产品时，需要标记自己支持什么账户**。账户能力以「账户币种」为单位配置，每个币种是一个独立的 attributes 配置单元。

### 账户类型（枚举）

SP 上架时需声明支持的账户类型（多选）：

| 账户类型     | 编码            | 说明                                       |
| ------------ | --------------- | ------------------------------------------ |
| 专用户       | `SPECIAL`     | 专项用途账户，如特定业务场景专用           |
| 待结算账户   | `SETTLEMENT`  | 交易资金待结算状态持有                     |
| 保证金账户   | `MARGIN`      | 用于保证金/押金等担保用途                 |
| 冻结户       | `FROZEN`      | 资金冻结状态账户，如风控/合规冻结         |

> SP 可按币种配置支持的账户类型。例如：USD 支持全部类型，HKD 只支持专用户和待结算账户。

## 1. 法币账户（FIAT_ACCOUNT）

### 1.1 基本信息

| 字段           | 类型 | 说明             | 示例值                  |
| -------------- | ---- | ---------------- | ----------------------- |
| 产品代码       | 固定 | 产品唯一标识     | `FIAT_ACCOUNT`        |
| 产品名称       | 文本 | 展示名称         | 法币账户                |
| SP 标识        | 文本 | 提供方           | BB / IPL                |
| 产品状态       | 下拉 | 上架状态         | 启用 / 停用             |

### 1.2 账户币种 Attributes（List 结构）

> SP 按币种配置账户能力，每个币种是一个配置单元。

| 字段             | 类型   | 必填 | 说明                                       |
| ---------------- | ------ | ---- | ------------------------------------------ |
| 账户币种         | 单选   | ✅   | USD, HKD, EUR, GBP, SGD 等                 |
| 支持的账户类型   | 多选   | ✅   | 专用户 / 待结算账户 / 保证金账户 / 冻结户   |
| 账户开户规则     | List   | ✅   | 见 1.2.1 账户开户规则配置                         |
| 充值币种配置     | List   | ✅   | 见 1.3 充值币种配置                         |
| 提现币种配置     | List   | ✅   | 见 1.5 提现币种配置                         |
| 充值限额         | 对象   | ✅   | 见 1.7 限额与数量限制                       |
| 提现限额         | 对象   | ✅   | 见 1.7 限额与数量限制                       |

#### 1.2.1 账户开户规则配置

> 根据产品/属性/属性值条件定义账户开户规则。

| 字段             | 类型   | 必填 | 说明                                       |
| ---------------- | ------ | ---- | ------------------------------------------ |
| 是否允许客户自主开户 | 开关 | 是   | 是否允许客户自主开户，默认否                 |
| 开户规则列表     | 列表   | 是   | 规则定义在何种条件下开启何种账户类型   |

**开户规则项：**

| 字段             | 类型   | 必填 | 说明                                       | 示例                                       |
| ---------------- | ------ | ---- | ------------------------------------------ | ------------------------------------------ |
| 产品             | 单选   | 是   | 触发账户开户的产品                         | 收款 / 付款 / OnRamp / OffRamp 等                 |
| 属性             | 单选   | 否   | 条件匹配的属性名称                         | 场景 / 渠道 / 地区等                         |
| 属性值           | 单选   | 否   | 条件匹配的属性值                           | 外贸收款 / 网络支付等                         |
| 开启账户类型     | 单选   | 是   | 规则匹配时开启的账户类型                   | 专用户 / 待结算账户等                       |
**Matching Logic:**

1. When merchant uses a product (e.g., creates Collection order), system matches rules in order
2. Match priority: Product + Attribute + Attribute Value > Product + Attribute > Product only
3. If no rule matches, open default account type (usually General Account)
4. If `Allow Customer Self-Opening = Yes`, merchant can manually select account type when opening

### 1.3 充值币种配置（List 结构）

> SP 配置每个账户币种支持哪些充值方式、入账规则、合规要求。
> **被 OnRamp 继承**：OnRamp 预约单的法币入金方式 = 法币账户的充值能力。

#### 充值币种配置项

每个充值币种配置包含以下字段：

| 字段           | 类型     | 必填 | 说明                                                         |
| -------------- | -------- | ---- | ------------------------------------------------------------ |
| 充值币种       | 单选     | ✅   | 充值来源币种，如 USD, HKD, EUR, CNH                          |
| 入账币种       | 多选     | ✅   | 可入账的目标币种列表。可以是原币种入账，也可以是换币种入账（多个，默认 USD），也可以由客户指定换什么币种入账 |
| 充值合规要求   | 多选     | ✅   | 同名 / 法人个人 / 关联公司（见下方合规要求枚举）              |
| 充值限额       | 对象     | ✅   | VA 账户限额 + 充值交易限额（见 1.7 限额与数量限制）            |

#### 充值合规要求（枚举）

| 合规要求   | 编码              | 说明                                       |
| ---------- | ----------------- | ------------------------------------------ |
| 同名       | `SAME_NAME`     | 汇款人与商户主体名称一致                   |
| 法人个人   | `LEGAL_REP`     | 汇款人为商户企业的法人代表（个人账户）     |
| 关联公司   | `AFFILIATE`     | 汇款人与商户为关联公司关系                 |

#### 入账币种配置规则

入账币种可放给客户配置，但必须在 SP 支持的范围内：

```
SP 层（上架时固定）：
  充值币种 = HKD
  入账币种 = [USD, HKD]  ← SP 支持这两种入账
  合规要求 = [同名, 法人个人]

TP 层（可收窄）：
  充值币种 = HKD
  入账币种 = [USD]       ← TP 只允许入 USD
  合规要求 = [同名]      ← TP 只允许同名
  是否允许商户自选入账币种 = 是

商户层（TP 允许时）：
  充值币种 = HKD
  入账币种 = USD         ← 商户自选入 USD
```

#### 配置示例

**示例 1：USD 账户充值配置**

```
账户币种 = USD
充值币种配置 = [
  {
    充值币种: USD,
    入账币种: [USD],           ← 原币种入账
    合规要求: [同名, 法人个人, 关联公司],
    充值限额: {单笔最大: 1,000,000 USD, 单月最大: 无限额}
  },
  {
    充值币种: HKD,
    入账币种: [USD, HKD],      ← 可入 USD 或 HKD
    合规要求: [同名, 法人个人],
    充值限额: {单笔最大: 500,000 USD, 单月最大: 5,000,000 USD}
  },
  {
    充值币种: CNH,
    入账币种: [USD],           ← 只能换汇入 USD
    合规要求: [同名],
    充值限额: {单笔最大: 100,000 USD, 单月最大: 1,000,000 USD}
  }
]
```

#### 充值方式配置

| 字段             | 类型 | 必填 | 说明                           |
| ---------------- | ---- | ---- | ------------------------------ |
| 是否开启同名充值 | 开关 | ✅   | 默认开启；关闭后仅支持持有余额 |

开启后展开：

| 字段     | 类型 | 必填 | 说明                              |
| -------- | ---- | ---- | --------------------------------- |
| 充值方式 | 多选 | ✅   | 银行转账 / 其他 EX 服务商账户转账 |

##### 方式一：银行转账（如支持）

> 商户通过外部银行账户向 SP 提供的收款账户转账入金。包括 VA（虚拟账户）和收付款账户两种形态。

**VA（虚拟账户）配置：**

| 字段                 | 类型 | 必填 | 说明                              |
| -------------------- | ---- | ---- | --------------------------------- |
| VA 支持的币种        | 多选 | ✅   | USD, HKD, EUR, CNH 等             |
| VA 支持的地区        | 多选 | ✅   | HK, SG, UK 等                     |
| VA 支持的银行        | 多选 | ✅   | HSBC, DBS, Barclays（按地区联动） |
| VA 同名校验          | 开关 | ✅   | 是否要求汇款人与商户同名          |
| VA Purpose Code 校验 | 开关 | ✅   | 是否要求转账备注含 Purpose Code   |

**SP 客资账户（Omnibus）配置：**

| 字段                 | 类型 | 必填 | 说明                                             |
| -------------------- | ---- | ---- | ------------------------------------------------ |
| 收付款账户模式       | 固定 | ✅   | Omnibus（SP 代收付大账户，多商户共用），本期固定 |
| 收付款账户支持的币种 | 多选 | ✅   | USD 等                                           |
| 入账识别方式         | 固定 | ✅   | 转账备注（Reference Code），本期固定             |

**商户准入限制：**

| 字段         | 类型 | 必填 | 说明                                              |
| ------------ | ---- | ---- | ------------------------------------------------- |
| 主体所在国家 | 多选 | ✅   | 允许使用银行转账充值的商户注册国家，如 HK, SG, US |
| 主体类型     | 多选 | ✅   | 企业 / 个人 等                                    |
| 商户行业     | 多选 | ✅   | 允许的商户行业（MCC 或行业分类）                  |

**报备要求：**

| 字段         | 类型 | 必填 | 说明                               |
| ------------ | ---- | ---- | ---------------------------------- |
| 是否需要报备 | 开关 | ✅   | 充值前是否需要向 SP 报备，默认需要 |

#### 方式二：其他 EX 服务商账户转账（如支持）

> 商户在 EX 平台内的**其他 SP 账户余额**，通过内部转账方式转入当前 SP 账户。
> 例如：IPL 法币账户 → BB 法币账户，BB 法币账户 → IPL 法币账户。
> 本质是 EX 平台内跨 SP 的同名资金划转。

| 字段                   | 类型 | 必填     | 说明                                           |
| ---------------------- | ---- | -------- | ---------------------------------------------- |
| 是否支持跨 SP 转账充值 | 开关 | ✅       | 是否允许从其他 EX 服务商账户转入               |
| 可接收转账的来源 SP    | 多选 | 条件必填 | 配置允许从哪些 SP 转入（如 BB 配置可接收 IPL） |
| 跨 SP 转账支持的币种   | 多选 | 条件必填 | 仅限双方 SP 都支持的币种（交集）               |
| 跨 SP 转账是否需要换汇 | 开关 | ✅       | 源币种≠目标币种时是否支持自动换汇             |

**商户准入限制：**

> SP 配置哪些商户可以使用跨 SP 转账充值。

| 字段         | 类型 | 必填 | 说明                             |
| ------------ | ---- | ---- | -------------------------------- |
| 主体所在国家 | 多选 | ✅   | 允许使用跨 SP 转账的商户注册国家 |
| 主体类型     | 多选 | ✅   | 企业 / 个人 等                   |
| 商户行业     | 多选 | ✅   | 允许的商户行业（MCC 或行业分类） |

**充值限额：**

| 字段           | 单位 | 可选值       | 说明                                     | 示例            |
| -------------- | ---- | ------------ | ---------------------------------------- | --------------- |
| 单笔限额       | USD  | 具体金额范围 | 单笔转账金额范围                         | 0 - 200,000 USD |
| 单月限额       | USD  | 具体金额范围 | 每月转账累计上限                         | 0 - 300,000 USD |
| 超限额处理方式 | 固定 | 人工处理     | 超过限额后的处理方式，本期固定为人工处理 | 人工处理        |

**报备要求：**

| 字段         | 类型 | 必填 | 说明                               |
| ------------ | ---- | ---- | ---------------------------------- |
| 是否需要报备 | 开关 | ✅   | 转账前是否需要向 SP 报备，默认需要 |

**跨 SP 转账规则：**

```
前提：商户同时在 SP-A 和 SP-B 开有法币账户
操作：从 SP-A 的 USD 账户 → 转入 SP-B 的 USD 账户

校验：
  1. SP-B 是否开启「跨 SP 转账充值」？
  2. SP-B 的「可接收来源 SP」是否包含 SP-A？
  3. 转账币种是否在「跨 SP 转账支持的币种」内？
  4. 商户在两个 SP 下是否都已完成 KYC（同名验证）？
  → 全部通过 → 执行内部划转
```

**示例：**

- BB 配置：可接收来源 = [IPL]，支持币种 = [USD, EUR]
- IPL 配置：可接收来源 = [BB]，支持币种 = [USD]
- 商户从 IPL USD → BB USD ✅
- 商户从 BB EUR → IPL EUR ❌（IPL 跨 SP 只配了 USD）

### 1.3 入账匹配规则

> **核心问题**：收款币种与账户持有币种不一致时如何处理？
> **入账模式**：固定为**混合模式**（原币种入账 + 换汇入账）。至少 USD 为原币种入账，不支持的币种自动换汇到指定目标币种。
> **三层配置**：SP 定义能力范围 → TP 配置默认规则（并决定商户是否可自选） → 商户自选（仅 TP 允许时）。

#### 1.3.1 入账模式说明

入账模式**固定为混合模式**，不提供模式选择。每一层配置的核心就是两件事：

| 配置项                         | 含义                                       | 示例     |
| ------------------------------ | ------------------------------------------ | -------- |
| **原币种入账的币种列表** | 这些币种收到后直接入对应币种账户           | USD, EUR |
| **换汇入账的目标币种**   | 不在原币种列表内的币种，自动换汇后入此账户 | USD      |

> USD 必须在原币种入账列表中（系统强制），确保美元收款始终原币入账。

#### 1.3.2 三层配置模型

```
第1层：SP 能力定义（上架时固定）
  → 定义哪些币种可原币入账、哪些币种可作为换汇目标
  → 不可被 TP/商户修改

第2层：TP 产品配置（TP 签约后配置）
  → 在 SP 能力范围内收窄：允许哪些原币种入账、默认换汇目标币种
  → 关键：配置「是否允许商户自选」开关
     ├── 开 → 商户可在 MP Portal 自行配置入账规则
     └── 关 → 商户无法配置，统一使用 TP 的配置（默认）

第3层：商户自选（仅 TP 允许时）
  → 商户在 TP 允许的范围内，选择原币入账币种、换汇目标币种
  → 商户可选范围 ⊆ TP 允许范围 ⊆ SP 能力范围
```

#### 1.3.3 SP 能力定义（第1层）

| 字段                   | 类型 | 必填 | 说明                                           |
| ---------------------- | ---- | ---- | ---------------------------------------------- |
| 原币种入账支持的币种   | 多选 | ✅   | SP 能力范围内可原币入账的币种（USD 必选）      |
| 换汇入账支持的目标币种 | 多选 | ✅   | SP 能力范围内可作为换汇目标的币种，如 USD, HKD |
| 换汇汇率来源           | 下拉 | ✅   | SP 实时汇率 / 固定汇率（本期仅 SP 实时汇率）   |

#### 1.3.4 TP 配置（第2层）

| 字段                       | 类型           | 说明                                                          | 约束                 | 默认值       |
| -------------------------- | -------------- | ------------------------------------------------------------- | -------------------- | ------------ |
| 允许的原币入账币种         | 多选           | TP 允许商户使用的原币入账币种                                 | ⊆ SP 支持，USD 必含 | = SP 全部    |
| 默认换汇目标币种           | 单选           | **商户未配置 / 不允许商户自选时**，使用此换汇目标       | ∈ SP 支持           | USD          |
| 允许的换汇目标币种         | 多选           | 商户可选的换汇目标范围（仅允许商户自选时生效）                | ⊆ SP 支持           | = SP 全部    |
| **是否允许商户自选** | **开关** | **核心开关**：决定商户能否在 MP Portal 自行配置入账规则 | —                   | **否** |

> **关闭时**（默认）：所有商户统一使用 TP 配置的「允许的原币入账币种」+「默认换汇目标币种」，商户无感知。
> **开启时**：商户可在 MP Portal 的 Settings 中自行配置，TP 配置作为默认值。

#### 1.3.5 商户自选配置（第3层）

> **前提**：TP 开启了「是否允许商户自选」。未开启时商户看不到此配置项。

| 字段         | 类型 | 说明                       | 约束                 |
| ------------ | ---- | -------------------------- | -------------------- |
| 原币入账币种 | 多选 | 商户选择哪些币种原币入账   | ⊆ TP 允许，USD 必含 |
| 换汇目标币种 | 单选 | 商户选择换汇入账的目标币种 | ∈ TP 允许的换汇目标 |

**生效优先级**：

```
TP 允许商户自选？
  ├── 否 → 直接使用 TP 配置
  └── 是 → 商户已配置？
        ├── 是 → 使用商户配置
        └── 否 → 使用 TP 配置（作为默认值）
```

#### 1.3.6 入账匹配完整逻辑

```
收到一笔入账（币种 = X）：
  0. 读取生效配置：
     - TP 未开放商户自选 → 用 TP 配置
     - TP 开放 + 商户已配置 → 用商户配置
     - TP 开放 + 商户未配置 → 用 TP 配置（默认值）
  1. X ∈ 原币种入账列表？（USD 一定在列表中）
     → 是：入 X 币种账户（原币入账）
     → 否：进入第2步
  2. X → 换汇 → 入「换汇目标币种」账户
  3. 换汇失败 → 异常处理：
     → 挂起，人工处理（本期）
     → 自动退回（下期）
```

#### 1.3.7 完整示例

```
SP 能力：
  原币种 = [USD, EUR, GBP, HKD]
  换汇目标 = [USD, HKD]

═══ 场景1：TP 不允许商户自选 ═══

TP 配置：
  允许原币种 = [USD, EUR]
  默认换汇目标 = USD
  允许商户自选 = 否              ← 商户无法配置

所有商户统一走 TP 配置：
  → 收 USD → 入 USD 账户（原币入账）
  → 收 EUR → 入 EUR 账户（原币入账）
  → 收 GBP → 换汇 → 入 USD 账户
  → 收 JPY → 换汇 → 入 USD 账户

═══ 场景2：TP 允许商户自选 ═══

TP 配置：
  允许原币种 = [USD, EUR]
  默认换汇目标 = USD
  允许换汇目标 = [USD, HKD]
  允许商户自选 = 是              ← 商户可以配置

商户 A（自行配置了）：
  原币入账 = [USD, EUR]
  换汇目标 = HKD
  → 收 USD → 入 USD 账户
  → 收 EUR → 入 EUR 账户
  → 收 GBP → 换汇 → 入 HKD 账户    ← 用商户自选

商户 B（未配置，走 TP 默认）：
  → 收 USD → 入 USD 账户
  → 收 EUR → 入 EUR 账户
  → 收 GBP → 换汇 → 入 USD 账户    ← 用 TP 默认
```

## 3. 收款人 vs 提现的属性边界

### 3.1 核心原则

```
收款人（Beneficiary/Payee）= "谁收钱" + "收到哪里"
  → 身份信息、银行/钱包/地址信息、审核状态
  → 一次添加，多次复用

提现（Withdrawal）= "这次转多少" + "走哪个渠道"
  → 金额、出款渠道、换汇、本次交易状态
  → 每次提现独立
```

### 3.2 对比表

| 维度       | 归属「收款人」          | 归属「提现」                |
| ---------- | ----------------------- | --------------------------- |
| 谁收钱     | ✅ 收款人姓名/公司名    | —                          |
| 与商户关系 | ✅ Self/法人/董事       | —                          |
| 账户信息   | ✅ 银行账号/钱包账号    | —                          |
| 收款人状态 | ✅ 待审核/已通过/已拒绝 | —                          |
| 出款渠道   | —                      | ✅ TT/Local/SEPA/Wallet     |
| 出款金额   | —                      | ✅ 本次提现金额             |
| 出款币种   | —                      | ✅ 本次提现币种             |
| 是否换汇   | —                      | ✅ 提现币种≠账户币种时触发 |
| 交易状态   | —                      | ✅ 处理中/成功/失败         |

### 3.3 本期代理模式的特殊约束

```
本期（代理模式）：
  ├── 收款人身份关系 = Self / Legal Representative / Director
  ├── 收款人类型 = 银行账户 / 本地钱包（Web3 地址归数币钱包提币）
  ├── 不支持第三方收款人（POBO 归 Payouts）
  └── 提现 = 选择已有收款人 + 选择渠道 + 输入金额

下期（Payouts 产品开放后）：
  ├── 收款人身份关系 += Third Party
  ├── 支持 POBO（付款人≠收款人）
  └── Payouts = 非同名付款，独立于提现
```

## 4. 本期 vs 下期

### 4.1 本期实现（Phase 1）

```
✅ 法币账户：开户、持有、同名充值（VA/代收付）、同名提现
✅ 数币钱包：开户、持有、同名充币、同名提币
✅ 入账匹配：三层配置（SP能力 → TP默认 → 商户自选）
✅ 提现渠道：TT / Local Bank Transfer / SEPA / Local Wallet（链上出款归数币钱包提币）
✅ 收款人身份：Self / Legal Representative / Director
✅ 收款人类型：银行账户 / 本地钱包（Web3 地址归数币钱包）
✅ 收款人审核：预审核（同名可免审）
✅ TP 配置：币种/渠道/地区/限额/入账模式/是否允许商户自选
✅ 商户自选：TP 开放后商户可在 MP Portal 自选入账账户/币种
```

### 4.2 下期规划（Phase 2）

```
☐ 收款人身份：增加 Third Party（异名收款人，配合 Payouts）
☐ 入账异常自动退回（本期人工处理）
☐ 入账匹配：支持固定汇率模式
☐ 商户自助扩展：提现渠道偏好、收款人默认值等更多自选项
☐ Local Wallet 扩展更多钱包类型
☐ VA 排序自定义
☐ 账户能力输出（跨 SP 共享）
```
