## 产品定位

**Crypto Checkout** 是 EX 平台的收单编排层产品：

- 商户以 **法币（USD）计价** 发起收款请求，付款人用**稳定币（USDT/USDC）**完成链上支付
- EX 本期以**四方聚合**模式外接一条链上收单通道，不自建链上能力
- 每笔订单生成**独立收款地址（One-time Address）**，付款人转账后通道确认到账，结算入商户数币钱包
- EX 是 B2B PPaaS，不直接面向 C 端；TP 通过 EX API 集成后，面向其自己的商户提供服务

**与其他产品的关系：**

| 产品                      | 收款方式                 | 计价                           | 场景                  |
| ------------------------- | ------------------------ | ------------------------------ | --------------------- |
| Crypto Wallet（充币）     | 链上转账，同名充值       | 数币计价                       | 商户自身资金管理      |
| Payins                    | VA / 链上 / 同名，纯记录 | —                             | 纯记录层，无收单编排  |
| **Crypto Checkout** | 链上支付，付款人主动发起 | **USD 计价，稳定币支付** | 商户向其用户/买家收款 |

**配置层级：**

```
第1层：产品层（SA 上架 Crypto Checkout 产品时定义，TP 不可修改）
  → 1.1 支付能力（Payment Capability）：币种、网络、地址模型
  → 1.2 订单配置（Order Settings）：订单有效期范围
  → 1.3 支付阈值配置（Payment Tolerance）：underpayment / overpayment 系统范围与默认值
  → 1.4 结算配置（Settlement Settings）：结算模式、币种规则、目标账户
  → 1.5 风控配置（Risk Control）：商户主体类型、所在国家/地区准入限制

第2层：TP 产品配置（TP 签约后配置，可对商户做特殊配置）
  → 2.1 支付能力收窄（币种/网络）
  → 2.2 订单配置（默认有效期）
  → 2.3 结算配置（商户结算账户指定）

第3层：商户配置（商户在 MP Portal 自行配置）
  → 3.1/3.2 支付阈值（threshold + action），未配置时继承产品层默认值

Webhook 通知事件 → 统一在开发者中心（Developer）管理，不在产品属性中定义
```

## 1. 产品层属性（Phase 1）

> 由 SA 上架 Crypto Checkout 产品时定义，TP 不可修改，所有商户共享同一套产品能力。
> 按功能域分为五类：支付能力、订单配置、支付阈值配置、结算配置、风控配置（商户主体准入）。

### 1.1 支付能力（Payment Capability）

> 定义该产品支持的支付币种、网络、收款地址模型等核心支付能力。

| 字段             | 字段名（EN）              | 类型 | 本期值                                        | 说明                                             |
| ---------------- | ------------------------- | ---- | --------------------------------------------- | ------------------------------------------------ |
| 订单计价币种     | pricing_currency          | 固定 | USD                                           | 商户创建订单时以法币计价                         |
| 支持的支付稳定币 | accepted_cryptocurrencies | 多选 | USDT / USDC                                   | 付款人可用的稳定币，预留扩展                     |
| 支持的区块链网络 | accepted_networks         | 多选 | 由接入通道决定（如 TRC-20 / ERC-20 / BEP-20） | 本期外接一条聚合通道，网络范围由通道提供         |
| 汇率来源         | exchange_rate_source      | 固定 | 通道实时汇率（USD → 稳定币等值换算）         | 创建订单时换算应付稳定币金额，展示给付款人       |
| 地址分配模式     | address_mode              | 固定 | 每单独立地址（One-time Address）              | 每笔订单生成唯一收款地址，不复用（行业标准做法） |

> 支付稳定币与网络的组合（如 USDT TRC-20 / USDC ERC-20）由通道能力决定，TP/商户不可修改。

### 1.2 订单配置（Order Settings）

> 定义订单的有效期范围等订单级参数。

| 字段           | 字段名（EN）  | 类型         | 本期值  | 说明                            |
| -------------- | ------------- | ------------ | ------- | ------------------------------- |
| 最短订单有效期 | order_ttl_min | 数字（分钟） | 15 分钟 | TP/商户配置的有效期不可低于此值 |
| 最长订单有效期 | order_ttl_max | 数字（小时） | 24 小时 | TP/商户配置的有效期不可超过此值 |

> 参考行业实践（BitPay 默认 15 分钟，Coinbase Commerce 默认 60 分钟），本期产品层定义允许范围，TP 配置默认值。

### 1.3 支付阈值配置（Payment Tolerance）

> 定义 Underpayment / Overpayment 的系统允许范围和默认值。
> 商户可在 MP Portal 自行修改（见 Section 3），TP 不参与此配置。

> **术语说明（行业标准）：**
>
> - **Underpayment**：付款人实际到账金额 **低于** 订单应付金额（常见于链上手续费扣减、手动输错金额等）
> - **Overpayment**：付款人实际到账金额 **高于** 订单应付金额（常见于手动输错、汇率波动等）
> - **Threshold（阈值）**：定义多大的偏差仍可视为"全额支付"自动接受，超出阈值则触发对应处理策略
>
> 参考 BitPay、BTCPay Server、NOWPayments 等行业实践。

> **阈值模式（Threshold Mode）：**
>
> | 模式     | 英文         | 说明                                           | 适用场景                                                     |
> | -------- | ------------ | ---------------------------------------------- | ------------------------------------------------------------ |
> | 百分比   | Percentage   | 按订单金额的百分比计算容忍值                   | 通用，但小额订单容忍额可能过小                               |
> | 固定金额 | Fixed Amount | 固定的容忍金额（与订单大小无关）               | 小额订单保底容忍，大额订单封顶容忍                           |
> | 两者取小 | Min of Both  | `容忍额 = min(订单金额 × 百分比, 固定金额)` | **推荐**：百分比适配大部分订单，固定金额为大额订单封顶 |
>
> **示例对比（underpayment，百分比=1%，固定金额=5 USD）：**
>
> | 订单金额        | 百分比模式                | 固定金额模式 | 两者取小 |
> | --------------- | ------------------------- | ------------ | -------- |
> | $100     | $1   | $5          |**$1** |              |          |
> | $1,000   | $10  | $5          |**$5** |              |          |
> | $10,000  | $100 | $5          |**$5** |              |          |

#### 1.3.1 Underpayment 配置

| 字段                        | 字段名（EN）                | 类型   | 系统允许范围   | 系统默认值 | 说明                                                  |
| --------------------------- | --------------------------- | ------ | -------------- | ---------- | ----------------------------------------------------- |
| underpayment_threshold_mode | underpayment_threshold_mode | 下拉   | 见上方三种模式 | Percentage | 阈值计算模式                                          |
| underpayment_threshold_pct  | underpayment_threshold_pct  | 百分比 | 0% ～ 10%      | 1%         | 百分比阈值（Percentage / Min of Both 模式时必填）     |
| underpayment_threshold_amt  | underpayment_threshold_amt  | 金额   | 0 ～ 100 USD   | 5 USD      | 固定金额阈值（Fixed Amount / Min of Both 模式时必填） |
| underpayment_action         | underpayment_action         | 下拉   | 见下方枚举     | 等待补款   | 超出阈值时的处理方式                                  |

**underpayment_action 枚举值：**

| 值         | 英文           | 说明                                                                                                     |
| ---------- | -------------- | -------------------------------------------------------------------------------------------------------- |
| 等待补款   | Await Top-Up   | 订单有效期内保持"部分支付"状态，付款人可多次转账补齐；到期仍不足则进入过期处理。商户可在 MP 后台查看并手动关单 |
| ~~自动关单~~ | ~~Auto Close~~ | ~~下期：单次到账金额不足且超出阈值，订单立即关闭；已到账资金入商户账户~~                                 |

#### 1.3.2 Overpayment 配置

| 字段                       | 字段名（EN）               | 类型   | 系统允许范围   | 系统默认值 | 说明                                                  |
| -------------------------- | -------------------------- | ------ | -------------- | ---------- | ----------------------------------------------------- |
| overpayment_threshold_mode | overpayment_threshold_mode | 下拉   | 见上方三种模式 | Percentage | 阈值计算模式                                          |
| overpayment_threshold_pct  | overpayment_threshold_pct  | 百分比 | 0% ～ 50%      | 3%         | 百分比阈值（Percentage / Min of Both 模式时必填）     |
| overpayment_threshold_amt  | overpayment_threshold_amt  | 金额   | 0 ～ 500 USD   | 50 USD     | 固定金额阈值（Fixed Amount / Min of Both 模式时必填） |
| overpayment_action         | overpayment_action         | 下拉   | 见下方枚举     | 全额暂挂 | 超出阈值时的处理方式                                    |

**overpayment_action 枚举值：**

| 值               | 英文                           | 说明                                                                                     |
| ---------------- | ------------------------------ | ---------------------------------------------------------------------------------------- |
| 自动接受全额入账 | Auto Accept & Settle All       | 超额部分连同订单金额一起结算到商户数币钱包，商户自行处理退款（本期 EX 不支持自动退款）   |
| 全额暂挂         | Hold & Await Merchant Decision | 多付订单全额暂挂（不结算），商户在后台确认"接受入账"或"发起退回"后才结算               |

### 1.4 结算配置（Settlement Settings）

> 定义订单支付成功后的结算规则。

| 字段             | 字段名（EN）         | 类型        | 本期值       | 说明                                          |
| ---------------- | -------------------- | ----------- | ------------ | --------------------------------------------- |
| 结算模式         | settlement_mode      | 固定        | 逐笔结算     | 每笔订单确认到账后单独结算，不做批次合并      |
| 结算币种规则     | settlement_currency  | 固定        | 同货币同网络 | 收到 USDT TRC-20，结算 USDT TRC-20；不做换汇  |
| 结算目标账户类型 | settlement_account   | 固定        | 商户数币钱包 | 结算入商户在 BB 的数币钱包余额                |
| ~~自动换汇~~    | ~~auto_conversion~~ | ~~不支持~~ | ~~—~~      | ~~本期不支持收数币→换汇→入法币账户（P2）~~ |

> **Webhook 通知事件** 不在产品属性中定义，统一由开发者中心（Developer）管理。
> 相关事件：`checkout.paid` / `checkout.expired` / `checkout.settled` / `checkout.underpaid`，详见开发者文档。

### 1.5 风控配置（Risk Control）

> 定义该产品的商户主体准入限制，包括主体所在国家/地区和主体类型。

| 字段 | 字段名（EN） | 类型 | 本期值 | 说明 |
| --- | --- | --- | --- | --- |
| 支持的商户主体类型 | accepted_entity_types | 多选 | Individual（个人）/ Enterprise（企业） | 允许接入该产品的商户主体类型，至少选一种 |
| 国家控制模式 | country_control_mode | 单选 | 不限制 / whitelist（白名单）/ blacklist（黑名单） | 不限制 = 所有国家均可；白名单 = 只允许选中的国家；黑名单 = 禁止选中的国家 |
| 支持/禁止的商户国家 | accepted_merchant_countries | 多选 | 国家/地区列表 | 白名单模式下为允许的国家，黑名单模式下为禁止的国家 |
| 国家 + 主体类型组合 | entity_restrictions | 组合 | 如：美国-企业、香港-个人+企业 | 可按「国家 + 主体类型」精细化配置准入规则 |

> **商户主体限制说明：**
>
> - 支持按「国家 + 主体类型」组合配置，例如：仅允许美国企业、香港个人 + 企业
> - 国家控制支持三种模式：不限制（所有国家均可）、正向选择（白名单：只允许选中的国家）、反向选择（黑名单：禁止选中的国家）
> - 订单限额（单笔金额、单日限额等）本期暂不配置，后续按需扩展

## 3. 商户配置（Phase 1）

> 商户可在 MP Portal 自行配置。未配置时使用产品层系统默认值（见 1.3 支付阈值配置）。

### 3.1 Underpayment 策略

| 字段                        | 类型   | 说明                 | 允许范围                                | 默认值（继承产品层） |
| --------------------------- | ------ | -------------------- | --------------------------------------- | -------------------- |
| underpayment_threshold_mode | 下拉   | 阈值计算模式         | Percentage / Fixed Amount / Min of Both | Percentage           |
| underpayment_threshold_pct  | 百分比 | 百分比阈值           | 0% ～ 产品层上限（10%）                 | 1%                   |
| underpayment_threshold_amt  | 金额   | 固定金额阈值         | 0 ～ 产品层上限（100 USD）              | 5 USD                |
| underpayment_action         | 下拉   | 超出阈值时的处理方式 | 等待补款                                | 等待补款             |

**策略说明：**

- **阈值内（偏差 ≤ 容忍额）**：自动视为全额支付，订单正常完成
  - Percentage 模式：容忍额 = 订单金额 × 百分比（如 $100 × 1% = $1）
  - Fixed Amount 模式：容忍额 = 固定金额（如 $5）
  - Min of Both 模式：容忍额 = min(订单金额 × 百分比, 固定金额)，大额订单自动封顶
- **超出阈值后：**
  - **等待补款（Await Top-Up）**：订单有效期内保持"部分支付"状态，付款人可多次转账补齐；到期仍不足则进入过期处理。商户可在 MP 后台查看并手动关单

### 3.2 Overpayment 策略

| 字段                       | 类型   | 说明                 | 允许范围                                | 默认值（继承产品层） |
| -------------------------- | ------ | -------------------- | --------------------------------------- | -------------------- |
| overpayment_threshold_mode | 下拉   | 阈值计算模式         | Percentage / Fixed Amount / Min of Both | Percentage           |
| overpayment_threshold_pct  | 百分比 | 百分比阈值           | 0% ～ 产品层上限（50%）                 | 3%                   |
| overpayment_threshold_amt  | 金额   | 固定金额阈值         | 0 ～ 产品层上限（500 USD）              | 50 USD               |
| overpayment_action         | 下拉   | 超出阈值时的处理方式 | 自动接受全额入账 / 全额暂挂 | 全额暂挂 |

**策略说明：**

- **阈值内（偏差 ≤ 容忍额）**：自动视为全额支付，全部金额结算到商户数币钱包
  - Percentage 模式：容忍额 = 订单金额 × 百分比（如 $100 × 3% = $3）
  - Fixed Amount 模式：容忍额 = 固定金额（如 $50）
  - Min of Both 模式：容忍额 = min(订单金额 × 百分比, 固定金额)，大额订单自动封顶
- **超出阈值后，按 overpayment_action 处理：**
  - **自动接受全额入账（Auto Accept & Settle All）**：超额部分连同订单金额一起结算到商户数币钱包，商户自行处理退款（本期 EX 不支持自动退款）
  - **全额暂挂（Hold & Await Merchant Decision）**：多付订单全额暂挂（不结算），商户在 MP 后台确认"接受入账"或"发起退回"后才结算

> **商户配置约束：** 商户设定的阈值百分比和固定金额不可超过产品层定义的系统允许范围上限（underpayment pct ≤ 10% / amt ≤ 100 USD，overpayment pct ≤ 50% / amt ≤ 500 USD）。未配置时继承产品层系统默认值。

## 4. 订单状态流转

```
创建订单
  └→ [待支付 Pending]
        ├→ 收到链上转账（金额足额）→ 通道确认 → [支付成功 Paid]
        │     └→ 结算入商户数币钱包 → [已结算 Settled]
        │
        ├→ 收到链上转账（underpaid，策略=等待补款）→ [部分支付 Partially Paid]
        │     ├→ 有效期内补款凑足 → [支付成功 Paid] → [已结算 Settled]
        │     └→ 有效期到期仍不足 → [已过期 Expired]（已到账资金按 overpaid 规则处理）
        │
        ├→ 收到链上转账（underpaid，策略=自动关单）→ [已关闭 Closed]
        │     └→ 已到账资金按 overpaid 规则处理（自动接受则入商户账户）
        │
        ├→ 收到链上转账（overpaid，策略=自动接受）→ [支付成功 Paid]（超额全部入账）
        │     └→ [已结算 Settled]
        │
        ├→ 收到链上转账（overpaid，策略=转人工）→ [异常 Exception]
        │
        └→ 有效期到期，未收到任何转账 → [已过期 Expired]
```

**异常场景处理：**

| 场景             | 处理方式                                            |
| ---------------- | --------------------------------------------------- |
| 订单超时未支付   | 自动关单，地址失效                                  |
| 通道通知延迟     | 订单保持待支付状态，通道到账通知后更新              |
| Webhook 通知失败 | 自动重试 3 次，超出后标记"通知失败"，商户可主动查询 |
| 结算失败         | 标记异常，SA 人工介入，通知商户                     |

## 5. MP Portal 订单列表（Phase 1）

商户可在 MP Portal → Checkout → Crypto Checkout 查询收单订单：

| 字段       | 说明                                                           |

## 6. 分阶段实现计划

### Phase 1（本期）— 核心收单闭环

**目标：** 跑通最小闭环，外接一条聚合通道，支持 USD 计价稳定币收单

```
产品层（SA 上架时定义）：
  ✅ 支付能力：USD 计价，USDT/USDC 支付，TRC-20/ERC-20/BEP-20（由通道决定），每单独立地址
  ✅ 订单配置：有效期范围 15 分钟 ～ 24 小时
  ✅ 支付阈值：underpayment 0%～10%（默认 1%）、overpayment 0%～50%（默认 3%）
  ✅ 结算配置：逐笔结算，同货币同网络，结算入商户数币钱包，不换汇

TP 配置层：
  ✅ 支付能力收窄：允许的支付币种/网络（可限制）
  ✅ 订单配置：默认订单有效期（可设置）
  ✅ 结算配置：商户结算账户指定

商户配置层（商户自配）：
  ✅ underpayment_threshold + underpayment_action
  ✅ overpayment_threshold + overpayment_action

商户端：
  ✅ MP Portal 收单订单列表（查询）
  ✅ Webhook 通知（在开发者中心管理）：checkout.paid / checkout.expired / checkout.settled / checkout.underpaid
```

**Phase 1 不做：**

```
  ❌ 自动换汇结算（收数币→换汇→入法币账户）
  ❌ 商户自助退款（overpaid 退款由商户线下处理或 SA 人工介入）
  ❌ 商户自定义订单有效期（P2 开放）
  ❌ 订阅支付 / 预授权
  ❌ 多通道聚合路由
  ❌ 托管支付页（Hosted Page）
```

```
结算增强：
  ☐ 自动换汇结算：收 USDT → 自动换汇 → 入商户法币账户（USD）

  ☐ 支付页面多语言支持
```

```
多通道：

支付币种扩展：
  ☐ 支持更多稳定币（USDC 更多网络 / DAI 等）
  ☐ 支持 ETH / BTC 等非稳定币支付（汇率波动风险由商户承担）

高级功能：

  ☐ 预授权：先授权后扣款
  ☐ 买家钱包直连：MetaMask / WalletConnect 等
  ☐ 收单数据分析：转化率、币种分布、网络分布报表
```
