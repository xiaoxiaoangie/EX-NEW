# 收付款 / 换汇 / 承兑（On-Ramp / Off-Ramp）边界定义分析

> 版本：v1.0 | 日期：2026-02-25 | 状态：讨论中（待最终确认）
> 关联文档：sa-product-listing.md / mp-menu-structure.html / onramp-complete.md / offramp-complete.md

## 1. 背景与问题

### 1.1 为什么需要厘清边界

在 EX 平台产品设计中，以下概念存在交叉和歧义：

- **FX（换汇）**：法币之间的兑换
- **On-Ramp（买入数币）**：法币 → 数币
- **Off-Ramp（卖出数币）**：数币 → 法币
- **Payins（收款）**：外部资金进入体系
- **Payouts（付款/提现）**：体系内资金出到外部
- **同名充值/提现**：自己的钱在体系内外流动

**核心矛盾：On-Ramp / Off-Ramp 既有"兑换"属性，又有"收付款"属性。** 当用户从外部银行充值并一步买入 USDT 时，这笔操作是"收款"还是"换汇"还是一个独立的编排产品？

### 1.2 需要回答的问题

1. On-Ramp / Off-Ramp 的产品归类（换汇？收付款？独立编排？）
2. FX 和 On-Ramp / Off-Ramp 是否共用同一个入口
3. 涉及外部资金流动的 Ramp 操作如何与纯体系内兑换区分
4. 这些定义如何影响 MP 端菜单、TP 端订单查询、产品目录

## 2. 场景拆解

### 2.1 五个关键场景

| 编号 | 场景 | 操作描述 | 资金流向 |
|------|------|---------|---------|
| S1 | 普通换汇 | USD 账户 → EUR 账户 | 体系内法币 → 体系内法币 |
| S2 | 买U（外部充值，一单到底） | 外部银行 → 充值法币 → 自动换成 USDT | 外部 → 体系内数币 |
| S3 | 买U（体系内余额直接换） | 法币账户 USD → 数币钱包 USDT | 体系内法币 → 体系内数币 |
| S4 | 卖U提现（一单到底） | USDT → 换成法币 → 提现到外部银行 | 体系内数币 → 外部 |
| S5 | 卖U到余额 | USDT → 换成 USD 到法币账户 | 体系内数币 → 体系内法币 |

**已确认：S2 和 S4 都是一步操作（一单到底），用户不需要分步执行。**

### 2.2 场景特征对比

| 场景 | 涉及外部资金？ | 涉及兑换？ | 跨资产类型？ | 用户操作步数 |
|------|--------------|-----------|------------|------------|
| S1 普通换汇 | ❌ | ✅ | ❌ 法币↔法币 | 1 步 |
| S2 充值买U | ✅ 外部入金 | ✅ | ✅ 法币→数币 | 1 步（一单到底） |
| S3 余额买U | ❌ | ✅ | ✅ 法币→数币 | 1 步 |
| S4 卖U提现 | ✅ 外部出金 | ✅ | ✅ 数币→法币 | 1 步（一单到底） |
| S5 卖U到余额 | ❌ | ✅ | ✅ 数币→法币 | 1 步 |

## 3. 原子能力模型

### 3.1 四个原子能力

系统底层只有 4 个原子能力，所有产品都是这些能力的组合：

| 原子能力 | 说明 | 方向 |
|---------|------|------|
| **Deposit（入金）** | 外部资金 → 体系内账户 | 外 → 内 |
| **Withdraw（出金）** | 体系内账户 → 外部 | 内 → 外 |
| **Conversion（兑换）** | 体系内 A 账户 → 体系内 B 账户（含法币↔法币、法币↔数币） | 内 → 内 |
| **Transfer（转账）** | 体系内资金付给非同名第三方 | 内 → 外（第三方） |

### 3.2 所有场景的原子能力拆解

| 场景 | 原子能力组合 | 涉及外部？ | 是否编排？ |
|------|------------|-----------|----------|
| S1 FX（USD→EUR） | Conversion | ❌ | ❌ 单一能力 |
| S2 充值买U | **Deposit + Conversion** | ✅ | ✅ 两个能力编排 |
| S3 余额买U | Conversion | ❌ | ❌ 单一能力 |
| S4 卖U提现 | **Conversion + Withdraw** | ✅ | ✅ 两个能力编排 |
| S5 卖U到余额 | Conversion | ❌ | ❌ 单一能力 |
| 同名充值 | Deposit | ✅ | ❌ 单一能力 |
| 同名提现 | Withdraw | ✅ | ❌ 单一能力 |
| VA 收款 | Deposit（第三方入金） | ✅ | ❌ 单一能力 |
| 法币付款 | Transfer | ✅ | ❌ 单一能力 |
| 链上付币 | Transfer | ✅ | ❌ 单一能力 |
| Remittance（收→换→付） | Deposit + Conversion + Transfer | ✅ | ✅ 三个能力编排 |

### 3.3 关键发现

1. **Conversion 是统一的底层兑换能力**，覆盖所有体系内资产互换（法币↔法币、法币↔数币、数币↔数币）
2. **S1、S3、S5 本质上都是纯 Conversion**，只是兑换的资产类型不同
3. **S2、S4 是编排操作**，在 Conversion 基础上叠加了 Deposit 或 Withdraw
4. **On-Ramp / Off-Ramp 不是一个单一能力，而是一个产品入口**，根据用户选择的资金来源/去向，决定底层是纯 Conversion 还是 Deposit+Conversion / Conversion+Withdraw

## 4. 竞品分析

### 4.1 Stripe

| 产品 | 定位 | 说明 |
|------|------|------|
| Payments | 收单（收款） | 独立产品，专注在线支付 |
| Payouts | 付款 | 独立产品，体系内→外部 |
| Treasury | 资金账户 | 独立产品，余额管理、内部转账 |
| Issuing | 发卡 | 独立产品，卡管理+卡交易+卡余额全在一起 |

**Stripe 不涉及数币，没有 On-Ramp / Off-Ramp 概念。** 但其设计原则是：**按产品线独立，每个产品内闭环。**

### 4.2 Airwallex

| 产品 | 定位 | 说明 |
|------|------|------|
| Global Accounts | 多币种账户 | 开户、持有、余额管理 |
| Payments | 收款 | 外部入金 |
| Payouts | 付款 | 外部出金 |
| Conversions | 换汇 | **体系内**法币↔法币，独立 API |

**Airwallex 不涉及数币。** Conversion 是独立的体系内能力，与 Payments/Payouts 分开。如果需要"付款+换汇"，是两个 API 调用，平台层封装成一步。

### 4.3 MoonPay / Transak / Ramp Network（专业 On/Off-Ramp 服务商）

| 产品 | 定位 | 说明 |
|------|------|------|
| On-Ramp | 法币买币 | 用户付法币（银行/卡）→ 收到数币，**一单到底** |
| Off-Ramp | 数币卖币 | 用户付数币 → 收到法币（银行），**一单到底** |

**关键特征：**
- On-Ramp / Off-Ramp 被视为**独立产品**，不归入换汇，也不归入收付款
- 本质是编排产品：封装了入金/出金 + 兑换
- 用户视角：一个统一入口，选币种和支付方式即可

### 4.4 Coinbase / Kraken（交易所）

| 功能 | 定位 | 说明 |
|------|------|------|
| Buy / Sell | 买入/卖出 | 统一入口，选择资金来源（余额 or 银行账户） |
| Convert | 币币兑换 | 体系内数币↔数币 |
| Deposit / Withdraw | 充值/提现 | 法币或数币的出入金 |

**关键特征：**
- **Buy/Sell 是统一入口**，不区分"从余额买"还是"充值买"
- 用户在 Buy 页面选择支付方式（USD Balance / Bank Account / Card）
- 系统根据选择决定底层是纯 Conversion 还是 Deposit + Conversion
- **Convert 专门用于体系内币币兑换**（USDT→BTC 等）

### 4.5 竞品总结

| 平台类型 | FX（法币↔法币） | 法币↔数币（体系内） | 法币↔数币（涉及外部） | 设计模式 |
|---------|----------------|-------------------|---------------------|---------|
| **传统支付**（Stripe/Airwallex） | 独立 Conversion | 不涉及 | 不涉及 | 按能力拆分 |
| **专业 Ramp**（MoonPay/Transak） | 不涉及 | 不涉及 | On-Ramp / Off-Ramp 独立产品 | 编排产品 |
| **交易所**（Coinbase/Kraken） | 不适用 | Buy/Sell 统一入口 | Buy/Sell 统一入口 | **统一入口，按资金来源区分** |

**EX 平台同时涉及法币和数币，最接近 Coinbase 模式** — 需要一个统一的兑换入口，内部根据资金来源/去向决定底层能力组合。

## 5. 设计决策

### 5.1 已确认的决策

| 编号 | 决策 | 原因 |
|------|------|------|
| D1 | **FX + Buy Crypto + Sell Crypto 统一到一个 Exchange 入口** | 三者本质都是 Conversion，只是资产类型不同。统一入口降低用户认知成本 |
| D2 | **On-Ramp 支持两种方式：从余额 / 从外部充值** | 同一个入口，用户选择资金来源，系统决定底层是纯 Conversion 还是 Deposit + Conversion |
| D3 | **Off-Ramp 支持两种目标：到法币余额 / 到外部银行** | 同一个入口，用户选择目标，系统决定底层是纯 Conversion 还是 Conversion + Withdraw |
| D4 | **S2（充值买U）和 S4（卖U提现）都是一步操作** | 用户不需要分步执行，系统一单到底 |

### 5.2 Conversion 统一模型

```
Exchange（统一兑换入口）
│
├── 选择兑换方向
│   ├── 法币 → 法币（FX）
│   ├── 法币 → 数币（Buy Crypto / On-Ramp）
│   └── 数币 → 法币（Sell Crypto / Off-Ramp）
│
├── 选择资金来源（Buy / On-Ramp 时）
│   ├── 从体系内余额 → 纯 Conversion
│   └── 从外部银行充值 → Deposit + Conversion（一单到底）
│
└── 选择资金去向（Sell / Off-Ramp 时）
    ├── 到体系内法币余额 → 纯 Conversion
    └── 到外部银行账户 → Conversion + Withdraw（一单到底）
```

### 5.3 边界定义总结

| 概念 | 定义 | 底层能力 | 产品归类 |
|------|------|---------|---------|
| **FX** | 体系内法币↔法币兑换 | Conversion | Exchange |
| **Buy Crypto（体系内）** | 体系内法币余额→数币 | Conversion | Exchange |
| **Sell Crypto（体系内）** | 体系内数币→法币余额 | Conversion | Exchange |
| **On-Ramp（含外部入金）** | 外部银行→法币→数币，一单到底 | Deposit + Conversion | Exchange（编排模式） |
| **Off-Ramp（含外部出金）** | 数币→法币→外部银行，一单到底 | Conversion + Withdraw | Exchange（编排模式） |
| **同名充值** | 外部银行→体系内法币/数币账户 | Deposit | 账户（Assets） |
| **同名提现** | 体系内法币/数币→外部银行/钱包 | Withdraw | 账户（Assets） |
| **VA 收款** | 第三方付款→体系内（非同名） | Deposit（第三方） | 收款（Payins） |
| **法币付款** | 体系内→第三方（非同名） | Transfer | 付款（Payouts） |
| **Remittance** | 收→换→付，一单到底 | Deposit + Conversion + Transfer | 编排产品（独立） |

## 6. 对产品目录和菜单的影响

### 6.1 产品目录（sa-product-listing）

维持三层分类不变，但澄清 Exchange 的范围：

| 层 | 产品 | 说明 |
|----|------|------|
| **账户类** | 法币账户、数币钱包 | 开户 + 持有 + 同名充提 |
| **收付类** | Payins、Payouts | 非同名收付 |
| **业务类** | **Exchange**（含 FX / On-Ramp / Off-Ramp）、VCC、Crypto Checkout | Exchange 是统一的兑换产品 |

### 6.2 MP 端菜单影响

```
Assets（资产）
├── Fiat Accounts（法币账户：余额、同名充提）
├── Crypto Wallet（数币钱包：余额、同名充提币）
└── Exchange（统一兑换入口）
    ├── FX（法币↔法币）
    ├── Buy Crypto（法币→数币，支持从余额或外部充值）
    └── Sell Crypto（数币→法币，支持到余额或外部提现）
```

### 6.3 TP 端客户订单查询影响

```
换汇（Exchange）
├── FX 换汇单
├── On-Ramp 单（买币，含体系内和外部充值）
└── Off-Ramp 单（卖币，含到余额和外部提现）
```
