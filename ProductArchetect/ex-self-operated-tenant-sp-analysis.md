## 1. 定义与类比

```
EX 平台的四种参与者：

              租户侧（拿客户）              SP 侧（出资金能力）
  外部       外部租户                      外部 SP
             （越南 PSP、大代理、货代平台）   （本地 PSP、结汇 SP、OTC）

  自营       自营租户                      自营 SP
             ≈ 集团自己开的「大代理」        ≈ 集团自己开的「OTC/结汇台」
             承接无系统、无归属的直客        承接 A/B 合规水位覆盖不了、
                                          或 A/B 签约太重的资金执行
```

**一句话定位**：

- **自营租户** = 平台上的「集团直营门店」：客户没系统、没适配的外部租户可挂靠，又不能直接进 A/B 时，由它承接。形态上就是画像①大代理，只是老板是自己。
- **自营 SP** = 平台上的「集团自营执行台」：某些资金执行（典型如数币背景资金的结汇）A 因合规水位不能碰、B 只管数币段，而用 A/B 主体去签外部渠道内耗巨大——由一个**非持牌/轻持牌的自营主体**作为 SP 挂入 EX 编排，快速补位。**它的定位是补充生态、不是竞争：自营 SP 只做外部 SP 覆盖不了的通道，原则上不参与、不抢外部 SP 能做的单——这是原则和底线（详见 6.3）。**

**关键区分**：自营租户解决的是「客户往哪放」，自营 SP 解决的是「这段钱谁来执行」。两者经常被混着讨论，但价值逻辑和风险等级完全不同——**自营租户主要是商业问题，自营 SP 主要是合规问题**。

## 2. 为什么必须存在：三个真实场景

### 场景 ①：客户缺系统，直接去 A/B 接不住

一个中型货代：有真实的收款+付款+换汇需求，但没有自己的系统，也不属于任何外部租户。直接给他 A 的 API？他没有开发能力。给他 A 的商户后台？A 的后台是为中国电商卖家设计的，覆盖不了他跨 SP 的组合需求（见场景③）。

→ **必须有一个租户来承接这类「有需求、无系统、无归属」的客户**。外部租户没长出来之前，这个租户只能是自营的。

### 场景 ②：A/B 的合规水位不齐，产生「能力裂缝」

最典型的例子：

```
客户持 U → B 承兑（数币段，合规） → 资金要结汇成本币/换汇出境
  ↓
  这笔钱能进 A 结汇吗？ 不能。
  A 的合规水位（MSO 合规框架）不接受数币背景资金。
  ↓
  但市场上存在合规水位可以承接数币背景资金结汇的 SP。
  它不能挂在 A 后面（A 不背这个书），
  但可以作为独立 SP 直接挂入 EX 编排。
  ↓
  客户在 EX 的账本下「跨 SP 操作」：
  数币段走 B → 结汇段走该结汇 SP → 出金段视情况走 A 或本地 SP
```

**这就是 EX 编排层存在的最硬核理由**：A、B、各 SP 的合规水位天然不齐，**单一主体的合规边界 = 客户旅程的断点**；EX 的价值就是让客户的完整旅程由多个「各自在其合规水位内」的主体分段接力完成。这个能力 A 给不了（它不能编排自己不敢碰的段），B 也给不了。

而在外部结汇 SP 谈成之前、或外部 SP 覆盖不了的量上，**自营 SP 就是这个裂缝的补位者**。

### 场景 ③：用 A/B 主体去签渠道，内耗大、周期长

A/B 是持牌机构，机构对机构的签约要过双方合规、法务、尽调，动辄数月；某些渠道（尤其 OTC 类）A/B 的合规委员会根本不会批。用一个非持牌的自营 SP 主体去签，快、轻、灵活。

→ **速度价值是真实的，但注意：这条理由是三条中最危险的一条**（详见风险 R1）——"A/B 合规不批，所以换个不受合规约束的主体去做"，这句话在监管听来就是规避。

## 3. 自营租户的价值

| #  | 价值                                     | 说明                                                                                                                                                   |
| -- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| V1 | **承接兜底**                       | 有需求、无系统、无归属的直客有地方放，不流失。没有它，EX 冷启动期只能眼看客户走掉                                                                      |
| V2 | **产品实验田**                     | 平台所有新功能（风控、路由、跨 SP 操作）先在自营租户上跑通，拿真实交易验证，再开放给外部租户——自营租户就是永远在线的 staging 环境 + 第一个吃螃蟹的人 |
| V3 | **样板房**                         | 对外部租户展示「一个租户长什么样、怎么赚钱」的活案例；BD 签外部租户时最有力的演示材料                                                                  |
| V4 | **跨 SP 组合的唯一入口（现阶段）** | 场景②的跨 SP 客户旅程，现阶段只有自营租户能承接——外部租户还没有能力理解和运营这种组合                                                               |
| V5 | **全链路毛利**                     | 自营租户的客户，租户分成层的利润不用分给别人。冷启动期的现金流补充                                                                                     |
| V6 | **需求雷达**                       | 直客的真实需求（什么 corridor、什么组合、卡在哪）第一手回流产品，不经过外部租户的过滤                                                                  |

**类比校准**：自营租户 ≈ 京东自营之于京东平台、亚马逊 Retail 之于 Marketplace。平台史的规律很明确：**自营先行验证品类 → 平台放大**，但也要接受同一个规律的后半段——**自营与第三方的利益冲突会伴随平台一生**（见 R3）。

## 4. 自营 SP 的价值

| #   | 价值                   | 说明                                                                                                                                |
| --- | ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| V7  | **合规裂缝补位** | 场景②：数币背景资金结汇等 A/B 水位覆盖不了的段，在外部 SP 到位前由自营 SP 执行，客户旅程不断链                                     |
| V8  | **签约速度**     | 非持牌主体签渠道/对手方，绕开 A/B 机构级签约的内耗，周期从月降到周                                                                  |
| V9  | **路由砝码**     | 与外部 SP 谈判时，自营 SP 的存在 = 「你不做有人做」的底牌，压价与 SLA 谈判的筹码                                                    |
| V10 | **能力孵化**     | 新 corridor/新业务先用自营 SP 小规模试跑，验证合规路径与单位经济后，再决定是引外部 SP 还是升级为持牌能力（注入 A/B 或新设持牌主体） |
| V11 | **利差留存**     | OTC/结汇价差是跨境链条中最厚的利润段之一，自营 SP 让这段毛利留在集团内                                                              |

**类比校准**：自营 SP ≈ 交易所的自营做市商。价值真实（流动性/补位/利差），但**利益冲突和合规风险也和自营做市商一样经典**——FTX/Alameda 就是这个结构失控的极端案例。这个类比应该让我们清醒。

## 5. 风险清单（按凶险程度排序）

### R1 · 监管套利定性风险（自营 SP，致命级）

**把话说透**：「A 因 MSO 合规不能做数币背景资金结汇 → 用非持牌自营 SP 做」——这个结构的另一种描述是：**把持牌主体不敢碰的业务，转移到一个没有牌照约束（也没有牌照保护）的主体上做**。风险没有消失，只是从「A 被监管处罚」变成了「自营 SP 裸奔 + 集团被穿透追责」。

- 监管穿透原则下，同一集团内「持牌主体拒绝 + 非持牌主体承接」的安排，会被认定为**故意规避（willful circumvention）**，殃及 A/B 牌照本身——这是用 EX 的便利换 A/B 的命。
- **唯一站得住的逻辑**：自营 SP 做的业务，必须在**它自己所在辖区是合法或无需牌照的**（如某些辖区 OTC 结汇不需牌照/已获豁免），并有法律意见书背书。「集团内部左手不能做所以右手做」永远不成立，「这件事在右手所在辖区本来就合法」才成立。
- ⚠️ 每一个自营 SP 承接的业务类型，都需要**逐辖区法律意见书**，没有意见书的业务 = 不做。

### R2 · 风险传染（自营 SP，致命级）

自营 SP 出事（账户冻结、对手方暴雷、被执法调查）时，它不是一个可以切割的外部供应商——它是集团的：

- 声誉传染：媒体/监管眼中「EX 平台的自家 OTC 出事」＝ EX 出事；
- 牌照传染：调查会顺藤摸到 A/B；
- 资金传染：若资金/账户与平台或 A/B 有任何混同，冻结会扩散。

外部 SP 出事我们换一家，自营 SP 出事我们上头条。**自营 SP 的风险上限，就是整个 ABE 的风险上限。**

### R3 · 既当裁判又当运动员（自营租户，战略级）

自营租户与外部租户抢客户，是「假中立」问题的终极形态：

- 定位文档对外承诺的四件套（non-circumvention、数据不外用……），**自营租户是天然的破坏者嫌疑人**：BD 拿到一个好客户，放进自营租户（全链路毛利）还是推荐给外部租户（只赚平台费）？激励天然指向自营。
- 外部租户一旦察觉「平台自己开店和我抢客户，还看得到我的数据」，赋能叙事崩塌——**越南 PSP 赋能方案的全部信任基础被自营租户一票否决**。

### R4 · 舒适区陷阱（自营租户，战略级）

自营租户签约快、利润厚、不用谈判——组织会自然滑向「所有客户都塞进自营租户」。后果：

- 外部租户生态永远长不出来，EX 退化成**一家做直客的支付公司**（且是不持牌的那种）；
- 平台估值叙事消失（见 §7）；
- 回头看就是：我们用「过渡安排」杀死了终局。

**自营是止痛药，止痛药吃成主粮，病就成了绝症。**

### R5 · 路由公平性坍塌（自营 SP，商誉级）

自营 SP 参与路由 = 裁判员亲自下场跑步。外部 SP 会问：单子是按 SLA 分的，还是先喂饱自家 OTC？一旦坐实偏袒，外部 SP 停止供给最优价格，多 SP 聚合的根基（Q4 承诺的量化 SLA 路由）失去可信度。

### R6 · 财务混同与关联交易（两者，资本级）

自营租户/自营 SP 的收入对 EX 是关联交易。占比过高时，资本市场尽调的结论是：**「平台收入是左手倒右手，第三方生态是装饰」**——平台倍数直接打回通道倍数（参考：电商平台上市时自营 GMV 占比都是招股书核心披露项）。

## 6. 治理机制：怎么用而不被反噬

### 6.1 总原则：自营是脚手架，不是大楼

```
每一个自营主体设立时，必须同时回答三个问题：
  ① 它替代谁？（这个功能的终局形态是外部租户/外部SP/持牌能力）
  ② 它何时退？（日落条件：外部替代者达到什么规模/SLA 时缩减）
  ③ 它出事谁兜？（防火墙结构 + 责任预案）
回答不了三问的自营主体，不批设立。
```

### 6.2 自营租户的四条纪律

| 纪律                   | 内容                                                                                                                           |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **同价同权**     | 自营租户拿的平台价格、功能、数据权限与外部租户完全一致，不得用内部优惠价抢客                                                   |
| **占比上限**     | 自营租户 GMV 占平台比例设上限并逐年递减（建议：Y1 ≤70% → Y2 ≤50% → Y3 ≤30%），写进平台 KPI                                |
| **客户归属仲裁** | BD 客户进自营还是推外部租户，按公开规则（客户所在地/行业/外部租户覆盖能力）裁决，不由 BD 自由裁量                              |
| **利益隔离**     | 自营租户运营团队与平台（路由/数据/风控）团队物理隔离，自营租户不得访问其他租户任何数据——把对外承诺的数据四件套先用在自己身上 |

### 6.3 自营 SP 的七条红线

| 红线                      | 内容                                                                                                     |
| ------------------------- | -------------------------------------------------------------------------------------------------------- |
| **补位不抢位（原则底线）** | 自营 SP 只做**外部 SP 覆盖不了的通道**（合规裂缝段、没人做的新 corridor）；凡外部 SP 能做的单，自营**原则上不进入候选池、不与外部 SP 竞争抢单**。这是供给侧信任的地基，对外可讲、对内必守（对 R3/R5） |
| **辖区合法性前置**  | 每类业务逐辖区法律意见书，无意见书不开业（对 R1）                                                        |
| **独立防火墙主体**  | 独立法人、独立账户、独立资金，与 EX/A/B 无资金往来混同；股权结构上考虑隔离层（对 R2）                    |
| **名单制 + 限额制** | 高危业务（数币背景结汇）只服务白名单客户，单客/单日限额，资金背景 100% 穿透留档（对 R1/R2）              |
| **同评分、无内部优惠价** | 在**确需补位**的通道里，自营 SP 与（若有）外部 SP 用同一套评分、决策日志可审计，且**不享受任何内部优惠价/优先权**——即使补位也不靠"自己人身份"取胜（对 R5，呼应 Q16 可审计承诺） |
| **禁做清单**        | 无论利润多厚不碰：制裁关联、无背景资料的数币结汇、任何「唯一目的是绕开 A/B 合规决定」的单笔业务（对 R1） |
| **日落条款**        | 每个自营 SP 承接的能力段，注明外部/持牌替代路径与触发条件；自营 SP 的成功标志是自己变小（对 R4）         |

### 6.4 一张判断表：这单业务该给谁

```
客户/业务进来 →
  ① A/B 水位内 + 客户有系统？        → 直连 A/B（不硬塞平台）
  ② A/B 水位内 + 客户无系统？        → 外部租户优先，无覆盖则自营租户
  ③ A/B 水位外 + 目标辖区有合法路径？ → 外部 SP 优先，无覆盖则自营 SP（六条红线内）
  ④ A/B 水位外 + 任何辖区都无合法路径？→ 不做。这不是商机，是案底。
```

## 7. 对资本叙事的影响

- **短期（融资期）**：自营租户/SP 支撑了 GMV 和收入的冷启动，让数据好看——但必须在数据房里**主动、清晰地拆分自营 vs 第三方占比**，并展示第三方占比的增长曲线。藏着掖着被尽调发现，估值折价比坦白高一个量级。
- **中期（上市准备）**：关联交易规范化是必答题——自营租户/SP 与平台的定价协议、独立审计、董事会关联交易委员会，提前 2 年布局。
- **叙事口径**：对投资人讲「自营是平台的种子客户与做市商，占比每年递减 X%」——递减曲线本身就是平台故事最有力的证据。**自营占比的下降速度 = 平台化的真实速度**，这是比 GMV 更诚实的北极星指标。

## 8. 自营体系的灵魂拷问（内部版 QA）

**SQ1.「自营 SP 做 A 不能做的结汇，这不就是监管套利吗？」**
诚实回答：如果仅有的逻辑是「A 不能做所以换个主体做」，是的，就是套利，且会反噬 A/B 牌照。唯一站得住的版本：该业务在自营 SP 所在辖区本来合法（有法律意见书），A 不做是因为 A 的辖区/牌照类型要求更严——**不同辖区不同水位是全球金融的常态，利用它合法，伪装它违法**。红线：意见书前置 + 禁做清单。

**SQ2.「自营租户和外部租户抢客户怎么办？BD 一定优先塞自营。」**
承认激励天然扭曲，靠制度不靠自觉：客户归属公开规则 + 自营占比上限入 KPI + 同价同权。终极答案是让 BD 签外部租户的提成 ≥ 塞自营租户——**用钱纠正钱造成的问题**。

**SQ3.「既然自营租户全链路毛利更高，为什么不全自营？」**
因为全自营 = 放弃平台估值 + 放弃本地化生存权（东南亚展业没有本地伙伴寸步难行）+ 规模天花板是自己团队的人数。自营毛利高是单客经济，平台是网络经济——**拿单客毛利换网络，是所有平台的既定亏损，不是漏洞**。

**SQ4.「自营 SP 出事，防火墙真的挡得住吗？」**
法律上挡得住部分（独立法人、无资金混同），声誉上挡不住——所以六条红线的真正逻辑不是「出事时切割」，而是**让它出不了大事**：名单制、限额制、禁做清单，把自营 SP 的风险敞口钉死在可赔付范围内。

**SQ5.「跨 SP 操作听起来很美，客户资金在多个主体间接力，出了差错算谁的？」**
这正是 EX 账本的存在意义：每一段的资金主体、交接点、状态在统一账本上有唯一记录，差错按段归责（关联文档 §6.8 异常流状态机）。但要承认：**跨 SP 旅程的差错处理复杂度是单主体的数倍，客服与差错 SOP 必须在开放外部客户前先在自营租户上磨熟**——这也是自营租户作为实验田的价值闭环。

## 附：一页总结

```
为什么要有：
  自营租户 = 承接「有需求、无系统、无归属」的客户 + 产品实验田 + 样板房
  自营 SP  = 补 A/B 合规水位裂缝 + 签约速度 + 路由谈判砝码

最大的价值：让客户的完整旅程（数币段→结汇段→出金段）
  在合规水位不齐的多主体间接力完成 —— 这是 EX 编排层的硬核证明

最大的风险：
  自营 SP  = 监管套利定性 + 风险传染（能烧掉 A/B 牌照）
  自营租户 = 裁判下场 + 舒适区陷阱（能杀死平台叙事）

一句话纪律：
  自营是脚手架，不是大楼——
  每个自营主体设立时必须回答：替代谁、何时退、出事谁兜。
  自营占比的下降速度，就是平台化的真实速度。
```
