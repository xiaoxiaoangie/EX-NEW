# Changelog 生成流程

## 1. 信息收集

从当前开发流程的各阶段产出物中收集变更信息：

1. **业务需求文档**（docs/designs/full-prd.md）：本服务业务设计的全量文档，提取变更的需求描述
2. **架构设计文档**（docs/designs/full-architecture-design.md）：提取领域模型变更、业务规则变更
3. **接口设计文档**（docs/designs/full-api-design.md）：提取接口变更、错误码变更
4. **工程代码**：扫描新增和修改的代码文件
5. **数据库脚本**（db/changelog/）：提取数据库表结构变更

## 2. 变更分类

将收集到的信息按以下维度分类：

- **接口变更**：新增/修改/废弃的接口，区分兼容与不兼容变更
- **数据库变更**：新增/修改的表、字段、索引
- **业务规则变更**：新增/修改/删除的业务规则
- **错误码变更**：新增/修改的错误码
- **代码变更**：新增/修改的文件清单

## 3. 文档生成

在任务目录 `docs/requirements/{JIRA}-{YYYYMMDD}-{变更简述}/` 下生成 changelog 文档。

当由开发流程（`ex-backend-develop-workflow`）调度时，会传入 JIRA 单号 `{JIRA}`，文件名加上 JIRA 前缀：
```
{JIRA}-changelog-{YYYYMMDD}-{变更简述}.md
```
- `{JIRA}`：JIRA 单号（由开发流程传入，独立调度时可省略前缀）

### 文档结构模板

参照 [changelog-template.md](references/changelog-template.md) 生成，包含以下章节：

1. **基本信息**：变更日期、类型、影响模块、变更描述
2. **需求说明**：变更的业务背景和具体规则
3. **变更影响分析**：
   - 接口变更（标注兼容/不兼容）
   - 新增错误码
   - 数据库变更（含表结构和索引）
   - 业务规则变更
4. **变更文件清单**：
   - 新增文件（路径 + 说明）
   - 修改文件（路径 + 变更内容）
   - 数据库迁移文件
   - 文档变更
5. **接口调用方迁移指南**：变更前后的请求/响应对比示例
6. **注意事项**：部署注意事项、存量数据处理、兼容性说明

## 4. 生成规则

### 接口变更判定
- 新增必填字段 → 不兼容变更
- 新增可选字段 → 兼容变更
- 修改字段类型 → 不兼容变更
- 新增响应字段 → 兼容变更
- 删除响应字段 → 不兼容变更
- 请求体结构变更 → 不兼容变更

### 文件路径规范
- 代码文件路径使用相对于模块的短路径，如 `domain/aggregate/Account.java`
- 数据库文件使用完整资源路径，如 `resources/db/changelog/1.0.0/001-create-tables.xml`
- 文档文件使用 docs/ 相对路径

### 迁移指南要求
- 对每个不兼容的接口变更，提供变更前后的完整请求/响应 JSON 示例
- 对新增的响应字段，展示包含新字段的响应示例
- 示例数据应贴近真实业务场景

### 注意事项要求
- 必须包含数据库迁移提醒（如有数据库变更）
- 必须说明接口兼容性影响
- 如有存量数据影响，必须说明处理方案
- 如有新增约束（如唯一性），必须提醒调用方
