## 1. 接口覆盖度

### 1.1 基础覆盖
| 编号 | 检查项 | 级别 |
|------|--------|------|
| IC-01 | 接口设计文档中的每个接口在测试用例文档中都有对应的测试用例章节 | P0 |
| IC-02 | 每个接口至少有1个正常场景测试用例 | P0 |
| IC-03 | 每个接口至少有1个异常场景测试用例 | P0 |
| IC-04 | 测试用例中的接口信息（路径、方法、描述）与接口设计文档一致 | P0 |

### 1.2 场景覆盖
| 编号 | 检查项 | 级别 |
|------|--------|------|
| IC-05 | 查询接口覆盖有数据返回和无数据返回两种场景 | P1 |
| IC-06 | 分页查询接口覆盖首页、中间页、末页场景 | P1 |
| IC-07 | 批量操作接口覆盖空列表、单条、多条场景 | P1 |
| IC-08 | 创建接口覆盖必填参数最小集和全参数两种场景 | P1 |
| IC-09 | 修改接口覆盖部分字段修改和全字段修改场景 | P2 |
| IC-10 | 删除接口覆盖存在数据和不存在数据两种场景 | P1 |

| 编号 | 检查项 | 级别 |

## 8. 用例规范

### 8.1 编号规范
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TC-01 | 用例编号符合 TC-{接口序号}{用例序号} 格式（如 TC-0101） | P2 |
| TC-02 | 用例编号在全量文档中唯一，无重复编号 | P1 |
| TC-03 | 增量文档中的用例编号与全量文档保持一致 | P1 |

### 8.2 命名规范
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TC-04 | 用例名称清晰表达测试意图（如"创建账户-名称为空-返回参数校验错误"） | P2 |
| TC-05 | 用例名称包含操作、条件和预期结果三要素 | P2 |
| TC-06 | 同一接口下的用例名称不重复 | P1 |

### 8.3 前置条件
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TC-07 | 前置条件描述充分，包含测试所需的数据状态 | P2 |
| TC-08 | 状态流转用例的前置条件明确指定当前状态 | P1 |
| TC-09 | 数据隔离用例的前置条件明确指定服务商/租户信息 | P1 |

### 8.4 请求参数
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TC-10 | 请求参数具体明确，使用示例值而非模糊描述 | P2 |
| TC-11 | 异常用例的请求参数明确标注哪个参数是异常值 | P2 |
| TC-12 | 请求参数中的字段名与接口设计文档一致 | P1 |

## 9. 文档规范

### 9.1 增量文档
| 编号 | 检查项 | 级别 |
|------|--------|------|

| DOC-05 | 增量文档的废弃用例章节列出废弃原因（如有） | P2 |

### 9.2 全量文档
| 编号 | 检查项 | 级别 |
|------|--------|------|
| DOC-06 | 全量文档包含最后更新日期和用例总数 | P2 |
| DOC-07 | 全量文档按接口分章节组织，每个接口包含接口信息和测试用例表 | P1 |
| DOC-08 | 全量文档中的用例编号连续且无遗漏 | P1 |
| DOC-09 | 增量文档中的变更已正确同步到全量文档 | P0 |

### 9.3 表格规范
| 编号 | 检查项 | 级别 |
|------|--------|------|
| DOC-10 | 测试用例表包含完整列：用例编号、用例名称、前置条件、请求参数、预期结果、用例类型 | P1 |
| DOC-11 | 用例类型标注正确（正常/异常） | P2 |
