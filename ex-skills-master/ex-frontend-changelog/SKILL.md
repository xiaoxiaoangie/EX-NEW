# 前端变更记录

基于开发流程的产出物，自动生成结构化的 changelog 文档。

## 输入

| 来源 | 说明 |
|------|------|
| 全流程产出物 | 任务拆解、技术方案、代码变更、联调报告 |

## 输出

文件：`docs/requirements/{JIRA}-{YYYYMMDD}-{变更简述}/{JIRA}-changelog-{YYYYMMDD}-{变更简述}.md`

### 文件命名

当由开发流程（`ex-frontend-develop-workflow`）调度时，会传入 JIRA 单号 `{JIRA}`，文件名加上 JIRA 前缀：

{JIRA}-changelog-{YYYYMMDD}-{变更简述}.md

- `{JIRA}`：JIRA 单号（由开发流程传入，独立调度时可省略前缀）
- `{YYYYMMDD}`：日期
- `{变更简述}`：简要概括变更内容

## 变更记录结构

```markdown
# {JIRA} 变更记录

## 基本信息

| 项目 | 值 |
|------|-----|
| JIRA | {JIRA} |
| 日期 | {日期} |
| 类型 | 新增功能 / 功能变更 / Bug修复 / 优化 |
| 概述 | {一句话描述} |

## 页面变更

| 页面 | 路由 | 变更类型 | 说明 |
|------|------|---------|------|
| {页面名} | /{path} | 新增/修改/删除 | {说明} |

## 组件变更

| 组件 | 类型 | 变更类型 | 说明 |
|------|------|---------|------|
| {组件名} | 通用/业务/页面私有 | 新增/修改/删除 | {说明} |

## API 变更

| API 模块 | 函数 | 变更类型 | 说明 |
|---------|------|---------|------|
| {模块名} | {函数名} | 新增/修改/删除 | {说明} |

## 类型变更

| 类型文件 | 类型名 | 变更类型 | 说明 |
|---------|-------|---------|------|
| {文件名} | {类型名} | 新增/修改/删除 | {说明} |

## 路由变更

| 路由 | 变更类型 | 说明 |
|------|---------|------|
| /{path} | 新增/修改/删除 | {说明} |

## Store 变更

| Store | 变更类型 | 说明 |
|-------|---------|------|
| {store名} | 新增/修改/删除 | {说明} |

## 依赖变更

| 依赖 | 变更前 | 变更后 | 说明 |
|------|-------|-------|------|
| {包名} | {旧版本/无} | {新版本/移除} | {说明} |

## 文件清单

### 新增文件
- `{文件路径}`

### 修改文件
- `{文件路径}`

### 删除文件
- `{文件路径}`
```

## 生成规则

1. **自动收集**：从流程各阶段的产出物中提取变更信息
   - 任务拆解文档 → 页面变更、组件变更
   - 代码变更 → 文件清单、API变更、类型变更
   - 联调报告 → 接口适配变更

2. **分类归纳**：按变更类型（新增/修改/删除）分类

3. **省略空节**：如果某个变更类型没有内容，省略该节

4. **增量追加**：如果 changelog 文件已存在，追加新的变更记录而非覆盖
