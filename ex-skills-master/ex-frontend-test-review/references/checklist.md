## 1. 页面覆盖度

### 1.1 状态覆盖
| 编号 | 检查项 | 级别 |
|------|--------|------|
| PC-01 | 每个页面组件至少覆盖 loading 状态的渲染用例 | P0 |
| PC-02 | 每个页面组件至少覆盖 empty（空数据）状态的渲染用例 | P0 |
| PC-03 | 每个页面组件至少覆盖 error（加载失败）状态的渲染用例 | P0 |
| PC-04 | 每个页面组件至少覆盖 success（正常数据）状态的渲染用例 | P0 |

### 1.2 路由与权限
| 编号 | 检查项 | 级别 |
|------|--------|------|
| PC-05 | 页面在不同路由参数下的渲染结果有对应用例 | P1 |
| PC-06 | 有权限控制的页面覆盖有权限和无权限两种场景 | P1 |
| PC-07 | 路由守卫（未登录跳转、权限不足跳转）有对应用例 | P1 |

## 8. 测试分层

### 8.1 组件测试层
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TL-01 | 组件测试聚焦渲染结果和用户交互 | P1 |
| TL-02 | 组件测试的 Mock 对象为 API 模块、Router、Store | P1 |
| TL-03 | 组件测试不直接测试业务逻辑（应由单元测试覆盖） | P2 |

### 8.2 单元测试层
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TL-04 | 单元测试聚焦纯逻辑和数据转换 | P1 |
| TL-05 | 单元测试无或最小外部依赖 | P1 |
| TL-06 | composable/store/utils 的测试归类为单元测试 | P2 |

### 8.3 E2E测试层
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TL-07 | E2E 测试聚焦完整用户流程，不测试单个组件细节 | P1 |
| TL-08 | E2E 测试覆盖的是用户视角的操作路径 | P1 |

### 9.3 E2E测试预期
| 编号 | 检查项 | 级别 |
|------|--------|------|

## 10. 用例规范

### 10.1 编号规范
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TC-01 | 用例编号符合 TC-{模块序号}{用例序号} 格式 | P2 |
| TC-02 | 用例编号在全量文档中唯一，无重复编号 | P1 |
| TC-03 | 增量文档中的用例编号与全量文档保持一致 | P1 |
| TC-04 | 编号分类正确：页面 TC-01xx~02xx、通用组件 TC-C0xx、Composables TC-H0xx、Store TC-S0xx、Utils TC-U0xx、E2E TC-E0xx | P2 |

### 10.2 命名规范
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TC-05 | 用例名称清晰表达测试意图（如"商户列表-筛选无结果-显示空状态"） | P2 |
| TC-06 | 用例名称包含测试对象、操作条件和预期结果三要素 | P2 |
| TC-07 | 同一模块下的用例名称不重复 | P1 |

### 10.3 前置条件与操作步骤
| 编号 | 检查项 | 级别 |
|------|--------|------|
| TC-08 | 前置条件描述充分，包含页面状态和数据状态 | P2 |
| TC-09 | 操作步骤具体明确，使用示例值而非模糊描述 | P2 |
| TC-10 | E2E 用例的操作步骤按用户实际操作顺序编写 | P1 |

## 11. 文档规范
