# 产品分类与定义 v1（Product Taxonomy）

---

> **文档目的**：重新定义产品体系，用"每个业务最显著的差异维度"来切分产品。
> **关联文档**：`mp-menu-v3.md`（菜单结构）
> **核心原则**：一条业务线只用**一个最显著的差异维度**往下切业务产品；其余差异降级为"属性/配置"。

---

## 零、三层结构（对外解决方案 vs 内部计费）

为便于后续计费区分，产品体系分为三层：

| 层级        | 名称                         | 面向               | 作用                              | 例子                             |
| ----------- | ---------------------------- | ------------------ | --------------------------------- | -------------------------------- |
| ① 业务分组 | Business Line                | 内部               | 顶层归类                          | 收款/付款/兑换/发卡/支付         |
| ② 业务产品 | Business Product / Solution  | **对外展示** | 解决方案形态，客户看到/选择的东西 | 余额付款、预约付款、委托挂单     |
| ③ 底层产品 | Underlying / Billing Product | **内部定义** | 计费单元（成本/费率按标的差异）   | 付法币/付数币、外汇兑换/数法兑换 |

> **对外展示的是业务产品（解决方案形态）；内部用底层产品来定义与计费。**
> 底层产品主要按**标的（数币 / 法币）**拆分，因为链上与银行渠道的成本结构不同。

---

## 一、两类产品

| 类型                           | 定义                                             | 特征                                                  |
| ------------------------------ | ------------------------------------------------ | ----------------------------------------------------- |
| **基础产品**（账户产品） | 账户/钱包本身 + 其配套能力（充、提、嵌入式兑换） | 自己的钱在自己账户/形态之间动，**无外部对手方** |
| **业务产品**（独立开通） | 与外部对手方产生资金往来，或有独立交互形态       | 跟外部对手方产生资金往来，或独立结算/交互形态         |

## 二、基础产品

| 产品               | 配套能力（属性，不单列产品线）                 |
| ------------------ | ---------------------------------------------- |
| **法币账户** | 充值（含 VA 展示）、提现、账户间划转           |
| **数币钱包** | 充币、提币、账户间划转                         |
| **即时汇兑** | 即时换汇（法币↔法币）、即时承兑（数币↔法币） |

> 充/提是账户的**属性能力**，在账户详情页发起，不单独立菜单入口。
> **即时汇兑作为基础产品**：只要开通收款/付款任一业务产品，**默认开通**即时汇兑（收付流程内的锁汇/换汇/承兑依赖它）。
> **底层产品（计费）**：即时换汇（FX，法币↔法币） / 即时承兑（数法，数币↔法币）。

## 三、业务产品

每条业务线的**最显著差异维度**决定子产品，其余维度降级为属性。

### 总览

| 业务分组             | 最显著差异维度  | 业务产品（② 对外·解决方案）                 | 底层产品（③ 内部·计费）             |
| -------------------- | --------------- | --------------------------------------------- | ------------------------------------- |
| 收款 Collection      | 收款标的形态    | 数币收款 / 法币收款                           | （与业务产品重合：数币收款/法币收款） |
| 付款 Payout          | 付款发起形态    | 余额付款 / 预约付款（本期暂无）               | 付法币 / 付数币                       |
| 兑换 Conversion      | 成交机制 / 时点 | 委托挂单 / 远期（本期暂无）；即期已入基础产品 | 外汇兑换(FX) / 数法兑换(承兑)         |
| 发卡 Cards           | 卡形态          | 虚拟卡 VCC / 实体卡                           | 虚拟卡/实体卡                         |
| 支付 Payment（收单） | 收款触发形态    | Checkout / PayByLink / Subscription           | 法币收单 / 数币收单                  |

### 1. 收款业务（Collection）— 按收款标的形态

> **切分维度说明**：付款/发卡/支付都是「商户主动发起的动作」，按*动作形态*切；
> 收款是「给对方一个账号、被动等入金」的**非交互式收款**，产品本体就是*收款工具*本身，
> 因此按*标的（数币地址 / 法币VA）*切最自洽。各业务"最显著特点"本就不同，不强求统一维度。
>
> 对比：**支付（收单）** 是*交互式收款*——给付款人一个界面（收银台/链接/订阅）；
> **收款（Collection）** 是*非交互式收款*——给对方一个账号，对方从自己银行/钱包打过来。

| 子产品   | 最特殊的点               | 关键要素                 |
| -------- | ------------------------ | ------------------------ |
| 数币收款 | **链上地址**       | 链、收款地址、KYT        |
| 法币收款 | **VA（虚拟账户）** | VA、汇款人匹配、清算入账 |

**属性（不单列产品）**：币种、链、VA 类型、申报/非申报、贸易背景。

### 2. 付款业务（Payout）— 按付款发起形态

> 付款是主动发起动作，最显著的差异是「怎么发起」，而非付的是数币还是法币。

| 子产品   | 定义                           | 关键要素                     |
| -------- | ------------------------------ | ---------------------------- |
| 余额付款 | 用账户已有余额直接付           | 余额充足、直接出金           |
| 预约付款 | 收+锁汇+付 / 换汇+付，一气呵成 | 锁汇时点、组合编排、到期执行 |

**底层产品（③ 内部·计费单元）**：每个业务产品向下按标的拆为付法币/付数币：

| 业务产品（对外） | 底层产品（计费）        |
| ---------------- | ----------------------- |
| 余额付款         | 余额付法币 / 余额付数币 |
| 预约付款         | 预约付法币 / 预约付数币 |

**属性（不单列产品）**：

- 付数币 / 付法币 → 标的属性（决定通道：链上地址 vs 银行；也是底层计费拆分依据）
- 申报 / 非申报（如 CNY vs USD/HKD）→ 合规属性（决定申报流程、贸易背景、购汇额度）
- 单笔 / 批量代付 → 交互属性（收费模式相同，不单列产品）
- 同名（提现）/ 第三方（付款）→ 由 Beneficiary 区分

### 3. 兑换业务（Conversion）— 按成交机制/时点

> **即期（即时汇兑）已上移为基础产品**，随收款/付款默认开通。
> 兑换**业务产品**只保留需要主动开通的高级成交形态。

| 业务产品（对外） | 定义                     |
| ---------------- | ------------------------ |
| 委托挂单         | 限价单，挂目标价等待成交 |
| 远期（Forward）  | 锁定未来某日的汇率       |

**底层产品（③ 内部·计费单元）**：按是否涉及数币拆分，两者成本/费率不同：

| 底层产品         | 形态         | 适用           |
| ---------------- | ------------ | -------------- |
| 外汇兑换（FX）   | 法币 ↔ 法币 | 即期/挂单/远期 |
| 数法兑换（承兑） | 数币 ↔ 法币 | 即期/挂单      |

### 4. 发卡业务（Cards）— 按卡形态

| 子产品     | 最特殊的点           |
| ---------- | -------------------- |
| 虚拟卡 VCC | 即开即用、无物流     |
| 实体卡     | 物流、激活、线下受理 |

**属性（不单列产品）**：充值资金来源（法币/数币）、卡组织。

### 5. 支付业务（Payment / 收单）— 按收款触发形态

| 子产品                      | 定义             |
| --------------------------- | ---------------- |
| Checkout（收银台）          | C 端在结账页付款 |
| PayByLink（支付链/Invoice） | 发链接/开票收款  |
| Subscription（循环支付）    | 周期性扣款       |

**属性（不单列产品）**：币种、结算周期。

> 收单 vs 收款：收单有独立结算周期，结算后才进账户余额；收款实时入账。

## 四、切分维度速查

| 业务 | 业务产品切分维度（②对外）                     | 底层产品（③计费）            | 降级为属性             |
| ---- | ---------------------------------------------- | ----------------------------- | ---------------------- |
| 收款 | 资金标的（数币地址 / 法币VA）—非交互式入金    | 数币收款 / 法币收款（重合）   | 币种、链、申报/非申报  |
| 付款 | 付款发起形态（余额付 / 预约付）                | 付法币 / 付数币               | 申报/非申报、单笔/批量 |
| 兑换 | 成交机制（挂单/远期；即期已入基础产品）        | 外汇兑换(FX) / 数法兑换(承兑) | 币种对                 |
| 发卡 | 卡形态（虚拟/实体）                            | 法币充值卡 / 数币充值卡       | 卡组织                 |
| 支付 | 收款触发形态（收银台/支付链/订阅）—交互式收钱 | 法币收单 / 数币收单           | 币种、结算周期         |

## 五、开通组合参考

| 开通业务产品       | 必然附带                          | 可选叠加         |
| ------------------ | --------------------------------- | ---------------- |
| 收款（Collection） | + 提现 +账户 +**即时汇兑**  | 付款、支付、发卡 |
| 付款（Payout）     | + 充值 + 账户 +**即时汇兑** | 收款、支付、发卡 |
| 支付（Payment）    | + 提现 + 账户                     | 付款、收款、发卡 |
| 发卡（Cards）      | + 充值 + 账户                     | 收款、付款、支付 |
| 兑换（Conversion） | + 账户                            | 可单独开         |
