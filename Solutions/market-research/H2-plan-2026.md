## 〇、一页纸（TL;DR）

1. **双线作战**：基本盘（全球大币种收付 / 下发 / 外汇）保稳定、保现金流；越南新市场走「调研 → 工具化 → 合规化 → 机构化」的产品演进。
2. **三公司协同**：**iPayLinks（法币持牌）+ Blancblock（数币持牌）+ Eurewax（AI 科技 + 风控产品）**——法币持牌 + 数币持牌 + 科技/风控，构成「别人难以复制」的合规承兑+跨境结算组合拳。
3. **战略红线**：**不与越南本地机构抢支付基础设施生意**（外来基因做不了本地基建）。我们只做两件事——**做合规的跨境支付业务**、**帮"想合规但不会合规"的企业合规**。
4. **客户策略**：目标客户是 **PSP / 银行**（计划），现实接触最多的是 **OTC**（有望转化为"租户"）。当前无业务压力大、市场鱼龙混杂、主动/被动筛选都难 → **Q3 以调研与提纯为主，不冲量、先看清**。
5. **H2 节奏**：**Q3 = 调研 + 系统化 + 打样 + 样板店**（沉淀客户画像与合规壁垒，并产出销售展业方案）；**Q4 = 机构化并网 + 跑通 2-3 个合规标杆客户 + 形成可复制解决方案**。
6. **销售联动**：销售 Q3 考核含**签 4 家租户 + 样板店**——我（产品）必须交付"**能把租户签进来的展业方案**"作硬支撑；越南方案以 **OTC 对敲为底层、分全合规/半合规两档准入**（先严后宽保客户质量）。
7. **长期护城河**：不只合规风控，更是**「跨境怎么做」（亚马逊官方 PSP / TikTok 等平台经验）+「风险在哪里」（越南及多地区风控认知）**的双重 know-how。
8. **基本盘并行**：大币种方案持续完善（稳定性、合规、产品补齐），保证 KPI 中"交易量/付费客户达成率"的基本盘不塌。

## 一、战略背景与定位

### 1.1 决心：扎根越南，长期主义

越南市场监管日趋正规化，短期红利与长期合规并存。我们**不做"街头野路子"的一次性套利**，而是**扎根做长期可持续、可审计的合规跨境支付网络**。

### 1.2 三公司协同矩阵（核心差异化）

| 公司                 | 牌照 / 能力                                      | 在越南方案中的角色                                         |
| -------------------- | ------------------------------------------------ | ---------------------------------------------------------- |
| **iPayLinks**  | 法币持牌（全球收付 / 下发 / 外汇 / 收单 / 发卡） | 法币侧落地：香港/全球持牌主体做跨国放款、结汇、合规通道    |
| **Blancblock** | 数币持牌（Web3 / 稳定币承兑）                    | 数币侧落地：USDT↔法币合规承兑、链上 KYT、数字资产合规对冲 |
| **Eurewax**    | AI 科技公司（整套跨境支付产品 + AI 风控）        | 科技中枢：中控系统、风控引擎、对账自动化、Agent 化运营     |

> **组合拳叙事**：**法币持牌 + 数币持牌 + AI 科技/风控**三位一体，把"承兑+跨境结算"从灰色手工搬到合规系统化——这是本地 OTC 和单一牌照机构都难复制的壁垒。

### 1.3 战略红线与定位

- **不抢本地基础设施生意**：支付基础设施天然属于本地持牌方，外来户硬抢做不下去。我们做**赋能者 / 合规网关 / 上游能力方**，与本地形成"合作而非竞争"。
- **只做合规业务**：业务 = 跨境支付。锚定两类机会：
  1. **做合规的跨境支付**（自身持牌 + 系统化承兑）；
  2. **帮"想合规但不知道怎么合规"的企业做合规**（输出合规工具 + 风控 + 凭证）。

### 1.4 长期护城河（我们真正"懂"的，不只是合规风控）

> 合规风控是入场券，不是终局护城河。我们的长期价值是**「跨境怎么做」+「风险在哪里」的双重 know-how**——这两点是本地 OTC / 单一牌照机构补不齐的。

| 护城河                                   | 内涵                                                               | 来源与体现                                                                                                           |
| ---------------------------------------- | ------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| **① 跨境怎么做（业务 know-how）** | 真正懂跨境电商与跨境贸易的资金流、场景与平台规则                   | iPayLinks 11 年跨境经验：**亚马逊官方 PSP**、**TikTok 等平台**的实操经验；懂平台规则、回款链路、商户痛点 |
| **② 风险在哪里（风控认知）**      | 不止"做风控"，更"懂风险"——知道每个市场/通道/客户类型的真实风险点 | **越南及多地区**的冻卡、电诈、资金来源、监管红线认知；Blancblock 链上 KYT；多市场合规经验沉淀                  |

- **对客叙事**：我们不是又一个"换汇通道"，而是**懂你生意（跨境电商/外贸）+ 懂你风险（合规/冻卡/监管）**的长期伙伴。
- **对内意义**：H2 在越南积累的调研与踩坑，沉淀为可迁移到其他市场的"跨境 + 风险"知识资产。

## 二、双线作战：基于 EX PaaS 的能力复用（不另建系统）

> 原则：**EX 已有一套 PaaS 系统，越南方案长在 EX PaaS 上**——以"新市场 / 新租户 / 新通道 / 新币种"的配置 + 轻量增量来落地，**不另起炉灶、不维护影子系统**。基本盘与越南共用同一平台底座，靠**配置隔离 + 权限隔离 + 灰度**保证互不干扰。

| 维度               | 大币种基本盘       | 越南新市场（PaaS 上的增量）                                                                        |
| ------------------ | ------------------ | -------------------------------------------------------------------------------------------------- |
| **承载**     | EX PaaS 现有产品线 | **同一 EX PaaS**，新增越南租户/通道/币种配置                                                 |
| **复用**     | 平台核心能力       | 直接复用 PaaS 的**账户/路由/清结算/跨国调拨/香港法币 Payout/数币合规对冲**                   |
| **增量开发** | 维持与迭代         | 仅做越南**特有的薄层**：本地银行流水对账（RPA/网银通知）、VietQR、承兑中控前端、合规风控规则 |
| **风险**     | 零容忍             | 增量上线走灰度，不动主链路                                                                         |

> **一句话**：越南是 EX PaaS 的**一个新市场实例**，能复用的全复用，只补越南本地化的薄层；既不重复造轮子，也不污染基本盘。

## 三、客户画像与机会地图

### 3.1 客户分层

| 客户类型                                                      | 现状                 | H2 策略                                                              |
| ------------------------------------------------------------- | -------------------- | -------------------------------------------------------------------- |
| **PSP / 银行**                                          | 计划主目标，量未跑出 | 调研 + 建立连接，作为机构化并网的长期目标                            |
| **OTC 承兑商**                                          | 现实接触最多         | **有望转化为"租户"**；先提纯、再系统化，**严筛资金来源** |
| **正规跨境企业**（出口电商 / 大外贸 / 出海游戏/货代等） | 种子机会             | Q4 锁定 2-3 个合规标杆                                               |

### 3.2 当前真实困境（诚实面对）

- **无业务压力大**：刚起步、急需开张，但**不能为冲量牺牲合规底线**。
- **市场鱼龙混杂**：OTC 客户想资金出入境，成分复杂。
- **筛选困难**：主动筛（看不清友商/客户底细）、被动筛（来者成分不明）都难 → **唯一解法是"调研驱动 + 风控前置"**，Q3 把"看清楚"作为首要交付物。

### 3.3 调研清单（Q3 重点）

- **友商调研**：本地派、外地派、**与我们同源的中国/香港机构**——他们做什么、怎么做、防冻卡/KYC 死穴、通道成本、**交易量**。
- **客户调研**：30+ 现有承兑大客 + 本地二三线 PSP，探明合规真实痛点。

### 5.1 7 月｜商业系统化

- **产品**：机构级承兑中控系统 1.0（Eurewax 主导研发）
- **Scope**：
  - 交易流水结构化录单：实时盘面 + 利润点(Spread) 自动报价，生成带**审计日志**的结构化订单。
  - 自动化撮合对账：RPA / 网银通知回调，自动读取越南本地核心银行（MB Bank、Techcombank 等）到账流水，实现"VND 到账 → 匹配订单 → 触发放款"半自动化。
- **交付**：业务全面数字化，沉淀首批合规审计数据集。

### 5.2 8 月｜合规风控化

- **产品**：中越跨境防冻卡与合规风控组件 1.0
- **Scope（基于前线调研动态设计）**：
  - **友商调研**：PM 制定卧底/测评模板，由 **BD/运营** 伪装客户走通 3-5 家主要友商（港派/国内派/本地派）的结汇全流程，摸清防冻卡、KYC、通道成本。
  - **客户田野调研**：PM 制定访谈提纲与画像卡模板，由 **BD 主导**拜访 30+ 承兑大客 + 本地二三线 PSP，探明"合规凭证、跨国主体隔离"痛点；PM 参与关键客户回访并汇总画像库。
  - **产品落地**：静态黑名单库、同名卡校验（付款户名 = KYC 实名）、结合 Blancblock 链上 KYT 对数字资产合规初筛。
- **交付**：《友商跨境通道技术架构与客户真实痛点分析白皮书》。

### 5.3 9 月｜机构化打样 + 样板店

- **产品**：中越合规承兑网关 MVP 1.0（作为大币种方案的新币种通道接入）
- **Scope**：
  - 多持牌主体联动：VND 资金过风控后，联动 iPayLinks 香港法币主体(MSO) / Blancblock 合规钱包，完成跨国放款与对冲。
  - 机构客户合规套件：为 2-3 个种子机构客户提供系统级对账单、可审计电子凭证导出。
  - **样板店落地**：选 1-2 家提纯后的优质租户，用全套系统（中控 + 风控 + 凭证）跑通完整闭环，作为**可复制、可演示的样板店**，供销售对外展业背书。
- **交付**：淘汰高风险散客，VND 通道平稳并入大币种总盘；**1-2 个可对外展示的样板店**。

#### OTC（对敲）合规 / 半合规打法

> 越南整体方案的现实形态：**OTC 对敲为底层，分"全合规 / 半合规"两档准入**——用"准入资料"做客户质量闸门，先严后宽。

| 档位             | 准入要求                                                                                     | 适用客户             | 后续                            |
| ---------------- | -------------------------------------------------------------------------------------------- | -------------------- | ------------------------------- |
| **全合规** | 完整 KYC/KYB + 资金来源证明 + 合规凭证                                                       | 正规跨境企业、机构   | 直接进机构通道                  |
| **半合规** | **前期要求一定资料**（实名、主体、基础来源）做质量筛选；**跑通后视表现适当放宽** | 优质 OTC（潜在租户） | 表现良好 → 升级全合规 / 转租户 |

- **核心逻辑**：用"前置资料门槛"过滤掉鱼龙混杂的散客，**保住客户质量**；对通过初筛、表现良好的客户**逐步放宽**，提升体验与黏性。
- **筛选机制**：主动筛（调研+画像）+ 被动筛（系统风控前置）双管，解决"主动/被动筛选都难"的现实困境。

#### Q3 销售客户画像 → 对应方案

| 客户画像                       | 准入档        | 对应方案                                   |
| ------------------------------ | ------------- | ------------------------------------------ |
| **优质 OTC（潜在租户）** | 半合规起步    | 承兑中控 + 风控组件 + 半合规准入 → 转租户 |
| **正规跨境企业**         | 全合规        | 合规承兑网关 + 凭证套件                    |
| **本地二三线 PSP**       | 全合规 / 合作 | 能力赋能 + 主体隔离（不抢基建，做上游）    |

### 5.6 Q3 终极交付

| **11 月** | 标杆客户签约 + 上量          | 签下 2-3 个合规机构客户（出口电商/大外贸/出海游戏）；交易量爬坡                   |
| **12 月** | 解决方案产品化 + 复制        | 沉淀"客户画像 → 标准解决方案"模板；冲击 1 个项目月交易量越 300 万 USD 竞争优势线 |

**Q4 形成的"有效解决方案"（对应你的需求②）**：

- 按客户画像分型（OTC-潜在租户 / 正规跨境企业 / 本地 PSP 合作），各配**标准化方案包**：账户结构 + 通道路由 + 合规凭证 + 风控规则。
- 输出"帮企业合规"的产品化套件（KYC/KYT + 凭证 + 主体隔离），把"合规咨询"变成"合规产品"。

### 7.2 进人时机（触发条件，二选一即启动）

| 触发信号 | 含义 | 动作 |
| --- | --- | --- |
| **① 交易量进来** | 越南/新业务出现稳定交易量，方案需求与对账/运营压力上升 | 启动招聘，让 Solution PM 专注方案落地与上量承接 |

> **判断原则**：**不为招而招**——在"有量"或"有新画像"的真实需求出现前不强行进人；一旦触发，快速补位，避免成为 Q4 翻倍目标的瓶颈。**具体时机待定，由上述信号驱动 + 与 CEO 对齐。**
