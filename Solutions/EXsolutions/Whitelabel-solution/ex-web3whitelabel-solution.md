## 一、方案概述

EX 提供**白标门户（White Label Portal）**方案 —— 无需开发，即开即用。

Web3 白标方案聚焦于**承兑产品、发卡产品和 Crypto Checkout 产品**三大产品板块，面向加密货币交易所、数字资产服务商、Web3 支付平台等客户。

EX 根据客户开通的产品线，由运营团队完成配置后，直接交付两个门户：

| 门户               | 英文                  | 使用方                 | 说明                                                                   |
| ------------------ | --------------------- | ---------------------- | ---------------------------------------------------------------------- |
| **租户门户** | Tenant Portal（TP）   | 客户（机构）           | 客户自己使用的运营管理后台，管理商户、查看交易、配置产品等             |
| **商户门户** | Merchant Portal（MP） | 客户的终端用户（商户） | 客户交付给其终端商户使用的操作门户，商户自助完成承兑、发卡、收单等操作 |

```
┌─────────────────────────────────────────────────────────────┐
│                       EX 平台                                │
│                                                              │
│  ┌──────────────────┐    ┌──────────────────┐               │
│  │   运营配置完成     │ →  │   交付 TP + MP    │               │
│  │ (产品开通+品牌配置)│    │  (即开即用)       │               │
│  └──────────────────┘    └──────────────────┘               │
│                                                              │
└──────────────────────┬──────────────────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼                         ▼
┌──────────────────┐     ┌──────────────────┐
│  TP（租户门户）    │     │  MP（商户门户）    │
│                   │     │                   │
│  客户（机构）使用   │     │  终端商户使用      │
│  管理商户/交易/配置 │     │  承兑/发卡/收单    │
└──────────────────┘     └──────────────────┘
```

> **核心价值**：客户无需任何开发投入，开通产品后即可获得完整的运营管理后台（TP）和面向终端商户的操作门户（MP），快速启动 Web3 业务。

## 二、谁适合 Web3 白标方案

| 客户类型                   | 场景                                       | 推荐理由                              |
| -------------------------- | ------------------------------------------ | ------------------------------------- |
| **加密货币交易所**   | 为用户提供法币出入金、数币充提             | 快速上线承兑业务，无需自建合规体系    |
| **数字资产服务商**   | 代理数币承兑、发卡服务给终端商户           | 无需开发，TP 管理商户，MP 交付商户    |
| **Web3 支付平台**    | 商户接受数币支付                           | 收银台即开即用，商户直接用 MP 操作    |
| **初创加密 Fintech** | 刚起步，优先验证 Web3 业务模型             | 零开发成本，先跑通业务再考虑 API 集成 |
| **VCC 发行平台**   | 批量发卡、资金管理                         | 发卡全流程门户化，即开即用            |
| **持牌金融机构**     | 拓展 Web3 新业务线（承兑、发卡、Crypto Checkout） | TP 管理，快速扩展产品线               |

## 三、产品矩阵

Web3 白标方案覆盖以下产品板块：

| 产品板块           | 产品                      | 能力                                                                         |
| ------------------ | ------------------------- | ---------------------------------------------------------------------------- |
| **承兑产品** | 法币账户（Fiat Account）  | 充值（Deposit）、提现（Withdrawal）、余额查询                                |
|                    | 数币钱包（Crypto Wallet） | 充币（Crypto Deposit）、提币（Crypto Withdrawal）、充币退款（Crypto Refund） |
|                    | 买入数币（Buy Crypto）    | 买入数币、法转数（Fiat to Crypto）                                           |
|                    | 卖出数币（Sell Crypto）   | 卖出数币、数转法（Crypto to Fiat）                                           |
| **收单产品** | Crypto Checkout           | 收单交易（Acq Authorization）、收单退款（Acq Refund）                        |
| **发卡产品** | VCC（Card Issuing）       | 开卡、卡管理、资金转入、销卡、资金转出、卡消费、3DS验证等                    |

## 四、白标门户能力矩阵

根据客户开通的产品线，TP 和 MP 自动展示对应的功能模块：

### 4.1 TP（租户门户）— 客户使用

| 功能模块                  | 说明                                                                  | 备注                                                                                                                   |
| ------------------------- | --------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **商户管理**        | 注册商户、商户基本信息提交与跟踪、商户状态管理                        |                                                                                                                        |
| 商户开通产品查询          | 查看商户开通的产品；<br />商户增开产品预计6月上线，目前只能开单一产品 | <br />如果客户开通了一个产品，再开通其他产品,<br />需要找运营团队支持                                                  |
| 商户收费管理              | 管理商户产品收费                                                      | 对客的标准报价，需要发邮件给运营；<br />如果商户要单独报价，可以进行单独收费配置；<br />需要有一个人配置，一个人审核； |
| 商户汇率报价（comingsoon) | 管理商户的承兑汇率加点                                                | 同样，默认统一的加点配置，可以发运营；<br />商户特殊配置在这里配置                                                     |
| 商户交易管理              | 查看所有商户的承兑交易、发卡交易、收单交易                            | 按产品线展示                                                                                                           |
| 工单审核                  | 费率，汇率的审核人员进行审核<br />审核通过后才能生效                  |                                                                                                                        |
| 我的分佣                  | 您获取的交易分佣账单和结算单                                          | 所有产品                                                                                                               |
| 增值功能：销售管理        | 给每个销售生成一个拓客链接                                            |                                                                                                                        |
| 增值功能：代理商管理      | 可以发展下一级的代理商；                                              | 目前只有交易可以返点，汇率的返点暂不支持<br />操作手册可以跟运营获取                                                   |
| 账号权限                  | 可以给其他人分配账号                                                  | 目前权限只能控制到页面，不能控制到操作                                                                                 |

### 4.2 MP（商户门户）— 终端商户使用

| 功能模块                | 说明                                                                   | 备注                          |
| ----------------------- | ---------------------------------------------------------------------- | ----------------------------- |
| 商户注册+开通产品       | 商户提交开通承兑/发卡/收单所需的资料<br />包括实名认证的信息和业务信息 |                               |
| **数币钱包**      | 查看数币钱包余额、充值地址、充币/提币记录                              | 支持多链多币种                |
| **法币账户**      | 查看法币账户余额、充值/提现记录                                        | 承兑产品的法币账户            |
| **买入/卖出数币** | 买入数币、卖出数币、法币转数币、数币转法币                             | 实时报价，报价有效期内确认    |
| **Crypto Checkout** | 创建收款订单、查看收单交易记录                                         | 支持 API 直连和收银台两种模式 |
| **发卡   产品** | 申请 VCC、查看卡信息、资金转入/转出、查看卡消费记录等                  | 支持多卡管理                  |

## 五、Web2 vs Web3 白标方案对比

| 维度                  | Web2 白标方案                         | Web3 白标方案                              |
| --------------------- | ------------------------------------- | ------------------------------------------ |
| **核心产品**    | 收付款产品（收款/付款/外汇/账户服务） | 承兑产品（数币充提/买卖）+ 发卡 + Crypto Checkout |
| **目标客户**    | 跨境电商、外贸 B2B、代理机构          | 加密交易所、Web3 支付、数字资产服务商      |
| **MP 核心操作** | VA 收款、法币付款、换汇               | 数币充提、买卖数币、发卡、数币收款         |
| **TP 管理重点** | 收付款交易、结汇材料、贸易背景        | 承兑交易、发卡管理、收单交易               |
| **代理商/分佣** | ✅ 相同                               | ✅ 相同                                    |
| **交付流程**    | ✅ 相同                               | ✅ 相同                                    |
| **品牌配置**    | ✅ 相同                               | ✅ 相同                                    |

> 如需同时覆盖 Web2 + Web3 场景，可组合开通两个方案的产品线，TP/MP 自动展示所有已开通产品。

## 六、交付流程

```
├── 1. 线下签约
│     └── 跟客户经理签订合同
│
├── 2. EX 运营配置（1-2 个工作日）
│     ├── 创建租户（Tenant）
│     ├── 配置品牌信息（Logo、域名、邮件模板）
│     └── 配置商户的标准报价/汇率
│
├── 3. 交付 TP（租户门户）
│     └── 客户获得 TP 管理后台访问权限
│     └── 客户可开始在 TP 中注册和管理商户
│
├── 4. 交付 MP（商户门户）
│     └── 商户通过租户的链接注册
│     └── 完成 KYC/KYB → 产品开通 → 开始使用
│
└── 5. 上线运营
      └── 客户通过 TP 管理日常运营
      └── 终端商户通过 MP 自助操作
```

### 交付时间线

| 阶段           | 内容                             | 预计耗时                               |
| -------------- | -------------------------------- | -------------------------------------- |
| 线下签约       | 签订合同                         | 1-2 天                                 |
| 运营配置       | 租户创建、费率配置，域名配置等   | 1-2 个工作日                           |
| 交付           | 交付MP 和TP                      | 1天 （含在上面的时间，配置完成就交付） |
| 生产验证       | 商户注册 → KYC → 产品开通      | 3-5 个工作日（含审核）                 |
| **总计** | **从签约到首批商户可操作** | 2-10工作日（看生产验证的情况）         |
