5Prerequisites — Environment Setup

  1. Activate Sandbox — Contact technical support for test account (Account No), test domain
  2. Get Credentials — APP ID, Platform Public Key, AES Key
  3. Generate Keys — RSA key pair (SHA256withRSA, 2048-bit) → Upload client public key to management platform
  4. Configure Webhook — Callback URL (HTTPS, supports POST)
  5. Integration Testing — Complete signing verification + AES encryption/decryption testing
ℹ️ After environment setup, proceed to business flow integration.Sandbox Environment Parameters →

6Business Flow Details

0 Merchant Registration & Product Activation (Shared by all products)
Register Merchant→ Get MID→ Submit Customer Info→ Webhook: KYC Result→ Apply for Product→ Webhook: Product Approval
Key Notes
• Submit customer info before product activation
• Product activation may require additional documents
• Attachments: call upload file API first to get URL
• Merchant info templates provided by technical support
Webhook Events
KYC/KYB Review Result
Product Approval approved/rejected/RFI
⚠️ RFI must be responded to promptly, timeout may cause rejection
1 Fiat CollectionE-commerce seller collection, B2B trade
Apply for VA→ Get VA Account→ Remitter sends funds→ Webhook: Incoming→ Confirm receipt
ℹ️ VA collection is passive credit — no active transaction API needed. System automatically pushes Webhook upon arrival. Based on SP requirements, store binding may be needed when applying for VA.
2 Fiat PayoutSupplier payment, cross-border remittance

Regular Fiat Payout

Add Payee→ Webhook: Review→ Initiate Payout→ Webhook: Result→ Query Status

Special Case: Settlement to Mainland China

⚠️ Due to foreign exchange control requirements, funds entering mainland China must provide trade background documents (orders, logistics receipts, contracts, etc.) and pass compliance review before generating RMB settlement quota.

Generate quota (CNY)

Merchant initiates payout
2Method 2: Store Authorization
Merchant authorizes store

↓
Auto-generates quota
↓
Merchant initiates payout
ℹ️ Both methods are automatically matched by EX based on the merchant's SP. Merchants don't need to understand underlying differences.
3 FX ConversionMulti-currency conversion, rate locking
Determine Currency Pair→ Query Rate→ Request Quote & Lock→ quote_id + rate→ Execute FX→ Query Status
⚠️ Quotes have limited validity. Expired quotes require re-requesting.
4 Crypto Deposit & WithdrawalExchange deposit/withdrawal, fund consolidation
Deposit
Get On-chain Address→User sends crypto→Webhook: Incoming
Withdrawal
Add Payee→Initiate Withdrawal→Webhook: Result
5 Buy/Sell Crypto / OnRamp / OffRampFiat to crypto, crypto to fiat
Buy Crypto / OnRamp (Fiat → Crypto)
Get Quote→Confirm Buy→Webhook
Sell Crypto / OffRamp (Crypto → Fiat)
Get Quote→Confirm Sell→Webhook
⚠️ All quotes have expiration times. Execute before expiry or request a new quote.
6 CheckoutMerchant accepts digital currency payments
Method 1: API Direct
Create Order→Return Payment Address→Payer sends crypto→Webhook
You display payment address and amount to the payer
Method 2: Hosted Checkout
Create Order→Redirect to Checkout→Payer completes payment→Webhook
EX hosts the complete payment page, suitable when not building your own UI
