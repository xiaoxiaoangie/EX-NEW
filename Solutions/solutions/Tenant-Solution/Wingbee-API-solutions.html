方案一：大账户方案（智朗银行）

二、方案一概述

2.1 核心思路

Wingbee 在智朗银行拥有自己的法币账户，直接归集客户资金。Wingbee 作为直客入网 BB，以自身身份（MID）发起 OnRamp / OffRamp 承兑交易。客户不需要在 BB 入网，所有操作均由 Wingbee 以自身商户号完成。

2.2 架构关系

客户终端用户
→ 入网/交易 →
Wingbee平台
→ API 对接 →
EX技术服务
↓ 银行转账
↓ 系统支撑
贸易公司BB代收付主体 · 智朗银行
BB持牌通道

系统上的身份关系：

  • Wingbee = BB 的直客商户（通过 EX 入网），拥有自己的 MID
  • 贸易公司 = BB 的代收付主体，拥有智朗银行账户，用于法币收付
  • Wingbee 通过智朗银行与贸易公司进行行内转账

资金通道：

  • BB 本身没有智朗银行账户，法币收付通过贸易公司（代收付主体）的智朗银行账户完成
  • Wingbee 有自己的智朗银行账户
  • Wingbee ↔ 贸易公司之间通过智朗银行转账（行内转账）

2.3 核心原则

OnRamp 先收后做

Wingbee 必须先收到客户法币，转账给贸易公司并确认到账后，才发起 OnRamp。绝不垫资

OffRamp 头寸自管

Wingbee 自行管理法币头寸，可选择先垫付客户再等结算，但风险自担

双方不互垫头寸

BB 不为 Wingbee 垫资，Wingbee 不为 BB 垫资。各自管理各自的资金池

2.4 资金流向总览

OnRamp（法币 → 加密货币）

客户法币 → Wingbee 智朗账户 → Wingbee 转账给贸易公司智朗账户 → Wingbee 发起 OnRamp → 加密货币入 Wingbee 账户

OffRamp（加密货币 → 法币）

Wingbee 发起 OffRamp（收款人=Wingbee智朗） → BB 执行承兑 → 法币到 Wingbee 智朗银行账户 → Wingbee 自行付款给客户

三、方案一前置流程

3.1 Wingbee 自身准备

├── 1. Wingbee 在智朗银行开设法币账户 │ └── 用于接收客户法币（OnRamp）和向客户付款（OffRamp） │ ├── 2. Wingbee 与 EX 签约 │ └── 作为租户入网 EX 平台 │ └── 开通 OnRamp / OffRamp 产品 │ ├── 3. Wingbee 作为直客入网 BB（如果已经入网，我们协助迁移） │ └── Wingbee 通过 EX 注册为 BB 的直客商户，完成 KYC/KYB │ └── 用于以 Wingbee 自身商户号发起 OnRamp / OffRamp 承兑交易 │ └── 客户不需要在 BB 入网，所有操作均以 Wingbee MID 进行 │ └── 4. 技术对接 └── 获取 Sandbox 环境 → 配置 APP ID / 公钥 / AES Key / Webhook └── 完成签名验签 + AES 加解密联调

3.2 贸易公司准备（BB 侧）

├── 1. 贸易公司已在智朗银行开设法币账户 │ └── 作为 BB 的代收付主体，接收 Wingbee 转入的法币（OnRamp） │ └── 2. BB 侧配置 └── 贸易公司由 BB 控制，作为法币收付通道

3.3 Wingbee 配置收款账户（OffRamp 用）

├── 1. Wingbee 在 EX 添加自己的智朗银行账户作为收款人（beneficiary） │ └── 收款账户 = Wingbee 的智朗银行账号 │ └── OffRamp 完成后，法币直接到达 Wingbee 智朗银行账户 │ └── 2. 客户提现账户由 Wingbee 自行管理 └── 客户在 Wingbee 前端添加提现银行账户 └── Wingbee 负责维护客户收款信息，不经过 EX

四、方案一 OnRamp 流程（法币 → 加密货币）

核心原则：Wingbee 必须先收到客户法币，转给贸易公司确认到账后，才发起 OnRamp。

Wingbee 银行转账给贸易公司 │ └── Wingbee 从智朗银行账户转账至贸易公司智朗银行账户（行内转账） │ ├── 5. Wingbee 发起 OnRamp │ └── 调用 EX API，以 Wingbee MID 发起 OnRamp 请求 │ └── EX 返回：OnRamp 订单信息（含应付法币金额、汇率、订单号） │ ├── 6. BB 执行 OnRamp │ └── BB 执行承兑（法币 → 加密货币） │ └── 加密货币入账到 Wingbee 的加密货币账户 │ └── 7. 交易完成 └── Webhook: OnRamp 交易结果通知 └── Wingbee 通知客户充值完成

六、方案一补充说明

6.1 贸易公司的角色

贸易公司 = BB 的代收付主体。BB 本身没有智朗银行账户，法币收付通过贸易公司完成。贸易公司由 BB 控制，拥有智朗银行账户。
场景资金流向说明
OnRampWingbee 智朗 → 贸易公司智朗（行内转账）Wingbee 先转法币，BB 收到后执行承兑
OffRampBB 承兑后 → 法币到 Wingbee 智朗（直接到账）Wingbee 配置自己的智朗为收款账户

6.2 头寸管理

1. OnRamp：先收后做

Wingbee 必须先收到客户法币，转给贸易公司确认到账后才能发起 OnRamp。不允许未收款就发起交易

2. OffRamp：头寸自管

Wingbee 自行管理智朗银行法币头寸。OffRamp 法币直接到 Wingbee 智朗，再自行付款给客户。可先垫付客户，再等 OffRamp 到账（风险自担）

3. 双方不互垫头寸

BB 不为 Wingbee 垫资，Wingbee 不为 BB 垫资。各管各的资金池

6.3 OffRamp 头寸策略

策略做法风险客户体验
保守策略（推荐） 等 OffRamp 法币到 Wingbee 智朗后再付款给客户 零风险 客户等待时间较长
激进策略 Wingbee 预先在智朗银行备足法币头寸，先垫付客户，再等 OffRamp 法币到账 汇率波动风险、资金占用 客户体验好，到账快
建议：初期采用保守策略，业务稳定后根据资金情况考虑激进策略。

方案二：VA 方案

七、方案二概述

7.1 核心思路

Wingbee 将客户信息推送到 SP，SP 为每个客户独立开立 VA。客户资金直接进入客户自己的 VA，Wingbee 不归集客户资金，Wingbee 自身也不能开 VA。

7.2 架构关系

客户终端用户
→ 注册/交易 →
Wingbee平台
→ API 对接 →
EX技术编排
↓ 法币直接入客户 VA
↓ 编排
SP（持牌服务商）客户A VA · 客户B VA · 客户N VA（独立账户）
BB（承兑通道）── 执行 OnRamp / OffRamp

7.3 与方案一的关键区别

维度方案一（大账户）方案二（VA）
客户资金流客户 → Wingbee 智朗账户（归集）客户 → 客户自己的 VA（不归集）
交易主体Wingbee 以自身 MID 发起交易以客户 MID 发起交易
Wingbee 角色资金归集方 + 交易发起方纯平台方，推送客户信息、编排交易
合规主体Wingbee 对客户资金负责SP 对客户资金负责（资金在 SP 的 VA 内）
贸易公司需要（作为 BB 的代收付主体）不需要
OffRamp 收款Wingbee 智朗银行账户（直接到账）客户 SP 法币账户

7.4 核心原则

Wingbee 不碰钱

客户资金直接进入 SP 开立的 VA，Wingbee 不归集、不中转

Wingbee 不开 VA

Wingbee 自身不在 SP 开立 VA，只推送客户信息

客户独立账户

每个客户在 SP 拥有独立 VA，资金隔离

Wingbee 编排交易

Wingbee 通过 EX API 代客户发起 OnRamp/OffRamp 交易

├── 1. Wingbee 与 EX 签约 │ └── 作为租户入网 EX 平台 │ └── 开通 OnRamp / OffRamp 产品 │ └── 2. 技术对接 └── 获取 Sandbox 环境 → 配置 APP ID / 公钥 / AES Key / Webhook └── 完成签名验签 + AES 加解密联调

十三、注意事项

方案一专属

  1. OnRamp 必须先收后做 — Wingbee 必须先收到客户法币，银行转账给贸易公司确认到账后，才能发起 OnRamp
  2. OffRamp 收款账户 — Wingbee 必须在 EX 配置自己的智朗银行账户作为 OffRamp 收款人（beneficiary），法币直接到 Wingbee 智朗
  3. 头寸监控 — 采用激进策略时需实时监控智朗账户余额，避免付款失败
  4. 对账 — Wingbee 应定期核对：智朗银行流水 ↔ EX 系统交易记录

方案二专属

  1. Wingbee 不能开 VA — Wingbee 自身不能在 SP 开立 VA，只能为客户开立
  2. Wingbee 不能归集客户资金 — 客户资金直接进入客户自己的 VA，不经过 Wingbee
  3. 客户信息必须推送到 SP — 客户 KYC/KYB 由 SP 审核，审核通过才能开 VA

两方案通用

  1. 汇率有时效性 — OnRamp/OffRamp 的报价有过期时间，过期需重新获取
  2. RFI 及时响应 — 审核过程中可能要求补充材料，超时可能导致审核失败
