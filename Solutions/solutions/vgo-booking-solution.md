> **文档类型**: 客户解决方案

> **版本**: v1.0

---

## 一、客户概况

**VGO Booking** 是一家综合 OTA（在线旅游代理）平台，提供 6 大旅行服务：

| 服务               | 说明                                     |
| ------------------ | ---------------------------------------- |
| Flight Booking     | 单程/往返/多程机票，含行李/选座/餐食附加 |
| Hotel Booking      | 酒店预订，支持灵活取消，24/7 服务        |
| Package Booking    | 度假/医疗/娱乐/活动套餐                  |
| Traveller SIM Card | eSIM / 实体 SIM 卡数据套餐               |
| Transport Booking  | 机场接送、网约车、配送服务               |
| Rental Services    | 租车/租自行车，含保险选项                |

---

## 二、VGO Booking 业务资金流

VGO Booking 作为 OTA 平台，核心资金流：

```
C 端用户（旅客）──付款──→ VGO Booking ──付款──→ 供应商（航司/酒店/租车公司等）
```

**VGO 的支付需求：**

| 环节       | 需求                                           | 方向     |
| ---------- | ---------------------------------------------- | -------- |
| C 端收款   | 接受旅客多种支付方式（卡支付、本地钱包、数币） | 资金流入 |
| 资金管理   | 多币种持有、法币↔数币兑换                     | 内部管理 |
| 供应商付款 | 向航司/酒店/GDS 等供应商付款（VCC / 法币转账） | 资金流出 |

## 四、EX 能力匹配分析

### 4.1 能力矩阵

| 需求                       | 能否提供 | 提供方   | 产品                         | 说明                                                    |
| -------------------------- | -------- | -------- | ---------------------------- | ------------------------------------------------------- |
| 数币收单（C端旅客付数币）  | ✅ EX    | EX       | 聚合收银（Crypto Checkout）  | API 收单 + 收银台模式，C 端旅客用数币支付旅行订单       |
| 承兑 OnRamp（法币→数币）  | ✅ EX    | EX       | OnRamp                       | VGO 自身法币兑数币，⚠️ 仅限 VGO 自身使用，不面向 C 端 |
| 承兑 OffRamp（数币→法币） | ✅ EX    | EX       | OffRamp                      | VGO 数币收入兑换为法币，⚠️ 仅限 VGO 自身使用          |
| VCC 发卡（供应商付款）     | ✅ EX    | EX       | VCC 发卡（Card Issuing）     | VGO 申请虚拟卡，用于向航司/酒店/GDS 等供应商结算        |
| 卡收单（C端信用卡/借记卡） | ❌       | 收单机构 | Stripe / Adyen / 本地 PSP 等 | EX 目前不提供法币卡收单能力，VGO 需自行对接收单机构     |
| 本地钱包支付（GCash等）    | ❌       | 收单机构 | 本地 PSP                     | EX 不提供本地电子钱包收单，VGO 需自行对接本地支付渠道   |
| 银行转账收款               | ❌       | 收单机构 | 本地 PSP / 银行直连          | EX 不提供法币收单能力                                   |
| 法币付款（供应商银行转账） | ✅ EXEX  |          | 付款                         | 如供应商接受银行转账，需评估是否通过 EX 法币付款产品线  |

## 五、推荐接入方案

### 5.1 接入产品

| 优先级 | 产品                        | 产品代码          | 用途                                     |
| ------ | --------------------------- | ----------------- | ---------------------------------------- |
| P0     | VCC 发卡                    | `CARD_ISSUING`  | 向航司/酒店/GDS 供应商付款，OTA 核心需求 |
| P0     | 聚合收银（Crypto Checkout） | `CRYPTO_WALLET` | 接受 C 端旅客数币支付                    |
| P1     | OffRamp（卖出数币）         | `FIAT_OFFRAMP`  | 数币收入兑换为法币，回笼资金             |
| P1     | OnRamp（买入数币）          | `FIAT_ONRAMP`   | 法币兑数币，用于数币钱包资金补充         |

### 5.2 典型业务场景

#### 场景 1：旅客用数币支付机票

```
旅客在 VGO 预订机票 → 选择数币支付 → VGO 调 EX 聚合收银创建订单
→ 旅客转币到指定地址 → EX 确认到账 → Webhook 通知 VGO → VGO 出票
```

#### 场景 2：VGO 用 VCC 向航司付款

```
VGO 在 EX 申请虚拟卡 → 充值资金到卡 → 用 VCC 卡号在 GDS/航司系统付款
→ 航司扣款成功 → EX 推送交易通知 → VGO 记录供应商结算完成
```

#### 场景 3：VGO 将数币收入兑换为法币

```
VGO 数币钱包累积一定 USDT → 调 EX OffRamp 获取报价
→ 确认卖出 → 数币兑换为法币 → 法币入账 VGO 账户
```

## 六、集成路线图

| 阶段           | 内容                                              | 预计耗时           |
| -------------- | ------------------------------------------------- | ------------------ |
| Phase 0        | 环境准备：Sandbox、密钥、Webhook 配置             | 1-2 天             |
| Phase 1        | 前置流程：商户注册 → 提交客户信息 → 产品开通    | 3-5 天             |
| Phase 2a       | VCC 发卡：充值 + 发卡 + 卡管理 + 交易通知         | 7-10 天            |
| Phase 2b       | 聚合收银：创建订单 + 支付结果回调（可与 2a 并行） | 3-5 天             |
| Phase 2c       | 承兑服务：OnRamp + OffRamp（可与 2a 并行）        | 3-5 天             |
| Phase 3        | 联调测试：端到端流程验证                          | 5-7 天             |
| Phase 4        | 生产上线                                          | 2-3 天             |
| **总计** |                                                   | **25-35 天** |

## 七、不在 EX 范围内的能力

以下能力需 VGO 自行对接第三方：

| 能力         | 说明                                 | 建议对接方              |
| ------------ | ------------------------------------ | ----------------------- |
| 法币卡收单   | C 端信用卡/借记卡支付                | Stripe / Adyen / dLocal |
| 本地钱包收单 | GCash / GrabPay / OVO 等本地电子钱包 | 本地 PSP / PayMongo 等  |
| 银行转账收单 | 本地银行转账                         | 本地 PSP / 银行直连     |

> EX 目前不提供法币收单能力。VGO 的 C 端支付渠道（卡支付、本地钱包）需通过第三方收单机构完成，EX 仅覆盖数币收单部分。
