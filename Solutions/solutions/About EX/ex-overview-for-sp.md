## 二、平台定位 — PPaaS

EurewaX 是跨境支付平台即服务（**Payment Platform as a Service, PPaaS**）。平台不是单一的技术中间件，而是将持牌机构的支付业务能力、合规审核与运营 know-how 体系化、产品化，构建一套可复用、可扩展的跨境支付基础设施。

### 2.1 平台五大能力层

```
┌──────────────────────────────────────────────────────────┐
│  ⑤ 生态网络层 │ 双边市场 · 多渠道协同接入                    │
├──────────────────────────────────────────────────────────┤
│  ④ 商户运营层 │ 白标门户 · 统一 API · 商户管理 · 报表对账      │
├──────────────────────────────────────────────────────────┤
│  ③ 清结算层  │ 资金归集 · 跨账户调拨 · 自动对账 · 差错处理     │
├──────────────────────────────────────────────────────────┤
│  ② 合规风控层 │ KYC/KYB · AML · AI 风控 · 监管报告              │
├──────────────────────────────────────────────────────────┤
│  ① 业务产品层 │ 收款 · 付款 · 换汇 · 钱包 · 收单 · 发卡 · Ramp │
└──────────────────────────────────────────────────────────┘
```

| 层级                    | 核心能力                                             | 价值定位                               |
| ----------------------- | ---------------------------------------------------- | -------------------------------------- |
| **① 业务产品层** | 收款 / 付款 / 换汇 / 钱包 / 收单 / 发卡 / On-OffRamp | 平台业务底座，覆盖跨境支付全场景       |
| **② 合规风控层** | KYC/KYB 进件、AI 风控决策、AML 监控、监管报告        | 合规即服务，将持牌机构的合规能力产品化 |
| **③ 清结算层**   | 多渠道统一清算、资金归集与调拨、自动对账、差错处理   | 屏蔽多渠道资金链路复杂度               |
| **④ 商户运营层** | 白标商户门户、商户管理后台、报表对账、API 聚合       | 支撑客户对终端商户提供完整支付体验     |
| **⑤ 生态网络层** | 双边市场、多渠道协同接入                             | 通过生态协同放大业务价值               |

> **核心主张**：业务能力是地基，合规风控是核心。EurewaX 将合规、清结算、商户运营、生态网络四层能力叠加于业务产品之上，形成完整的跨境支付平台。

## 三、产品能力全景

EurewaX 通过 **统一 API 接口 + 白标门户** 两种交付形态，覆盖跨境支付完整能力矩阵：

| 产品线                 | 核心能力                                   |
| ---------------------- | ------------------------------------------ |
| **收款**         | 多币种 VA 虚拟账户、全球法币收款、数币收款 |
| **付款**         | 法币批量付款、数币转账、POBO 代付          |
| **换汇**         | 多币种法币换汇、CNY 结汇                   |
| **钱包**         | 法币账户 + 数币钱包（USDT / BTC / ETH 等） |
| **On / OffRamp** | 法币 ↔ 数币双向兑换                       |
| **收单**         | 法币 + 数币收单、Checkout / API / H5       |
| **发卡**         | 卡发行与管理                               |
| **AI 风控**      | 合规审核、AML 监控、交易反欺诈、链上风控   |

平台采用统一签名加密、统一 Webhook、统一商户进件流程，**一次集成，全部产品线按需开通复用**。服务商可选择能力对应的产品线接入，无需覆盖全部。

## 四、客户结构 — 双边平台

EurewaX 是一个连接需求方与供给方的双边平台，服务两类客户：**EX 客户**（需要跨境支付能力的机构 / 平台 / 银行）与**服务商**（提供支付牌照与渠道能力的持牌机构）。

```
              ┌───────────────────────────────┐
              │       需求方 · EX 客户           │
              │   机构 / 平台 / 银行             │
              │  通过 EX 获取完整跨境支付能力    │
              └─────────────┬─────────────────┘
                            │
                  统一 API + 白标门户 + 合规风控
                            │
              ┌─────────────▼─────────────────┐
              │      EurewaX PPaaS 平台         │
              │  合规 · 清结算 · 商户运营 · 风控  │
              └─────────────▲─────────────────┘
                            │
                       渠道协同接入
                            │
              ┌─────────────┴─────────────────┐
              │       供给方 · 服务商           │
              │  持牌支付机构 · 银行 · Web3 机构 │
              │  通过 EX 触达客户与终端商户      │
              └───────────────────────────────┘
```

平台通过双边网络效应持续放大价值：客户群体的扩大持续吸引更多服务商接入，服务商能力的丰富反向提升平台对客户的服务能力。

## 五、面向服务商的价值主张

### 5.1 目标合作伙伴画像

适用于具备支付、清算或牌照能力，希望快速扩大渠道与客户覆盖的机构：

- 当地持牌支付机构（PSP）
- 持牌商业银行 / EMI / MPI
- Web3 支付机构 / VASP / 稳定币发行与兑换商
- 持牌发卡机构 / 卡组织 / 卡商
- 具备本地清算能力的资金调度商

### 5.2 核心价值

| # | 价值维度             | 说明                                                          |
| - | -------------------- | ------------------------------------------------------------- |
| 1 | **渠道触达**   | 接入一次即可触达平台 EX 客户及其旗下商户，无需逐一商务拓展    |
| 2 | **免 BD 获客** | 商户由 EX 客户带入，服务商专注能力交付即可获得稳定流量        |
| 3 | **合规协同**   | 商户进件与 KYC/KYB 审核由平台统一完成，降低服务商重复审核压力 |
| 4 | **运营自动化** | 平台提供统一对账、资金调拨与差错处理能力，降低服务商运营成本  |
| 5 | **风控加持**   | AI 风控前置降低欺诈与合规事件，保护服务商牌照与品牌声誉       |

### 5.3 商业回报模型

- **手续费收入**：按交易量收取手续费，服务商自主定价
- **换汇点差**：参与多币种兑换业务，赚取汇率点差
- **规模化效应**：随平台 EX 客户与商户增长，服务商交易量自然规模化
- **生态绑定**：平台 EX 客户分布于多个垂直领域，交易类型多元，抗风险能力强

## 六、自研 AI 智能风控体系

EurewaX 自主研发的 AI 智能风控引擎，是平台的核心护城河。

### 6.1 风控六大能力

| 能力                       | 说明                                                                  |
| -------------------------- | --------------------------------------------------------------------- |
| **商户进件 KYC/KYB** | AI 自动识别证照、UBO 穿透、董事/法人核验、不良名单筛查                |
| **交易反欺诈**       | 规则引擎与模型评分双层决策，识别异常交易、设备指纹、IP 风险、行为异常 |
| **AML / CFT 监控**   | 持续交易监控、关联方分析、可疑交易报告（STR）自动化生成               |
| **链上风控**         | 稳定币地址画像、链上溯源、混币器识别、制裁地址筛查                    |
| **规则可配置**       | 风控策略支持按客户、产品、地区、币种维度灵活配置                      |
| **监管报告**         | 合规报表自动生成、监管检查留痕、KYC/KYB 资料归档                      |

### 6.2 全链路风控流程

```
进件审核 → 交易前置 → 实时监控 → 事后分析 → 监管报告
  KYC/KYB    风险评分    AML/反欺诈   关联方/可疑   合规留痕
```

### 6.3 灵活交付方式

风控能力支持**模块化拆分调用**与**全链路统一服务**两种交付形态：

| 交付形态                 | 适用场景                                                                                   |
| ------------------------ | ------------------------------------------------------------------------------------------ |
| **单点服务**       | 按需调用单一风控能力（如仅 KYB 审核、仅 AML 名单筛查、仅链上地址核验等），快速补齐特定环节 |
| **组合服务**       | 多项能力组合调用（如 KYC + 反欺诈 + AML），适配多产品线的差异化风控需求                    |
| **全链路统一服务** | 从商户进件到交易监控、从事后分析到监管报告全流程一站式接入，构建完整合规闭环               |

风控策略支持按**客户、产品、地区、币种**等多维度灵活配置，避免一刀切，平衡合规要求与业务体验。
