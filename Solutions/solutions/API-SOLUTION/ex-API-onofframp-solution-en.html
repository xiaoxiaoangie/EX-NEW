↓ Get quote → Execute exchange

↓ Get quote → Execute exchange
Fiat credited → Fiat Account
↓ Withdraw to beneficiary bank
Fiat withdrawal → Beneficiary bank

4End-to-End Flow

Prerequisites

1 Merchant Onboarding + Product Activation
Submit Merchant→ Forward to SP→ KYC/KYB Review→ KYB_APPROVED→ Apply OnRamp/OffRamp→ PRODUCT_ACTIVE
Onboarding includes product application. SP performs all KYC/KYB.
2 Fiat Deposit (VA Path)
Apply VA→ VA Review→ VA_ACTIVE→ Bank Transfer (offline)→ Risk Review→ FIAT_DEPOSIT_SUCCESS
3 Crypto Deposit (On-chain)
Get Deposit Address→ Return {address, chain}→ On-chain transfer→ Confirm + AML→ CRYPTO_DEPOSIT_SUCCESS

OnRamp — Fiat → Crypto

4 OnRamp — Buy Crypto
Get OnRamp Quote→ Return {rate, fee, expiry}→ Execute OnRamp→ Deduct fiat + exchange + credit crypto→ ONRAMP_SUCCESS
⚠️ Quotes have expiry time — must execute before expiration or re-fetch
5 OnRamp — Fiat-to-Crypto
Get Fiat-to-Crypto Quote→ Execute Fiat-to-Crypto→ Deduct fiat + exchange + credit crypto→ FIAT_TO_CRYPTO_SUCCESS
Converts fiat account balance directly to crypto (no on-chain deposit needed)

OffRamp — Crypto → Fiat

6 OffRamp — Sell Crypto
Get OffRamp Quote→ Return {rate, fee, expiry}→ Execute OffRamp→ Deduct crypto + exchange + credit fiat→ OFFRAMP_SUCCESS
7 OffRamp — Crypto-to-Fiat
Get Crypto-to-Fiat Quote→ Execute Crypto-to-Fiat→ Deduct crypto + exchange + credit fiat→ CRYPTO_TO_FIAT_SUCCESS

Withdrawal

8 Fiat Withdrawal
Create Beneficiary→ Beneficiary Review→ BENEFICIARY_ACTIVE→ Initiate Withdrawal→ Risk + Payment→ WITHDRAWAL_SUCCESS
Beneficiary must be created and approved before fiat withdrawal.
9 Crypto Withdrawal
Initiate Withdrawal→ AML Review→ On-chain broadcast→ CRYPTO_WITHDRAWAL_SUCCESS

8Important Notes

EX Role: API aggregation + orchestration. No KYC, no risk, no exchange execution.
SP Role: Licensed institution performing reviews, exchange, fund custody.
Quote Expiry: All quotes have TTL — re-fetch after expiration.
Beneficiary First: Must create & approve beneficiary before fiat withdrawal.
Webhook Required: All async operations notify results via Webhook.
Idempotency: Pass idempotency ID for all critical operations to prevent duplicates.
