3账户关系 — 入金 → 兑换 → 出金路径

OnRamp 法币 → 数币

银行转账 → VA → 法币账户
↓ 获取报价 → 执行兑换
数币到账 → 数币钱包
↓ 可选：提币到外部地址
链上提币 → 外部钱包

OffRamp 数币 → 法币

链上充币 → 充币地址 → 数币钱包
↓ 获取报价 → 执行兑换

↓ 提现到 Beneficiary 银行账户
法币提现 → Beneficiary 银行卡

4全链路流程

前置流程

1 商户入网 + 产品开通
推送商户入网→ 转发 SP→ KYC/KYB 审核→ KYB_APPROVED→ 申请 OnRamp/OffRamp 产品→ PRODUCT_ACTIVE
入网即同时申请承兑产品。SP 执行所有 KYC/KYB，EX 不做审核。
2 法币充值（VA 路径）
申请 VA→ VA 审核→ VA_ACTIVE→ 线下银行转账→ 入账风控→ FIAT_DEPOSIT_SUCCESS
3 数币充值（链上充币）
获取充币地址→ 返回 {address, chain}→ 链上转入→ 链上确认 + AML→ CRYPTO_DEPOSIT_SUCCESS

OnRamp — 法币→数币

4 OnRamp — 买入数币
获取 OnRamp 报价→ 返回 {汇率, 费用, 过期时间}→ 执行 OnRamp→ 扣法币 + 承兑 + 入数币→ ONRAMP_SUCCESS
⚠️ 报价有过期时间，必须在过期前执行或重新获取报价
5 OnRamp — 法转数（Fiat-to-Crypto）
获取法转数报价→ 执行法转数→ 扣法币 + 兑换 + 入数币→ FIAT_TO_CRYPTO_SUCCESS
法转数 = 法币账户余额直接兑换为数币（不经过链上充币）

OffRamp — 数币→法币

6 OffRamp — 卖出数币
获取 OffRamp 报价→ 返回 {汇率, 费用, 过期时间}→ 执行 OffRamp→ 扣数币 + 承兑 + 入法币→ OFFRAMP_SUCCESS
7 OffRamp — 数转法（Crypto-to-Fiat）
获取数转法报价→ 执行数转法→ 扣数币 + 兑换 + 入法币→ CRYPTO_TO_FIAT_SUCCESS

出金流程

8 法币提现（Fiat Withdrawal）
创建 Beneficiary→ 收款人审核→ BENEFICIARY_ACTIVE→ 发起法币提现→ 风控 + 付款→ WITHDRAWAL_SUCCESS
提现到 Beneficiary 的银行账户，需先创建并审核通过收款人
9 数币提币（Crypto Withdrawal）
发起链上提币→ AML 风控→ 链上广播→ CRYPTO_WITHDRAWAL_SUCCESS

5RFI（补充材料）处理

SP 需补材料→ RFI_REQUEST→ 上传文件→ 提交 RFI→ 转发 SP→ RFI_RESOLVED / REJECTED
场景可能要求的文件
商户入网 KYB营业执照、董事证明、业务说明
VA 申请资金来源证明、业务合同
大额法币充值银行流水、贸易单据
数币充值链上资金来源证明
Beneficiary 创建收款人身份证明、关系证明
大额兑换交易背景说明、资金用途证明

8注意事项

EX 角色：API 聚合 + 编排层，不做 KYC、不做风控、不做承兑
SP 角色：持牌机构，执行审核、承兑兑换、资金托管
报价过期：所有报价有 TTL，过期后需重新获取
Beneficiary 先审后用：法币提现前必须创建并通过审核
Webhook 必须：所有异步操作通过 Webhook 通知结果
幂等设计：建议所有关键操作传入幂等 ID，防止重复执行
