4Card Issuance Full Flow

1 Merchant Onboarding + Product Activation
Submit Merchant→ Forward to SP→ KYC/KYB Review→ KYB_APPROVED→ Apply VCC Product→ PRODUCT_ACTIVE
Merchant created by tenant, pushed to SP via EX. SP performs KYC/KYB. May trigger RFI.
2 Fiat Deposit (VA Path)
Apply VA→ VA Review→ VA_ACTIVE→ Bank Transfer (offline)→ Risk Review→ FIAT_DEPOSIT_SUCCESS
3 Crypto Deposit (Deposit Address Path)
Get Deposit Address→ Return {address, chain}→ Transfer USDT/USDC→ On-chain + AML→ CRYPTO_DEPOSIT_SUCCESS
4 Issue Shared Card (First Time)
Create Shared Account→ Fund Shared Account→ FUNDED→ Issue Card→ CARD_ISSUED
First time: create & fund shared account; subsequent cards reuse existing account.
5 Issue Shared Card (Reuse)
Query Shared Accounts→ Issue Card (existing)→ CARD_ISSUED
6 Issue Standard Card
Issue + Fund→ Risk Review→ CARD_ISSUED_AND_FUNDED
7 Card Spending & Sync
Cardholder spends→ Card network auth→ SP settlement→ CARD_TRANSACTION

8Important Notes

EX Role: API aggregation + orchestration layer. No KYC/KYB, no risk control.
SP Role: Licensed institution performing reviews, account opening, card issuance, fund custody.
Batch Issuance: Shared accounts ideal for batch card issuance; standard cards for independent fund management.
Balance Monitoring: Subscribe to Webhooks for real-time wallet/account/card balance updates.
