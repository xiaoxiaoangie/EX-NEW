## 一、方案概述

EX 提供**白标门户（White Label Portal）**方案 —— 无需开发，即开即用。

EX 根据客户开通的产品线，由运营团队完成配置后，直接交付两个门户：

| 门户 | 英文 | 使用方 | 说明 |
|------|------|--------|------|
| **租户门户** | Tenant Portal（TP） | 客户（机构） | 客户自己使用的运营管理后台，管理商户、查看交易、配置产品等 |
| **商户门户** | Merchant Portal（MP） | 客户的终端用户（商户） | 客户交付给其终端商户使用的操作门户，商户自助完成收付款、发卡等操作 |

```
┌─────────────────────────────────────────────────────────────┐
│                       EX 平台                                │
│                                                              │
│  ┌──────────────────┐    ┌──────────────────┐               │
│  │   运营配置完成     │ →  │   交付 TP + MP    │               │
│  │ (产品开通+品牌配置)│    │  (即开即用)       │               │
│  └──────────────────┘    └──────────────────┘               │
│                                                              │
└──────────────────────┬──────────────────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼                         ▼
┌──────────────────┐     ┌──────────────────┐
│  TP（租户门户）    │     │  MP（商户门户）    │
│                   │     │                   │
│  客户（机构）使用   │     │  终端商户使用      │
│  管理商户/交易/配置 │     │  收付款/发卡/查询  │
└──────────────────┘     └──────────────────┘
```

> **核心价值**：客户无需任何开发投入，开通产品后即可获得完整的运营管理后台（TP）和面向终端商户的操作门户（MP），快速启动业务。

## 二、谁适合白标方案

| 客户类型 | 场景 | 推荐理由 |
|---------|------|---------|
| **代理机构** | 代理 EX 产品给终端商户 | 无需开发，TP 管理商户，MP 交付商户 |
| **跨境电商服务商** | 为卖家提供收付款服务 | 快速上线，卖家直接用 MP 操作 |
| **初创 Fintech** | 刚起步，优先验证业务模型 | 零开发成本，先跑通业务再考虑 API 集成 |
| **传统贸易公司** | 需要数字化收付工具但无技术团队 | 门户即开即用，降低数字化门槛 |
| **持牌金融机构** | 拓展新业务线（如发卡、加密） | TP 管理，快速扩展产品线 |

## 三、白标门户能力矩阵

根据客户开通的产品线，TP 和 MP 自动展示对应的功能模块：

### 3.1 TP（租户门户）— 客户使用

| 功能模块 | 说明 | 依赖产品 |
|---------|------|---------|
| **商户管理** | 注册商户、KYC/KYB 提交与跟踪、商户状态管理 | 基础（所有产品） |
| **产品开通** | 为商户申请开通各产品线、查看审核进度 | 基础（所有产品） |
| **交易监控** | 查看所有商户的收付款、换汇、发卡交易 | 按产品线展示 |
| **结算管理** | 商户结算、资金归集 | 法币收付 |
| **合规管理** | RFI 响应、调单处理、合规状态跟踪 | 所有产品 |
| **报表中心** | 交易汇总、对账单、导出报表 | 所有产品 |
| **开发者设置** | API Key 管理、Webhook 配置（如需 API + 门户混合使用） | 可选 |
| **品牌配置** | Logo、域名、邮件模板定制 | 基础 |

### 3.2 MP（商户门户）— 终端商户使用

| 功能模块 | 说明 | 依赖产品 |
|---------|------|---------|
| **资产总览** | 法币账户余额、加密钱包余额、按币种汇总 | 基础 |
| **法币收款** | 申请 VA、查看收款记录、下载凭证 | FIAT_COLLECTION |
| **法币付款** | 添加收款人、发起付款、结汇、查看付款记录 | FIAT_PAYOUT |
| **换汇** | 查询汇率、发起换汇 | 法币产品 |
| **加密钱包** | 查看钱包余额、充币、提币 | CRYPTO_WALLET |
| **买卖币** | 法币 ↔ 加密货币兑换 | FIAT_ONRAMP / FIAT_OFFRAMP |
| **收银台** | 创建收款订单、查看支付状态 | 聚合收银 |
| **发卡** | 申请虚拟卡、卡账户充值/转出、查看卡交易、卡管理 | CARD_ISSUING |
| **报表** | 交易明细、对账单 | 所有产品 |
| **贸易单据** | 上传/管理贸易背景文件 | 法币产品 |

## 四、交付流程

```
├── 1. 签约与产品确认
│     └── 确认开通的产品线组合
│
├── 2. EX 运营配置（1-2 个工作日）
│     ├── 创建租户（Tenant）
│     ├── 开通产品线
│     ├── 配置品牌信息（Logo、域名、邮件模板）
│     └── 配置合规规则和费率
│
├── 3. 交付 TP（租户门户）
│     └── 客户获得 TP 管理后台访问权限
│     └── 客户可开始在 TP 中注册和管理商户
│
├── 4. 交付 MP（商户门户）
│     └── 商户通过邀请链接注册
│     └── 完成 KYC/KYB → 产品开通 → 开始使用
│
└── 5. 上线运营
      └── 客户通过 TP 管理日常运营
      └── 终端商户通过 MP 自助操作
```

### 交付时间线

| 阶段 | 内容 | 预计耗时 |
|------|------|---------|
| 签约确认 | 产品组合、费率、品牌信息 | 1-2 天 |
| 运营配置 | 租户创建、产品开通、品牌配置 | 1-2 个工作日 |
| TP 交付 | 客户验收管理后台 | 1 天 |
| 首批商户入驻 | 商户注册 → KYC → 产品开通 | 3-5 个工作日（含审核） |
| **总计** | **从签约到首批商户可操作** | **5-10 个工作日** |

## 五、品牌定制能力

| 定制项 | 说明 |
|--------|------|
| **Logo** | TP 和 MP 门户顶部 Logo 替换为客户品牌 |
| **自定义域名** | 支持绑定客户自有域名（如 portal.yourcompany.com） |
| **邮件模板** | 商户注册邀请、审核通知等邮件使用客户品牌 |
| **配色方案** | 支持主色调定制（按需） |

> 终端商户看到的是客户品牌，感知不到 EX 的存在。

## 六、开始接入

准备好了吗？联系您的 EX 客户经理：

1. **确认产品组合** — 选择需要开通的产品线
2. **提供品牌素材** — Logo、域名、配色偏好
3. **签约** — 确认费率和合作协议
4. **交付** — EX 运营配置完成后，交付 TP + MP

> 从签约到首批商户可操作，最快 **5 个工作日**。
