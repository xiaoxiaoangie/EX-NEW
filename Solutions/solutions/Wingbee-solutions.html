Wingbee OnRamp / OffRamp — 解决方案

📄 Wingbee 承兑业务解决方案 📌 版本 v1.0 🕐 最后更新: 2026-04-16 👤 适用对象: Wingbee 平台

一、方案概述

1.1 各方角色

角色说明
Wingbee面向终端客户的平台，客户在 Wingbee 入网并操作 OnRamp / OffRamp
BB持牌合规通道，承兑执行方。Wingbee 将 BB 作为背后的承兑通道
EX技术服务商，为 BB 提供系统支撑。Wingbee 通过 EX API 与 BB 交互
客户Wingbee 的终端用户，在 Wingbee 平台完成法币 ↔ 加密货币兑换

1.2 架构关系

客户终端用户
→ 入网/交易 →
Wingbee平台
→ API 对接 →
EX技术服务
↓ 法币通道
↓ 系统支撑
智朗银行法币账户
BB持牌通道
  • Wingbee 作为客户入网到 EX，是 EX 平台上的一个租户
  • BB 是背后的承兑通道，负责 OnRamp/OffRamp 的实际执行和合规
  • EX 是技术服务商，为 BB 提供系统，Wingbee 通过 EX API 完成所有操作

1.3 核心原则

OnRamp 先收后做

Wingbee 必须先收到客户法币，确认到账后才发起 OnRamp。绝不垫资

OffRamp 头寸自管

Wingbee 自行管理法币头寸，可选择先垫付客户再等 BB 结算，但风险自担

双方不互垫头寸

BB 不为 Wingbee 垫资，Wingbee 不为 BB 垫资。各自管理各自的资金池

1.4 资金流向总览

OnRamp（法币 → 加密货币）

客户法币 → Wingbee 智朗账户 → 智朗内部转账给 BB → BB 执行 OnRamp → 加密货币入客户账户

OffRamp（加密货币 → 法币）

客户 USDT → Wingbee → USDT 转给 BB → BB 执行 OffRamp → 智朗行内转账给 Wingbee → Wingbee 付款给客户

二、前置流程

2.1 Wingbee 自身准备

├── 1. Wingbee 在智朗银行开设法币账户 │ └── 用于接收客户法币（OnRamp）和向客户付款（OffRamp） │ ├── 2. Wingbee 与 EX 签约 │ └── 作为租户入网 EX 平台 │ └── 开通 OnRamp / OffRamp 产品 │ ├── 3. Wingbee 通过自身作为商户通过 BB 入网（如果已经入网，我们协助迁移） │ └── Wingbee 需通过 EX 将自己注册为 BB 的商户，完成 KYC/KYB │ └── 用于以 Wingbee 自身账号发起 OnRamp / OffRamp 交易 │ └── OffRamp 收款账户（beneficiary）填写 Wingbee 的智朗银行账号 │ └── 4. 技术对接 └── 获取 Sandbox 环境 → 配置 APP ID / 公钥 / AES Key / Webhook └── 完成签名验签 + AES 加解密联调

2.2 客户入网（Wingbee 为客户完成）

客户入网、产品开通等流程

├── 1. 注册客户 │ └── Wingbee 通过 EX API 注册终端客户 → 获取 MID │ ├── 2. 提交产品开通所需信息（KYC/KYB 等） │ └── 根据产品要求上传审核材料 │ └── Webhook: 审核结果通知（APPROVED / REJECTED / RFI） │ └── 3. 产品审核通过 → 客户可使用 OnRamp / OffRamp

三、OnRamp 业务流程（法币 → 加密货币）

核心原则：Wingbee 必须先收到客户法币，确认到账后再发起 OnRamp。

步骤详解

├── 1. 客户发起法币充值 │ └── 客户在 Wingbee 前端发起法币充值请求 │ └── Wingbee 向客户展示智朗银行收款账户信息 │ ├── 2. 客户打款 │ └── 客户通过银行转账将法币打入 Wingbee 的智朗银行账户 │ ├── 3. Wingbee 确认到账 │ └── 在智朗银行确认收到客户法币（⚠️ 必须确认到账） │ ├── 4. Wingbee 发起 OnRamp │ └── 调用 EX API 发起 OnRamp 请求 │ └── EX 返回：OnRamp 订单信息（含应付法币金额、汇率、订单号） │ ├── 5. Wingbee 内部转账给 BB │ └── 根据 EX 返回的法币金额，从智朗账户转账至 BB 指定账户 │ ├── 6. BB 执行 OnRamp │ └── BB 收到法币后执行承兑（法币 → 加密货币） │ └── 加密货币入账到客户账户 │ └── 7. 交易完成 └── Webhook: OnRamp 交易结果通知 └── Wingbee 通知客户充值完成

四、OffRamp 业务流程（加密货币 → 法币）

4.1 前置：设置 Wingbee 收款账户

本期原则：OffRamp 统一走 Wingbee 自身商户号，收款人只有 Wingbee 智朗银行账号。BB 执行 EX 上 Wingbee 的指令，法币结算给 Wingbee，Wingbee 自行付款给客户。
├── 1. Wingbee 在 EX 添加自己的收款账户（beneficiary） │ └── 收款账户填写 Wingbee 的智朗银行账号 │ └── BB OffRamp 完成后，法币通过智朗银行内部转账到达 Wingbee │ ├── 2. 审核通过后，Wingbee 可用自身商户号发起 OffRamp │ └── 3. 客户提现账户由 Wingbee 自行管理 └── 客户在 Wingbee 前端添加提现银行账户 └── Wingbee 负责维护客户收款信息，不经过 EX

4.2 OffRamp 交易流程

├── 1. 客户发起提现 │ └── 客户在 Wingbee 前端发起加密货币提现（如 USDT → 法币） │ ├── 2. Wingbee 以自身商户号发起 OffRamp │ └── 调用 EX API，以 Wingbee 自己的 MID 发起 OffRamp │ └── 收款人 = Wingbee 智朗银行账号（前置已配置） │ └── EX 返回：OffRamp 订单信息（含应付法币金额、汇率、订单号） │ ├── 3. USDT 到达 BB │ └── 若客户使用 BB 钱包地址：USDT 直接到达 BB，此步免操作 │ └── 若客户使用其他钱包地址：Wingbee 需将 USDT 转给 BB │ ├── 4. BB 执行 OffRamp │ └── BB 执行承兑（USDT → 法币） │ └── 法币通过智朗银行内部转账到 Wingbee 账户 │ ├── 5. Wingbee 自行付款给客户 │ └── Wingbee 根据客户提现信息，从智朗银行账户付款给客户 │ └── 客户提现账户由 Wingbee 自行管理，不经过 EX/BB │ └── 6. 交易完成 └── Webhook: OffRamp 交易结果通知 └── Wingbee 通知客户提现完成

五、头寸管理与资金原则

5.1 核心原则

1. OnRamp：先收后做

BB 必须先确认收法币，才发起 OnRamp。不允许未收款就发起交易

2. OffRamp：头寸自管

Wingbee 自行管理智朗银行法币头寸。可先垫付客户，再等 BB 结算（风险自担）

3. 双方不互垫头寸

BB 不为 Wingbee 垫资，Wingbee 不为 BB 垫资。各管各的资金池

5.2 OffRamp 头寸策略

Wingbee 在 OffRamp 场景下有两种付款策略：

策略做法风险客户体验
保守策略（推荐） 等 BB OffRamp 完成，法币到 Wingbee 账户后再付款给客户 零风险 客户等待时间较长
激进策略 Wingbee 预先在智朗银行备足法币头寸，先垫付客户 OffRamp，再等 BB 结算 汇率波动风险、资金占用 客户体验好，到账快
建议：初期采用保守策略，业务稳定后根据资金情况考虑激进策略。激进策略下，Wingbee 需自行管理汇率敞口和流动性风险。

六、完整业务总览

前置流程：智朗开户 → EX签约入网 → 产品开通 → 技术对接 → 客户入网

OnRamp

法币 → 加密货币
  1. 客户法币 → Wingbee 智朗
  2. 确认到账
  3. 调 EX 发起 OnRamp
  4. 内部转账给 BB
  5. BB 执行 OnRamp
  6. 加密货币入客户账户

OffRamp

加密货币 → 法币
  1. 客户发起提现
  2. Wingbee 发起 OffRamp
  3. USDT 转给 BB
  4. BB 执行 OffRamp
  5. 法币到 Wingbee 智朗
  6. Wingbee 付款给客户
