3Fund Flow Breakdown

AInbound: Collecting from End Users

Path A1: Fiat Collection (Card / Local Wallet / Bank Transfer)

End User→ Acquirer Payment→ Acquirer Settlement→ VGO receives local currency
Acquirers settle in local currency to VGO. EX is not involved in this step.

Path A2: Crypto Collection

End User→ EX Crypto Checkout→ On-chain confirmation→ VGO crypto wallet credited
BConversion: Internal Fund Management

Path B1: Fiat → Crypto (OnRamp)

VGO local currency→ Convert to USD→ EX OnRamp→ USD → USDT→ VGO crypto wallet
VGO converts acquirer settlement (local currency) to USD independently, then uses EX OnRamp to convert to crypto.

Path B2: Crypto → Fiat (OffRamp)

VGO crypto wallet→ EX OffRamp→ USDT → USD→ VGO fiat account
COutbound: Paying Suppliers

Path C1: VCC Card Payment

VGO→ EX Fund + Issue Card→ VCC→ GDS / Airline charge→ Supplier receives
Funding source: crypto income directly / OnRamp crypto / fiat deposit
Applicable: airlines, GDS, hotels and other card-accepting suppliers

Path C2: OffRamp + Fiat Payout

VGO crypto→ EX OffRamp→ USDT → USD→ EX Payout→ Supplier bank account
Applicable: suppliers accepting bank transfers

Full Fund Flow Overview

End User → Acquirer → Local ccy settlement → VGO Booking → → USD → EX OnRamp → Fund / Issue VCC → Supplier
End User → EX Crypto Checkout → Crypto credited → VGO Booking → Issue VCC or OffRamp → Payout → Supplier
