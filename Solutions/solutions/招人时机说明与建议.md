# 关于招人时机的说明与建议

> 本报告只回答一个问题：**要不要招人、什么时候招人。**
> 结论：**现在不是招人的时机；正确的时机是"这几个方案中有一个真正落地、并且有业务量进来"。** 后文用工作量现状、团队状态和业务现实来支撑这个判断。

---

## 一、核心结论

- **现在不招人。**
- **招人的触发点不是"现在"，而是"几个方案中有一个落地、且有实际工作量/业务量进来"时**，再按业务量招对应的人。
- **根本原因**：现在的瓶颈**不是"水流的开口不够大"（人手/产能不足），而是"没有水"（没有业务）**。在没有水的时候把开口扩大（加人），水也不会变多，只会徒增成本。
- **而且团队现在很脆弱**：关键岗位有暂代、级别不足、稳定性存疑等问题。当务之急是**先把团队稳住**，而不是在不稳的结构上再加人。

---

## 二、为什么不是现在：不是"开口不够大"，而是"没业务"

这是最关键的一点，请先理解这个区别：

| 如果问题是…… | 那么…… | 现实是否如此 |
| --- | --- | --- |
| 水流的**开口不够大**（人太少、接不住涌入的业务） | 加人 = 扩大开口，**有效** | **否**——没有业务涌入 |
| **没有水**（没有业务量） | 扩大开口也没用，**先要把水引进来** | **是**——这才是当前真实情况 |

- 我手上几个方案（越南、结汇、卡台、大币种、PF、肯尼亚等）**目前无一落地、无一产生业务量**。
- 在"没有水"的阶段加人，等于先把管道开口修得更大——但没有水流过，**只增加确定的成本，换不来任何产出**。
- 所以正确的顺序是：**先把某个方案做落地、把水引进来；等有了稳定水流（业务量），再按需要扩大开口（招人）。**

---

## 三、团队现状：已经铺开，但很脆弱

招人之前必须看清现状——团队**人已经铺开（编制不缺），但结构很脆弱**，经不起此时再做大的人员变动。

### 1. 工作量分类

| 类别 | 内容 | 性质 |
| --- | --- | --- |
| **A：存量 / 日常** | 接渠道、日常需求、退易通的问题 | 持续性、维护性 |
| **B：迁移类** | VCC 迁移；**7 月中起**进入承兑迁移 | 阶段性、有明确节点 |
| **E：战略大项目** | 结汇、EX1.0 法币收付款、发卡、卡台、消费卡、承兑多 SP / MOR / 越南、数币收单 / PF、清结算、资管 / 换汇 / 端上等 | 战略性、决定未来业务量 |

### 2. 各团队分工与职责

| 团队 | 负责人 | 工作范围 | 覆盖 |
| --- | --- | --- | --- |
| **收付款团队** | 超湖（**暂代**） | 结汇方案、EX1.0 具备法币收付款能力、所有法币渠道 / SP 的对接 | A + E |
| **发卡团队** | 宾如 | VCC 迁移；E 的发卡（与 Occam 从头梳理所有发卡流程、AB/SP 的卡如何给到 E 用）；卡台项目；消费卡；对接卡渠道 | A + B + E |
| **承兑团队** | Kevin | SRO 方案、E 的多 SP 方案、MOR 方案、越南方案（OTC 方案 + PSP 对接等） | E |
| **收单团队** | 张婧 | 数币收单建设、PF 方案、对接渠道 | E |

| **资管 / 换汇 / 端上** | 占棋 | 资管、换汇、端上相关的统一设计 | E（基础能力） |

### 3. 团队很脆弱：三处关键人岗风险

| 风险点 | 现状 | 潜在影响 |
| --- | --- | --- |
| **收付款 - 超湖** | 仅为**暂代** | 关键团队缺稳定 owner，需尽快定岗或安排平稳交接 |
| **清结算 - 程艳** | 仅为**中级** | 清结算是 A/B/E 全线的计费分佣结算枢纽，岗位重要性与级别不完全匹配 |
| **资管/换汇/端上 - 占棋** | **稳定性**存疑 | 关键基础能力若人员波动，端上与资管/换汇会出现断档风险 |

> **结论**：团队当前**编制不缺、但结构脆弱**。此时最该做的是**先把团队稳住、把交接做平稳**，而不是再加新人去扰动一个本就不稳的结构。

## 四、什么时候招、招到哪里

### 1. 触发条件：有业务量了再招

招人的正确时机是——**这几个方案中有一个真正落地、并且有业务量 / 工作量进来**（例如越南方案跑通、开始有实际单量）。到那时：

- 才是"水来了、开口需要变大"，加人**才有意义**；
- 才能按真实业务量判断**招到哪个团队、招什么人**。

### 2. 为什么现在招会"留不住人"

即便不谈成本，单从"人能不能留住"角度，现在招也是个死结：

- **考核与业务量挂钩** → 现在没量，新人拿不到钱，**大概率留不住**；没量还扣钱，谁也顶不住。
- **考核不挂钩业务量** → 那就是**纯固定成本**，正撞老板"成本在涨"的担忧。

> **有业务量，岗位才能自己养活自己。**

### 3. 招人节奏：先稳团队、再抓聚焦、最后才按量补人

| 阶段 | 做什么 | 是否招人 |
| --- | --- | --- |
| 第一步：稳团队 | 处理 3 处人岗风险，确保涉及交接的部分平稳过渡 | 不招 |
| 第二步：抓聚焦 | Q3 聚焦大币种、结汇 SP、卡台、越南，把业务量跑出来 | 不招 |
| 第三步：按量补人 | 某方案（如越南）跑出业务量、且仍有真实缺口，再按业务量定向补人 | 条件性招 |

## 五、为什么这不是"人手不足"问题（补充推理）

> 这一节是给"为什么现在不招"做更深一层的支撑，核心仍是：**瓶颈是没业务，不是没人。**

### 1. 忙 ≠ 价值

| 现象（看得见） | 本质（要追问） |
| --- | --- |
| 我一个人 cover 不过来 | 是事情真的多，还是事情该不该做？ |
| 部门很忙 | 忙在哪？这些忙有没有转化成业务结果？ |
| 想招人来分担 | 招人是解决"产能不足"，但我们缺的真的是产能吗？ |

我们部门是出商业化方案的部门，**不直接创造收入**。所以招人是 **100% 确定的成本**，产出却**不确定、滞后**——在没有业务量的前提下招人，就是**用确定成本赌不确定回报**。

### 2. 在手 6 个项目，无一产生收益

| 项目 | 性质 | 当前状态 | 是否产生收益 |
| --- | --- | --- | --- |
| 越南 | 区域/市场拓展 | 推进中 | 否 |
| 结汇方案 | 资金/方案设计 | 推进中 | 否 |
| 卡台 | 渠道/能力建设 | 推进中 | 否 |
| PF 方案 | 方案设计 | 推进中 | 否 |
| 大币种方案 | 方案设计 | 推进中 | 否 |
| 肯尼亚 | 区域/市场拓展 | 推进中 | 否 |

> 与其在 6 个都没产出的项目上平摊精力再加人，不如**先聚焦 1–2 个最可能落地的**（大币种、结汇 SP、卡台、越南），把业务量跑出来。

### 3. 现有人力其实是被"重叠"消耗，而非真的不够

链路上的角色其实**已经就位**，问题不在售前，而在**我和产品经理高度重叠**：

| 角色 | 现状 | 核心职责 |
| --- | --- | --- |
| 售前（已有人支持，不在我这里） | 已就位、运转尚可 | 针对客户的实施方案、客户支持 |
| 我（市场/商业化产品） | 压力集中在我这里 | 出产品/商业化方案 **+ 方案的实现细节** |
| 产品经理 / 产品部 | 在产品侧 | 产品专业实现；但现在**又被要求"懂业务 + 懂市场 + 接渠道 + 财务合作"** |

**两个角色重复做商业化、实现细节又没归位 = 重复投入 + 责任不清**。这部分靠**理边界**就能释放，不需要靠加人。建议边界如下：

| 职责 | 归属（建议） |
| --- | --- |
| 懂市场/业务、商业化方案、业务方向、怎么变现 | **我主导**（避免和产品经理重复） |
| 产品专业实现、产品领域深度（发卡/收单等） | **产品经理 / 产品部** |
| 方案的实现细节 | **建议归产品侧**（需和产品部确认） |
| 客户实施方案、客户支持 | **售前**（已就位） |

## 六、Q3 要做什么：为"招人"创造条件

### 1. 工作原则

1. **团队稳定性优先**：先保证团队稳定；凡涉及交接的部分，必须确保**顺利、平稳交接**，避免关键模块断档。
2. **聚焦重点项目**：Q3 集中资源在当前最重要的项目上——**大币种、结汇 SP、卡台、越南**。

### 2. Q3 资源分配（按团队）

| 团队 | Q3 承接重点 |
| --- | --- |
| **收付款团队** | 产品经理**到港后顺利留存**；承接 EX1.0 的法币建设 + 法币渠道 + 结汇 SP |
| **承兑团队（Kevin）** | 越南方案 + 多 SP 等方案 |
| **发卡团队（宾如）** | 消费卡 + 卡台 |
| **资管/换汇/端上（占棋）** | 团队稳定 + 换汇 + 资管（数币相关）+ 端上产品建设 |
| **清结算（程艳）** | 能把清算的中后台等**承接好即可** |

### 3. 整体分工总览（Q3）

1. **我（本团队）**：越南 + 多 SP + MOR 的方案及细化。
2. **端上设计**：由占棋团队牵头统一设计，联合 Kevin 承兑、收付款团队、发卡团队**共同承接端上落地**。
3. **收单团队（张婧）**：从需求紧急度看当前工作量不大；但收单**必须补全**——当前我们最缺的是**"聚合收单的能力"**。
4. **Carl**：对接所有租户实施 + 售前 + 已对接租户 / SP 的需求。

## 七、下一步行动

1. **稳团队**：优先处理 3 处人岗风险（超湖定岗 / 程艳承接 / 占棋稳定），涉及交接的确保平稳。
2. **理边界**：和产品部把"我 vs 产品经理"的职责边界谈清楚（零成本，最高优先级）。
3. **抓聚焦**：Q3 集中资源在大币种、结汇 SP、卡台、越南，把业务量跑出来。
4. **按量招**：以"某方案落地、有业务量进来"为触发条件，再按业务量决定是否、以及在哪个团队补人。

## 八、一句话总结

> "现在不是开口不够大，而是**没有水（没有业务）**；而且团队现在很脆弱。所以现在不该招人——应该先把团队稳住、把重点项目跑出业务量。**等这几个方案中有一个真正落地、有业务量进来，我再按业务量招人**，那时岗位才能自己养活自己。"
