## 二、客户痛点：收单平台的结算困境

收单平台（Payment Facilitator，PF）通过收单行（如 Worldpay）为旗下商户提供收单服务。收单行定期将结算款打给 PF，PF 再分发给各商户。跨境场景下，PF 面临六大核心痛点：

| 痛点                   | 具体表现                            | 影响                  |
| ---------------------- | ----------------------------------- | --------------------- |
| **结算成本高**   | 给中国商户打款走 SWIFT，单笔 $15-40 | 小额结算利润被吃掉    |
| **到账时效慢**   | SWIFT T+2~T+5，叠加收单结算周期 T+3 | 商户实际 T+5~T+8 到账 |
| **换汇不透明**   | 中间行换汇加点多                    | 商户到手金额不可预期  |
| **合规文件繁琐** | 中国商户结汇需还原订单明细          | 商户操作成本高        |
| **入网效率低**   | 逐个为商户开立收款账户              | 运营成本高            |
| **缺乏账户体系** | PF 没有自己的账户系统，逐笔打款     | 无法统一管理商户资金  |

## 三、解决方案简介

EurewaX 为收单 PF 提供从「收单结算收款」到「商户分账结算」的全链路解决方案。

### 3.1 方案核心思路

1. **开立 VA 接收结算款**：PF 在支付的服务商处入网，开立 VA 接收收单行结算款
2. **同名充值**：收单行如结算到 PF 自有银行账户，PF 可通过同名充值到 VA
3. **账户结算提现**：PF 主账户收到结算款后，通过账户结算提现即时结算给商户
4. **商户自助操作**：商户到账后可使用换汇/结汇/提现等能力

### 3.2 方案亮点

| 维度               | 亮点                                                            |
| ------------------ | --------------------------------------------------------------- |
| **即时结算** | 账户结算提现在同一服务商体内完成，秒级到账，零银行手续费        |
| **账户体系** | PF 主账户 + 商户子账户，主子关系清晰，资金管理规范              |
| **灵活下发** | API / 白牌后台批量结算                                          |
| **商户自助** | 商户到账后可自主换汇、结汇、数承兑服务商侧 OnRamp（法币→数币） |
| **租户定价** | 换汇/结汇/OnRamp 费率由租户自主加价报价，EX 提供灵活计费能力    |
| **返点分佣** | 商户在数承兑服务商做承兑业务也归属租户体系，支持对租户返点分佣  |

## 五、方案架构

### 5.1 整体架构

```
收单行（如 Worldpay）
    │
    │ 收单行结算款
    ├──→ 方式①：直接打入 PF 的 VA
    └──→ 方式②：打入 PF 银行账户 → 同名充值到 VA
    ▼
收单 PF（通过 EurewaX API 或白牌后台管理）
    │
    │ 账户结算提现（即时到账）
    ▼
商户账户（PF 下的子商户）
    │
    ├──→ 换汇/结汇/提现
    └──→ 同名提现/充值到数承兑服务商 → OnRamp → 数币钱包
```

全部通过 EurewaX 统一 API 或白牌后台操作。合规 · 风控 · 开户 · 资金托管 —— 由持牌服务商执行。

### 5.2 参与方与角色

| 参与方                 | 角色               | 说明                                                     |
| ---------------------- | ------------------ | -------------------------------------------------------- |
| **收单 PF**      | 结算发起方         | 收单平台，通过收单行收单，需将结算款分发给各商户         |
| **商户**         | 结算接收方         | PF 的签约商户，接收 PF 的账户结算提现款                  |
| **支付的服务商** | 法币服务商         | 提供 VA 开户、账户体系、账户结算提现、换汇结汇           |
| **数承兑服务商** | 数字资产承兑服务商 | 提供 OnRamp 承兑能力（法币→数币）                       |
| **EX**           | 技术服务平台       | API 编排层 + 白牌运营平台，统一对接支付的服务商和数承兑服务商，不碰资金 |

### 5.3 核心卖点：同一服务商内资金处理

| 特性                     | 说明                                                                               |
| ------------------------ | ---------------------------------------------------------------------------------- |
| **即时到账**       | 同一服务商内账户结算提现 = 记账操作，无需走银行通道，秒级完成                      |
| **零通道成本**     | 服务商内部转账不产生银行手续费                                                     |
| **跨租户协同**     | 不同租户在同一服务商内可互转，商户可无缝使用多租户的能力                           |
| **资金不出服务商** | 收单行打入 VA → 账户结算提现给商户 → 商户换汇/结汇 → 提现，全程资金在服务商体内 |
| **资金安全**       | 资金全程在持牌服务商体系内流转，合规清晰，安全有保障                               |

## 七、前置流程与集成时间

### 7.1 前置流程

**整体顺序：PF 入网 → Sandbox 联调 → 商户批量入网 → 业务上线**

1. **PF 入网（KYB）**

   - PF 向 EurewaX 提交 KYB 材料（营业执照、法人信息、业务说明）
   - 支付的服务商审核 KYC/KYB
   - Webhook 通知审核结果
   - 审核通过 → 开立 PF 主账户 + VA
   - PF 获得 VA 账号，提供给收单行作为结算账户
2. **商户批量入网**

   - PF 收集商户 KYB 材料
   - PF 通过 EX API 或白牌后台批量推送商户入网
   - 支付的服务商审核 → 开立商户子账户，挂到 PF 主账户下
   - 建立主子账户关系
3. **开通 Sandbox 测试环境**

   - 步骤 1：开通 Sandbox，获取测试账号、测试域名
   - 步骤 2：获取 APP ID、平台公钥、AES Key
   - 步骤 3：生成 RSA 密钥对（SHA256withRSA，2048 位）→ 上传公钥
   - 步骤 4：配置 Webhook 回调地址
   - 步骤 5：完成签名验签 + AES 加解密联调

### 7.2 集成时间规划

| 阶段              | 内容                                     | 预计耗时 | 累计  |
| ----------------- | ---------------------------------------- | -------- | ----- |
| **Phase 0** | Sandbox 开通、密钥配置、签名验签联调     | 1-2 天   | 2 天  |
| **Phase 1** | PF KYB + VA 开立 + 主子账户体系建立      | 3-5 天   | 7 天  |
| **Phase 2** | 商户批量入网 API + 账户结算提现联调      | 5-7 天   | 14 天 |
| **Phase 3** | 换汇/结汇接入联调（可选）                | 3-5 天   | 19 天 |
| **Phase 4** | 承兑链路接入联调（可选，数承兑服务商侧） | 5-7 天   | 26 天 |
| **Phase 5** | 端到端测试 + 异常场景 + 小额验证         | 3-5 天   | 31 天 |
| **Phase 6** | 正式上线 + 监控配置                      | 2-3 天   | 34 天 |

> 说明：Phase 3（换汇/结汇）和 Phase 4（承兑）为可选阶段，可根据业务需求并行或延后。
