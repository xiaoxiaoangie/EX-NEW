# 2026 H2 / Q3 OKR（商业化产品）

> 角色：商业化产品经理 ｜ 周期：H2（Q3 详细 + Q4 展望）
> 对齐：`H2-plan-2026.md`、`vietnam-research-execution-plan.md`
> 考核映射：产品达成率(40%/目标60%) + 竞争优势项目 + 配合 BD 付费客户/交易量
> 协同说明：**FIBD 负责开拓金融渠道，与我（产品）做能力对齐；我不为 FIBD 背业绩，我的产出服务于 BD 的展业与签约。**

---

## H2 总目标

> H2 要拿到三个结果——**大币种稳固、越南方案成型、EX 架构升级**：

| 支柱                      | H2 结果目标                                                                              |
| ------------------------- | ---------------------------------------------------------------------------------------- |
| **1. 大币种稳固**   | 大币种方案落地并夯实，保证 EX 基本能力零事故，守住基本盘客户                             |
| **2. 越南方案成型** | 越南方案**方向明确、可复制、有竞争力**，从样板店沉淀为标准化方案，配合 BD 展业签约 |
| **3. EX 架构升级**  | 形成**EX V2.0产品架构方案**（多 SP、BB 资金中心等平台底座），支撑规模化与可复制    |

# Q3（7-9 月）

> 重点：**① 稳基本盘（大币种方案 + 结汇能力） ② 越南方案 + 样板店（含支付调研，先定客户画像） ③ 补齐中后台能力底座（EX 架构对齐 / SP 协同闭环）**
>
> （本版结合 Q2 复盘：“SP 签约多、上线少”、样板店难产、画像错配的根因在中后台与协同，故新增 O3 补底座）
>

## O1：稳住基本盘——大币种方案落地 + 结汇能力建设

> 核心逻辑：基本盘是现金流命脉，**不能塌、不能扰**。FIBD 找来的金融渠道与我做能力对齐，但 O1 的考核落点是基本盘自身的稳定与业绩，不替 FIBD 背指标。

### KR1：大币种方案落地，保证 EX 基本能力不出问题

| Task 阶段                     | 交付物（Q3 内）                                                                                                                          |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **① 方案给出-6月底**   | 输出大币种 OffRamp / 收付解决方案（渠道接入、合规路径、清结算/路由、成本模型）                                                           |
| **② 项目开发上线-7月** | 按方案完成开发并上线**≥ N 项**；配合 FIBD 金融渠道做能力对齐（对齐，不背业绩）                                                          |
| **③ 项目验证交付-7月** | 跑通闭环；核心系统 SLA 维持**0 重大事故**、无"交易量 < 5 万 USD"需求差；支撑 BD 在基本盘展业，交易量达成率向 **≥ 60%** 推进 |

### KR2：结汇能力建设，补齐基本盘后端弹药

| Task 阶段                     | 交付物（Q3 内）                                                                                        |
| ----------------------------- | ------------------------------------------------------------------------------------------------------ |
| **① 方案给出-6月底**   | 输出合规数币结汇方案（USDT↔法币 / 跨境结汇路径、合规口径、成本模型）                                  |
| **② 项目开发上线-7月** | 打通**≥ 1 条**结汇通道并上线，沉淀为 EX PaaS 通用能力；配合 FIBD 金融渠道做结汇适配（对齐，不背业绩） |
| **③ 项目验证交付-7月** | 闭环验证结汇能力补齐基本盘出金/结汇短板，支撑 BD 交易量与客户留存；能力可被越南 VND 场景复用           |

## O2：越南地区方案——调研看清市场，方案与样板店支撑 BD 签约

> 核心逻辑：Q3 产品不直接背交易量，**核心交付是一套能签约的越南方案 + 可演示的样板店**，让 BD 拿着方案能谈、指着样板店能成单。路径：先做透调研 → 输出商业产品方案 → 落地样板店 → 支撑 BD 完成 4 家租户签约。
>
> 吸取 Q2 教训：先圈定**目标客户画像**、做**可复制的标准方案**，避免“来一个客户做一套”的不可复制定制；样板店客群需与 EX 能力画像**匹配**（避免 Q2 画像错配）。

### KR1：越南支付调研（市场 / 友商 / 产品）

| Task 阶段          | 交付物（Q3 内）                                                                                          |
| ------------------ | -------------------------------------------------------------------------------------------------------- |
| **① 7月**   | 建立越南调研 SOP 与模板（竞品画像卡 / 客户画像卡 / 通道测评卡），统一归档至`market-research/`          |
| **② 7月中** | 协助 BD（+ FIBD 渠道侧）完成一线调研：市场摸底、友商通道实测、产品/客户走访                              |
| **③ 7月下** | 交付《越南支付调研报告》（市场机会、客户画像、合规路径、竞品模式、准入规则建议），反哺方案设计与准入规则 |

### KR2：越南地区方案落地，做出样板店支撑 BD 签约

| Task 阶段                     | 交付物（Q3 内）                                                                                                                  |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| **① 方案给出-7月初**   | 输出越南商业产品方案：OTC 对敲合规/半合规准入、承兑中控、合规风控组件、合规承兑网关 MVP、分润模型                                |
| **② 项目开发上线-8月** | 7 月承兑中控 1.0、8 月合规风控组件 1.0、9 月合规承兑网关 MVP 1.0 依次上线                                                        |
| **③ 项目验证交付-8月** | 落地**1-2 个**可演示、可复制样板店 + Sales Enablement Kit，**支撑 BD 完成 4 家租户签约**（产品配合，签约由 BD 主背） |

## O3：补齐中后台能力底座——EX 架构对齐 + 协同闭环

> 核心逻辑：Q2 暴露“SP 签约多、上线少”、样板店难产的**系统性根因在中后台**——ABE 架构未同频、前后端脱节。Q3 以**架构对齐为主**、协同机制为辅，把底座补上，让前端盘出的需求后端**接得住、跑得通、可复用**。（off-ramp 能力已并入 O1 大币种收付方案）

### KR1：EX 产品架构（ABE）层级关系对齐（重点）

| Task 阶段 | 交付物（Q3 内） |
| ----------------- | ---------------------------------------------------------------------------------------------- |
| **① 7-8月** | 明确 **ABE 在系统 / 产品层级的关系与处理规范**，统一各模块对 SP / 能力的承接口径 |
| **② 9月** | 形成 **EX V2.0 产品架构方案**（多 SP、BB 资金中心底座），支撑规模化与可复制 |

### KR2：建立“需求 → SP 选型 → 上线”协同闭环

| Task 阶段 | 交付物（Q3 内） |
| ----------------- | ---------------------------------------------------------------------------------------------- |
| **① 7月** | 联合销售 / FIBD 建立 SP 需求-选型对齐机制，输出 **SP 能力地图 + 优先级清单**，避免“签了用不上” |
| **② 8月** | 沉淀 **SP 接入标准化 SOP**，推动已签约 SP 上线；签约 → 上线转化率向 **≥ X%** 推进（口径与 FIBD 对齐） |
| **③ 9月** | 闭环验证：SP 能力可被**多租户 / 多场景复用**，打破“FIBD 子闭环” |

# Q4（10-12 月）展望

> Q4 只列方向、不展开 KR task；量化目标随 Q3 进展与业务信号再细化。

| 方向                              | 内涵                                                                    |
| --------------------------------- | ----------------------------------------------------------------------- |
| **1. 大币种方案持续夯实**   | 在 Q3 基础上继续打磨稳定性、扩展能力，巩固基本盘                        |
| **2. EX 整体产品建设架构**  | 推进多 SP、BB 资金中心等平台底座建设                                    |

## 考核映射（Q3）

| OKR                     | 对应考核                            | 说明                                                          |
| ----------------------- | ----------------------------------- | ------------------------------------------------------------- |
| O1-KR1 大币种方案       | 产品达成率 / 需求差                 | 方案上线率 ≥ 60%；无低交易量扣分项；支撑基本盘业绩与 BD 展业 |
| O1-KR2 结汇能力         | 产品达成率 / 竞争优势项目           | 结汇通道落地并复用；补齐基本盘后端、支撑 BD 交易量            |
| O2-KR1 越南支付调研     | 不可量化项目（奖金）                | 调研报告按成果审批；BD/FIBD 提供一线情报                      |
| O2-KR2 越南方案与样板店 | 产品达成率 / 配合 BD 付费客户达成率 | 方案三里程碑 + 1-2 样板店 + 支撑 BD 完成 4 家租户签约         |
| O3-KR1 ABE 架构对齐     | 竞争优势项目 / 不可量化（奖金）   | ABE 处理规范 + EX V2.0 架构方案                            |
| O3-KR2 SP 协同闭环       | 产品达成率 / 配合 BD 交易量       | SP 接入 SOP + 签约→上线率提升；能力可复用                |
