## 1. 法币账户（FIAT_ACCOUNT）

### 1.1 基本信息

| 字段           | 类型 | 说明             | 示例值                  |
| -------------- | ---- | ---------------- | ----------------------- |
| 产品代码       | 固定 | 产品唯一标识     | `FIAT_ACCOUNT`        |

### 1.2 同名充值能力（资金转入）

> **被 OnRamp 继承**：OnRamp 预约单的法币入金方式 = 法币账户的充值能力。

| 字段             | 类型 | 必填 | 说明                           |
| ---------------- | ---- | ---- | ------------------------------ |
| 是否开启同名充值 | 开关 | ✅   | 默认开启；关闭后仅支持持有余额 |

开启后展开：

| 字段     | 类型 | 必填 | 说明                              |
| -------- | ---- | ---- | --------------------------------- |
| 充值方式 | 多选 | ✅   | 银行转账 / 其他 EX 服务商账户转账 |

#### 方式一：银行转账（如支持）

> 商户通过外部银行账户向 SP 提供的收款账户转账入金。包括 VA（虚拟账户）和收付款账户两种形态。

**收款类型配置：**

**VA（虚拟账户）：**

| 字段                 | 类型 | 必填 | 说明                              |
| -------------------- | ---- | ---- | --------------------------------- |
| VA 支持的币种        | 多选 | ✅   | USD, HKD, EUR, CNH 等             |
| VA 支持的地区        | 多选 | ✅   | HK, SG, UK 等                     |
| VA 支持的银行        | 多选 | ✅   | HSBC, DBS, Barclays（按地区联动） |
| VA 同名校验          | 开关 | ✅   | 是否要求汇款人与商户同名          |
| VA Purpose Code 校验 | 开关 | ✅   | 是否要求转账备注含 Purpose Code   |

**收付款账户（SP 代收付大账户 / Omnibus）：**

> SP 的代收付大账户，多商户共用同一个银行账号，通过转账备注（Reference）识别入账归属。

| 字段                 | 类型 | 必填 | 说明                                             |
| -------------------- | ---- | ---- | ------------------------------------------------ |
| 收付款账户模式       | 固定 | ✅   | Omnibus（SP 代收付大账户，多商户共用），本期固定 |
| 收付款账户支持的币种 | 多选 | ✅   | USD 等                                           |
| 入账识别方式         | 固定 | ✅   | 转账备注（Reference Code），本期固定             |

**商户准入限制：**

> SP 配置哪些商户可以使用银行转账充值。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 主体所在国家 | 多选 | ✅ | 允许使用银行转账充值的商户注册国家，如 HK, SG, US |
| 主体类型 | 多选 | ✅ | 企业 / 个人 等 |
| 商户行业 | 多选 | ✅ | 允许的商户行业（MCC 或行业分类） |

**汇款方限制：**

> SP 配置允许的汇款来源类型。每条规则包含关系类型、主体类型、机构类型、机构名称。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 汇款方关系类型 | 单选 | ✅ | 同名（本期仅同名） |
| 汇款方主体类型 | 单选 | ✅ | 企业 / 个人 |
| 汇款方机构类型 | 单选 | ✅ | 银行 / 机构（三方支付机构） |
| 汇款方机构名称 | 多选 | 条件必填 | 机构类型=机构时，指定允许的三方支付机构名称（如 Pingpong, XTransfer）；银行类型则不限 |
| 不支持汇款方处理 | 固定 | ✅ | 人工处理（本期固定） |

> 支持配置多条汇款方规则。例如：
> - 规则1：同名 + 企业 + 银行 → 允许所有银行同名转入
> - 规则2：同名 + 企业 + 机构 → 仅允许 Pingpong、XTransfer 转入

**充值限额：**

| 字段 | 单位 | 可选值 | 说明 | 示例 |
|------|------|--------|------|------|
| 单笔限额 | USD | 具体金额范围 | 单笔充值金额范围 | 0 - 200,000 USD |
| 单月限额 | USD | 具体金额范围 | 每月充值累计上限 | 0 - 300,000 USD |
| 超限额处理方式 | 固定 | 人工处理 | 超过限额后的处理方式，本期固定为人工处理 | 人工处理 |

**报备要求：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 是否需要报备 | 开关 | ✅ | 充值前是否需要向 SP 报备，默认需要 |

#### 方式二：其他 EX 服务商账户转账（如支持）

> 商户在 EX 平台内的**其他 SP 账户余额**，通过内部转账方式转入当前 SP 账户。
> 例如：IPL 法币账户 → BB 法币账户，BB 法币账户 → IPL 法币账户。
> 本质是 EX 平台内跨 SP 的同名资金划转。

| 字段                   | 类型 | 必填     | 说明                                           |
| ---------------------- | ---- | -------- | ---------------------------------------------- |
| 是否支持跨 SP 转账充值 | 开关 | ✅       | 是否允许从其他 EX 服务商账户转入               |
| 可接收转账的来源 SP    | 多选 | 条件必填 | 配置允许从哪些 SP 转入（如 BB 配置可接收 IPL） |
| 跨 SP 转账支持的币种   | 多选 | 条件必填 | 仅限双方 SP 都支持的币种（交集）               |
| 跨 SP 转账是否需要换汇 | 开关 | ✅       | 源币种≠目标币种时是否支持自动换汇             |

**商户准入限制：**

> SP 配置哪些商户可以使用跨 SP 转账充值。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 主体所在国家 | 多选 | ✅ | 允许使用跨 SP 转账的商户注册国家 |
| 主体类型 | 多选 | ✅ | 企业 / 个人 等 |
| 商户行业 | 多选 | ✅ | 允许的商户行业（MCC 或行业分类） |

**充值限额：**

| 字段 | 单位 | 可选值 | 说明 | 示例 |
|------|------|--------|------|------|
| 单笔限额 | USD | 具体金额范围 | 单笔转账金额范围 | 0 - 200,000 USD |
| 单月限额 | USD | 具体金额范围 | 每月转账累计上限 | 0 - 300,000 USD |
| 超限额处理方式 | 固定 | 人工处理 | 超过限额后的处理方式，本期固定为人工处理 | 人工处理 |

**报备要求：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 是否需要报备 | 开关 | ✅ | 转账前是否需要向 SP 报备，默认需要 |

**跨 SP 转账规则：**

```
前提：商户同时在 SP-A 和 SP-B 开有法币账户
操作：从 SP-A 的 USD 账户 → 转入 SP-B 的 USD 账户

校验：
  1. SP-B 是否开启「跨 SP 转账充值」？
  2. SP-B 的「可接收来源 SP」是否包含 SP-A？
  3. 转账币种是否在「跨 SP 转账支持的币种」内？
  4. 商户在两个 SP 下是否都已完成 KYC（同名验证）？
  → 全部通过 → 执行内部划转
```

**示例：**

- BB 配置：可接收来源 = [IPL]，支持币种 = [USD, EUR]
- IPL 配置：可接收来源 = [BB]，支持币种 = [USD]
- 商户从 IPL USD → BB USD ✅
- 商户从 BB EUR → IPL EUR ❌（IPL 跨 SP 只配了 USD）

### 1.3 入账匹配规则

> **核心问题**：收款币种与账户持有币种不一致时如何处理？
> **入账模式**：固定为**混合模式**（原币种入账 + 换汇入账）。至少 USD 为原币种入账，不支持的币种自动换汇到指定目标币种。
> **三层配置**：SP 定义能力范围 → TP 配置默认规则（并决定商户是否可自选） → 商户自选（仅 TP 允许时）。

#### 1.3.1 入账模式说明

入账模式**固定为混合模式**，不提供模式选择。每一层配置的核心就是两件事：

| 配置项                         | 含义                                       | 示例     |
| ------------------------------ | ------------------------------------------ | -------- |
| **原币种入账的币种列表** | 这些币种收到后直接入对应币种账户           | USD, EUR |
| **换汇入账的目标币种**   | 不在原币种列表内的币种，自动换汇后入此账户 | USD      |

> USD 必须在原币种入账列表中（系统强制），确保美元收款始终原币入账。

#### 1.3.2 三层配置模型

```
第1层：SP 能力定义（上架时固定）
  → 定义哪些币种可原币入账、哪些币种可作为换汇目标
  → 不可被 TP/商户修改

第2层：TP 产品配置（TP 签约后配置）
  → 在 SP 能力范围内收窄：允许哪些原币种入账、默认换汇目标币种
  → 关键：配置「是否允许商户自选」开关
     ├── 开 → 商户可在 MP Portal 自行配置入账规则
     └── 关 → 商户无法配置，统一使用 TP 的配置（默认）

第3层：商户自选（仅 TP 允许时）
  → 商户在 TP 允许的范围内，选择原币入账币种、换汇目标币种
  → 商户可选范围 ⊆ TP 允许范围 ⊆ SP 能力范围
```

#### 1.3.3 SP 能力定义（第1层）

| 字段                   | 类型 | 必填 | 说明                                           |
| ---------------------- | ---- | ---- | ---------------------------------------------- |
| 原币种入账支持的币种   | 多选 | ✅   | SP 能力范围内可原币入账的币种（USD 必选）      |
| 换汇入账支持的目标币种 | 多选 | ✅   | SP 能力范围内可作为换汇目标的币种，如 USD, HKD |
| 换汇汇率来源           | 下拉 | ✅   | SP 实时汇率 / 固定汇率（本期仅 SP 实时汇率）   |

#### 1.3.4 TP 配置（第2层）

| 字段                       | 类型           | 说明                                                          | 约束                 | 默认值       |
| -------------------------- | -------------- | ------------------------------------------------------------- | -------------------- | ------------ |
| 允许的原币入账币种         | 多选           | TP 允许商户使用的原币入账币种                                 | ⊆ SP 支持，USD 必含 | = SP 全部    |
| 默认换汇目标币种           | 单选           | **商户未配置 / 不允许商户自选时**，使用此换汇目标       | ∈ SP 支持           | USD          |
| 允许的换汇目标币种         | 多选           | 商户可选的换汇目标范围（仅允许商户自选时生效）                | ⊆ SP 支持           | = SP 全部    |
| **是否允许商户自选** | **开关** | **核心开关**：决定商户能否在 MP Portal 自行配置入账规则 | —                   | **否** |

> **关闭时**（默认）：所有商户统一使用 TP 配置的「允许的原币入账币种」+「默认换汇目标币种」，商户无感知。
> **开启时**：商户可在 MP Portal 的 Settings 中自行配置，TP 配置作为默认值。

#### 1.3.5 商户自选配置（第3层）

> **前提**：TP 开启了「是否允许商户自选」。未开启时商户看不到此配置项。

| 字段         | 类型 | 说明                       | 约束                 |
| ------------ | ---- | -------------------------- | -------------------- |
| 原币入账币种 | 多选 | 商户选择哪些币种原币入账   | ⊆ TP 允许，USD 必含 |
| 换汇目标币种 | 单选 | 商户选择换汇入账的目标币种 | ∈ TP 允许的换汇目标 |

**生效优先级**：

```
TP 允许商户自选？
  ├── 否 → 直接使用 TP 配置
  └── 是 → 商户已配置？
        ├── 是 → 使用商户配置
        └── 否 → 使用 TP 配置（作为默认值）
```

#### 1.3.6 入账匹配完整逻辑

```
收到一笔入账（币种 = X）：
  0. 读取生效配置：
     - TP 未开放商户自选 → 用 TP 配置
     - TP 开放 + 商户已配置 → 用商户配置
     - TP 开放 + 商户未配置 → 用 TP 配置（默认值）
  1. X ∈ 原币种入账列表？（USD 一定在列表中）
     → 是：入 X 币种账户（原币入账）
     → 否：进入第2步
  2. X → 换汇 → 入「换汇目标币种」账户
  3. 换汇失败 → 异常处理：
     → 挂起，人工处理（本期）
     → 自动退回（下期）
```

#### 1.3.7 完整示例

```
SP 能力：
  原币种 = [USD, EUR, GBP, HKD]
  换汇目标 = [USD, HKD]

═══ 场景1：TP 不允许商户自选 ═══

TP 配置：
  允许原币种 = [USD, EUR]
  默认换汇目标 = USD
  允许商户自选 = 否              ← 商户无法配置

所有商户统一走 TP 配置：
  → 收 USD → 入 USD 账户（原币入账）
  → 收 EUR → 入 EUR 账户（原币入账）
  → 收 GBP → 换汇 → 入 USD 账户
  → 收 JPY → 换汇 → 入 USD 账户

═══ 场景2：TP 允许商户自选 ═══

TP 配置：
  允许原币种 = [USD, EUR]
  默认换汇目标 = USD
  允许换汇目标 = [USD, HKD]
  允许商户自选 = 是              ← 商户可以配置

商户 A（自行配置了）：
  原币入账 = [USD, EUR]
  换汇目标 = HKD
  → 收 USD → 入 USD 账户
  → 收 EUR → 入 EUR 账户
  → 收 GBP → 换汇 → 入 HKD 账户    ← 用商户自选

商户 B（未配置，走 TP 默认）：
  → 收 USD → 入 USD 账户
  → 收 EUR → 入 EUR 账户
  → 收 GBP → 换汇 → 入 USD 账户    ← 用 TP 默认
```

### 1.4 同名提现能力（资金转出）

> **被 OffRamp 继承**：OffRamp 承兑后法币出款 = 法币账户的提现能力。
> **本期限制**：代理模式下，本期仅支持"提现"（资金回到自己账户），不支持付款给第三方。

| 字段             | 类型 | 必填 | 说明                           |
| ---------------- | ---- | ---- | ------------------------------ |
| 是否开启同名提现 | 开关 | ✅   | 默认开启；关闭后不支持同名提现 |

### 1.5 提现渠道配置

> **本期支持的提现渠道（出款方式）完整列表：**

| 渠道代码           | 渠道名称                                  | 说明                                             | 适用场景                         |
| ------------------ | ----------------------------------------- | ------------------------------------------------ | -------------------------------- |
| `TT`             | 电汇（Telegraphic Transfer）              | 国际电汇，通过 SWIFT 网络                        | 跨境大额，USD/EUR/GBP 等主流币种 |
| `LOCAL_BANK`     | 本地银行转账（Local Bank Transfer）       | 当地清算网络（如 FPS/CHATS/FAST/ACH）            | 同地区本地币种转账，到账快       |
| `SEPA`           | 欧洲单一支付区（SEPA Transfer）           | 欧洲区域内 EUR 转账                              | 欧洲区域 EUR 提现                |
| `LOCAL_WALLET`   | 本地电子钱包（Local Wallet）              | 出款到本地电子钱包（如 GCash/GrabPay/PromptPay） | 东南亚等地区小额快速出款         |
| `EX_SP_TRANSFER` | EX 其他服务商账户（SP Internal Transfer） | 划转到商户在 EX 平台其他 SP 的法币账户           | 跨 SP 资金调拨，如 BB→IPL       |
| —                 | ~~Web3 地址~~                            | ~~已移至数币钱包提币能力~~                      | 见 2.3 同名提币                  |

#### SP 渠道能力定义

| 字段           | 类型 | 必填 | 说明                           |
| -------------- | ---- | ---- | ------------------------------ |
| 支持的提现渠道 | 多选 | ✅   | 从上述渠道中选择 SP 支持的渠道 |

**TT 电汇（如支持）：**

| 本地转账支持的清算网络    | 多选 | ✅   | FPS / CHATS / FAST / ACH / BACS 等（按地区联动） |

**SEPA（如支持）：**

| 字段            | 类型 | 必填 | 说明                                                 |
| --------------- | ---- | ---- | ---------------------------------------------------- |

**Local Wallet 本地钱包（如支持）：**

| 字段             | 类型 | 必填 | 说明                                       |
| ---------------- | ---- | ---- | ------------------------------------------ |
| 支持的钱包类型   | 多选 | ✅   | GCash / GrabPay / PromptPay / TrueMoney 等 |

**EX 其他服务商账户转账（如支持）：**

> 与充值的「方式二：其他 EX 服务商账户转账」对称。从当前 SP 账户划转到商户在 EX 平台其他 SP 的法币账户。
> 例如：BB 法币账户 → IPL 法币账户，IPL 法币账户 → BB 法币账户。

| 字段                   | 类型 | 必填     | 说明                                            |
| ---------------------- | ---- | -------- | ----------------------------------------------- |
| 是否支持跨 SP 转账提现 | 开关 | ✅       | 是否允许划转到其他 EX 服务商账户                |
| 可划转的目标 SP        | 多选 | 条件必填 | 配置允许划转到哪些 SP（如 BB 配置可划转到 IPL） |
| 跨 SP 转账支持的币种   | 多选 | 条件必填 | 仅限双方 SP 都支持的币种（交集）                |
| 跨 SP 转账是否需要换汇 | 开关 | ✅       | 源币种≠目标币种时是否支持自动换汇              |

> **与充值方向的关系**：充值的「可接收来源 SP」和提现的「可划转目标 SP」是独立配置，允许单向。

> **链上出款（Web3 地址）**：不属于法币账户提现渠道。链上出款 = 数币钱包的同名提币能力（见第2章 2.3 节）。
> 如需从法币余额出款到链上地址，路径为：法币账户提现 → OffRamp 换汇 → 数币钱包提币。

## 3. 收款人 vs 提现的属性边界

### 3.1 核心原则

```
收款人（Beneficiary/Payee）= "谁收钱" + "收到哪里"
  → 身份信息、银行/钱包/地址信息、审核状态
  → 一次添加，多次复用

提现（Withdrawal）= "这次转多少" + "走哪个渠道"
  → 金额、出款渠道、换汇、本次交易状态
  → 每次提现独立
```

### 3.2 对比表

| 维度       | 归属「收款人」          | 归属「提现」                |
| ---------- | ----------------------- | --------------------------- |
| 谁收钱     | ✅ 收款人姓名/公司名    | —                          |
| 与商户关系 | ✅ Self/法人/董事       | —                          |
| 账户信息   | ✅ 银行账号/钱包账号    | —                          |
| 收款人状态 | ✅ 待审核/已通过/已拒绝 | —                          |
| 出款渠道   | —                      | ✅ TT/Local/SEPA/Wallet     |
| 出款金额   | —                      | ✅ 本次提现金额             |
| 出款币种   | —                      | ✅ 本次提现币种             |
| 是否换汇   | —                      | ✅ 提现币种≠账户币种时触发 |
| 交易状态   | —                      | ✅ 处理中/成功/失败         |

### 3.3 本期代理模式的特殊约束

```
本期（代理模式）：
  ├── 收款人身份关系 = Self / Legal Representative / Director
  ├── 收款人类型 = 银行账户 / 本地钱包（Web3 地址归数币钱包提币）
  ├── 不支持第三方收款人（POBO 归 Payouts）
  └── 提现 = 选择已有收款人 + 选择渠道 + 输入金额

下期（Payouts 产品开放后）：
  ├── 收款人身份关系 += Third Party
  ├── 支持 POBO（付款人≠收款人）
  └── Payouts = 非同名付款，独立于提现
```

## 4. 本期 vs 下期

### 4.1 本期实现（Phase 1）

```
✅ 法币账户：开户、持有、同名充值（VA/代收付）、同名提现
✅ 数币钱包：开户、持有、同名充币、同名提币
✅ 入账匹配：三层配置（SP能力 → TP默认 → 商户自选）
✅ 提现渠道：TT / Local Bank Transfer / SEPA / Local Wallet（链上出款归数币钱包提币）
✅ 收款人身份：Self / Legal Representative / Director
✅ 收款人类型：银行账户 / 本地钱包（Web3 地址归数币钱包）
```
