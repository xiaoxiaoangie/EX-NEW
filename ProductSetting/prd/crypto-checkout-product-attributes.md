## 产品定位

**数币收单（Crypto Checkout）** 是 EX 平台的收单编排层产品：

- 商户以 **法币（USD）计价** 发起收款请求，付款人用**稳定币（USDT/USDC）**完成链上支付
- EX 本期以**四方聚合**模式外接一条链上收单通道，不自建链上能力
- 每笔订单生成**独立收款地址（One-time Address）**，付款人转账后通道确认到账，结算入商户数币钱包
- EX 是 B2B PPaaS，不直接面向 C 端；TP 通过 EX API 集成后，面向其自己的商户提供服务

**与其他产品的关系：**

| 产品                      | 收款方式                 | 计价                           | 场景                  |
| ------------------------- | ------------------------ | ------------------------------ | --------------------- |
| Crypto Wallet（充币）     | 链上转账，同名充值       | 数币计价                       | 商户自身资金管理      |
| Payins                    | VA / 链上 / 同名，纯记录 | —                             | 纯记录层，无收单编排  |
| **Crypto Checkout** | 链上支付，付款人主动发起 | **USD 计价，稳定币支付** | 商户向其用户/买家收款 |

**配置层级：**

```
第1层：产品层（SA 上架时定义，TP 不可修改）
  → 支持的支付币种与网络、每单独立地址、订单有效期范围、结算规则

第2层：TP 产品配置（TP 签约后配置，可对商户做特殊配置）
  → 订单有效期默认值、结算账户配置

第3层：商户配置（商户在 MP Portal 自行配置）
  → underpaid / overpaid 处理策略（未配置时继承产品层默认值）
```

## 1. 产品层属性（Phase 1）

> 由 SA 上架时定义，TP 不可修改，所有商户共享同一套产品能力。

### 1.1 支付币种与网络

| 字段             | 类型 | 本期值                                        | 说明                                       |
| ---------------- | ---- | --------------------------------------------- | ------------------------------------------ |
| 订单计价币种     | 固定 | USD                                           | 商户创建订单时以 USD 计价                  |
| 支持的支付稳定币 | 多选 | USDT / USDC                                   | 付款人可用的稳定币，预留扩展               |
| 支持的区块链网络 | 多选 | 由接入通道决定（如 TRC-20 / ERC-20 / BEP-20） | 本期外接一条聚合通道，网络范围由通道提供   |
| 汇率来源         | 固定 | 通道实时汇率（USD → 稳定币等值换算）         | 创建订单时换算应付稳定币金额，展示给付款人 |

> 支付稳定币与网络的组合（如 USDT TRC-20 / USDC ERC-20）由通道能力决定，TP/商户不可修改。

### 1.2 收款地址模型

| 字段                | 类型   | 本期值                           | 说明                             |
| ------------------- | ------ | -------------------------------- | -------------------------------- |
| 地址分配模式        | 固定   | 每单独立地址（One-time Address） | 每笔订单生成唯一收款地址，不复用 |
| 链上确认数          | 固定   | 由通道决定，产品层不配置         | 通道确认到账后通知 EX            |
| 预授权（0-confirm） | 不支持 | —                               | 本期不支持                       |

### 1.3 订单有效期范围

| 字段           | 类型         | 本期值  | 说明                            |
| -------------- | ------------ | ------- | ------------------------------- |
| 最短订单有效期 | 数字（分钟） | 15 分钟 | TP/商户配置的有效期不可低于此值 |
| 最长订单有效期 | 数字（小时） | 24 小时 | TP/商户配置的有效期不可超过此值 |

### 1.4 结算规则

| 字段             | 类型   | 本期值       | 说明                                         |
| ---------------- | ------ | ------------ | -------------------------------------------- |
| 结算模式         | 固定   | 逐笔结算     | 每笔订单确认到账后单独结算，不做批次合并     |
| 结算币种规则     | 固定   | 同货币同网络 | 收到 USDT TRC-20，结算 USDT TRC-20；不做换汇 |
| 结算目标账户类型 | 固定   | 商户数币钱包 | 结算入商户在 BB的数币钱包余额                |
| 自动换汇         | 不支持 | —           | 本期不支持收数币→换汇→入法币账户           |

### 1.5 Webhook 通知事件

| 事件                   | 说明                                                |
| ---------------------- | --------------------------------------------------- |
| `checkout.paid`      | 订单支付成功（通道确认到账）                        |
| `checkout.expired`   | 订单有效期到期，未完成支付                          |
| `checkout.settled`   | 结算完成，资金入商户数币钱包                        |
| `checkout.underpaid` | 收到转账但金额不足（underpaid 策略=等待补款时触发） |

### 1.6 underpaid / overpaid 系统默认值

| 字段               | 类型 | 枚举值              | 系统默认值 | 说明                                                       |
| ------------------ | ---- | ------------------- | ---------- | ---------------------------------------------------------- |
| underpaid 处理方式 | 下拉 | 等待补款 / 自动关单 | 等待补款   | 商户未自行配置时使用此默认值                               |
| overpaid 处理方式  | 下拉 | 自动接受 / 转人工处理 | 自动接受   | 商户未自行配置时使用此默认值                               |

> 产品层定义枚举范围和系统默认值，商户可在 MP Portal 自行修改。TP 不参与此配置。

## 2. TP 产品配置（Phase 1）

> TP 在产品层能力范围内配置，不可超出产品层上限。

### 2.1 支付币种与网络

| 字段             | 类型 | 说明                    | 约束          | 默认值       |
| ---------------- | ---- | ----------------------- | ------------- | ------------ |
| 允许的支付稳定币 | 多选 | TP 允许商户接收的稳定币 | ⊆ 产品层支持 | = 产品层全部 |
| 允许的区块链网络 | 多选 | TP 允许商户使用的链     | ⊆ 产品层支持 | = 产品层全部 |

### 2.2 订单有效期

| 字段                 | 类型         | 说明                                 | 约束                   | 默认值  |
| -------------------- | ------------ | ------------------------------------ | ---------------------- | ------- |
| 默认订单有效期       | 数字（分钟） | 商户创建订单未传有效期时使用的默认值 | 产品层最短～最长范围内 | 30 分钟 |
| 允许商户自定义有效期 | 开关         | 商户创建订单时是否可传入自定义有效期 | —                     | 是      |

### 2.3 结算账户配置

| 字段         | 类型 | 说明                       | 约束                        | 默认值           |
| ------------ | ---- | -------------------------- | --------------------------- | ---------------- |
| 商户结算账户 | 关联 | 指定结算入哪个数币钱包账户 | 商户在 BB 已开通的数币钱包 | 商户默认数币钱包 |

## 3. 商户配置（Phase 1）

> 商户可在 MP Portal 自行配置。未配置时使用产品层系统默认值（见 1.6）。

### 3.1 underpaid 处理策略

| 字段               | 类型 | 说明                         | 枚举值              | 默认值   |
| ------------------ | ---- | ---------------------------- | ------------------- | -------- |
| underpaid 处理方式 | 下拉 | 付款金额不足订单金额时的处理 | 等待补款 / 自动关单 | 等待补款 |

- **等待补款**：订单有效期内，付款人可多次转账，累计金额达到订单金额后视为支付成功
- **自动关单**：只要单次到账金额不足，订单立即关闭；已到账资金按 overpaid 规则处理（入商户账户）

### 3.2 overpaid 处理策略

| 字段              | 类型 | 说明                         | 枚举值                | 默认值   |
| ----------------- | ---- | ---------------------------- | --------------------- | -------- |
| overpaid 处理方式 | 下拉 | 付款金额超出订单金额时的处理 | 自动接受 / 转人工处理 | 自动接受 |

- **自动接受**：超出部分也入商户数币钱包，商户自行处理退款（本期 EX 不支持自动退款）
- **转人工处理**：订单标记为异常，由 SA 运营介入处理

## 4. 订单状态流转

```
创建订单
  └→ [待支付 Pending]
        ├→ 收到链上转账（金额足额）→ 通道确认 → [支付成功 Paid]
        │     └→ 结算入商户数币钱包 → [已结算 Settled]
        │
        ├→ 收到链上转账（underpaid，策略=等待补款）→ [部分支付 Partially Paid]
        │     ├→ 有效期内补款凑足 → [支付成功 Paid] → [已结算 Settled]
        │     └→ 有效期到期仍不足 → [已过期 Expired]（已到账资金按 overpaid 规则处理）
        │
        ├→ 收到链上转账（underpaid，策略=自动关单）→ [已关闭 Closed]
        │     └→ 已到账资金按 overpaid 规则处理（自动接受则入商户账户）
        │
        ├→ 收到链上转账（overpaid，策略=自动接受）→ [支付成功 Paid]（超额全部入账）
        │     └→ [已结算 Settled]
        │
        ├→ 收到链上转账（overpaid，策略=转人工）→ [异常 Exception]
        │
        └→ 有效期到期，未收到任何转账 → [已过期 Expired]
```

**异常场景处理：**

| 场景             | 处理方式                                            |
| ---------------- | --------------------------------------------------- |
| 订单超时未支付   | 自动关单，地址失效                                  |
| 通道通知延迟     | 订单保持待支付状态，通道到账通知后更新              |
| Webhook 通知失败 | 自动重试 3 次，超出后标记"通知失败"，商户可主动查询 |
| 结算失败         | 标记异常，SA 人工介入，通知商户                     |

## 6. 分阶段实现计划

### Phase 1（本期）— 核心收单闭环

**目标：** 跑通最小闭环，外接一条聚合通道，支持 USD 计价稳定币收单

```
产品层（固定）：
  ✅ USD 计价，USDT/USDC 支付，支持 TRC-20 / ERC-20 / BEP-20（由通道决定）
  ✅ 每单独立地址，无确认数配置，无预授权
  ✅ 订单有效期：15分钟 ～ 24小时
  ✅ 逐笔结算，同货币同网络，结算入商户数币钱包，不换汇

TP 配置层：
  ✅ 允许的支付币种/网络（可限制）
  ✅ 默认订单有效期（可设置）
  ✅ 商户结算账户指定

商户配置层（商户自配）：
  ✅ underpaid 处理策略：等待补款（产品层默认）/ 自动关单
  ✅ overpaid 处理策略：自动接受（产品层默认）/ 转人工

商户端：
  ✅ MP Portal 收单订单列表（查询）
  ✅ Webhook 通知：支付成功 / 订单过期 / 结算完成 / 部分支付
```

**Phase 1 不做：**

```
  ❌ 自动换汇结算（收数币→换汇→入法币账户）
  ❌ 商户自助退款（overpaid 退款由商户线下处理或 SA 人工介入）
  ❌ 商户自定义订单有效期（P2 开放）
  ❌ 订阅支付 / 预授权
  ❌ 多通道聚合路由
  ❌ 托管支付页（Hosted Page）
```

---

### Phase 2 — 结算增强 + 商户自助

```
结算增强：
  ☐ 自动换汇结算：收 USDT → 自动换汇 → 入商户法币账户（USD）
  ☐ 批次结算：支持按时间批次合并结算（T+1 等）

商户自助配置：
  ☐ 商户自定义订单有效期（在 TP 允许范围内）

退款能力：
  ☐ 商户通过 API/Portal 发起退款（链上退回付款人地址）
  ☐ 支持部分退款

支付体验：
  ☐ 托管支付页（Hosted Page）：EX 提供标准支付页，含二维码 + 倒计时
  ☐ 支付页面多语言支持
```

---

### Phase 3 — 能力扩展

```
多通道：
  ☐ 接入多条链上收单通道，支持路由策略（成本优先 / 速度优先）
  ☐ 通道故障自动切换

支付币种扩展：
  ☐ 支持更多稳定币（USDC 更多网络 / DAI 等）
  ☐ 支持 ETH / BTC 等非稳定币支付（汇率波动风险由商户承担）

高级功能：
  ☐ 订阅支付（Recurring）：周期性自动扣款
  ☐ 预授权：先授权后扣款
  ☐ 买家钱包直连：MetaMask / WalletConnect 等
  ☐ 收单数据分析：转化率、币种分布、网络分布报表
```
