# EX 平台产品属性定义

## 文档概述

本文档定义 EX PPaaS 平台的产品属性，按照平台已定义的产品体系拆分。每个产品区分 **SP 能力定义**（SP 上架时固定）和 **TP 产品配置**（TP 可在 SP 能力范围内调整）。

> 计费相关见 `productfee.md`，汇率相关见 `fx.md`，退款相关见 `refund.md`，风控由独立风控系统管理。本文档不重复这些内容。

**产品体系：**

| 产品代码          | 产品名称           | 说明                   | 依赖关系                                            |
| ----------------- | ------------------ | ---------------------- | --------------------------------------------------- |
| `CRYPTO_WALLET` | 数币钱包           | 充币 + 提币            | 基础产品，开通任何数币相关产品时自动开通            |
| `FIAT_ACCOUNT`  | 法币账户           | 同名转入（充值）+ 提现 | 基础产品，按 SP 独立开通                            |
| `ON_RAMP`       | 入金（法币→数币） | 法币承兑为数币         | 依赖数币钱包；VA 能力来自法币账户的充值能力         |
| `OFF_RAMP`      | 出金（数币→法币） | 数币承兑为法币         | 依赖数币钱包；出款/收款人能力来自法币账户的提现能力 |

**产品能力继承关系：**

```
┌─────────────────────────────────────────────────────────────────┐
│                       产品能力继承                                │
│                                                                  │
│  法币账户 (FIAT_ACCOUNT)                                         │
│  ├── 充值能力（同名转入）                                         │
│  │   ├── VA 同名充值（币种/地区/银行）                            │
│  │   └── 代收付账户充值                                           │
│  │       ↓ 继承到                                                │
│  │       OnRamp 预约单的法币入金方式                              │
│  │                                                               │
│  └── 提现能力（出款）                                             │
│      ├── 支持的出款方式（SWIFT/本地转账/SEPA）                    │
│      ├── 支持的出款地区                                           │
│      └── 收款人管理能力                                           │
│          ↓ 继承到                                                │
│          OffRamp 承兑后出款到收款人的能力                          │
│                                                                  │
│  数币钱包 (CRYPTO_WALLET)                                        │
│  ├── 充币能力（链上接收）                                         │
│  │   ├── 支持的币种/网络                                         │
│  │   └── 充币地址管理                                             │
│  └── 提币能力（链上发送）                                         │
│      ├── 支持的币种/网络                                         │
│      └── 提币地址管理                                             │
│          ↓ 继承到                                                │
│          OffRamp 承兑后提币到数币地址的能力                        │
└─────────────────────────────────────────────────────────────────┘
```

**配置层级（本期2层）：**

```
第1层：SP 能力定义（SP 上架时填写，不可被 TP/商户修改）
第2层：TP 产品配置（TP 签约后配置，可对商户做特殊配置；本期商户级也由 TP 代配）
下期：第3层 商户自助配置
```

### 3.1 SP 能力定义

| 产品状态 | 下拉 | 上架状态       | 启用 / 停用        |

| -------------- | ---- | -------------------- | ------------------------------- |

| 支持的数币币种 | 多选 | 承兑的目标数币       | USDT, USDC                      |
| 支持的货币对   | 列表 | 允许的法币→数币组合 | USD→USDT, USD→USDC, HKD→USDT |

#### 3.1.2 交易模式

| 字段       | 类型 | 说明             | 示例值 |
| ---------- | ---- | ---------------- | ------ |
| 支持即时单 | 开关 | 法币余额直接承兑 | 是     |
| 支持预约单 | 开关 | 先下单后汇款     | 是     |

#### 3.1.3 法币入金方式（继承自法币账户充值能力）

```
OnRamp 的法币入金方式不在 OnRamp 产品自身定义，
而是继承商户已开通的法币账户(FIAT_ACCOUNT)的充值能力：

即时单：
  → 使用法币账户余额直接扣款（无需外部入金）

预约单：
  → 法币入金方式 = 商户法币账户的充值能力
  → 如果商户开了 BB 法币账户 → 可用 BB 的 VA/代收付账户充值
  → 如果商户开了 IPL 法币账户 → 可用 IPL 的 VA 充值
  → 如果都开了 → 两者都可用（VA 排序见法币账户的 VA 配置）
```

> VA 能力的具体币种、地区、银行等属性，由法币账户产品的 2.1.1 充值能力定义。

#### 3.1.4 预约单能力

| 预约单匹配模式   | 多选         | 支持的匹配方式               | 精确匹配 / 容差匹配 / 人工匹配 |
| 容差匹配阈值     | 百分比       | 入账金额与预约金额的允许偏差 | 5%                             |

| 字段         | 类型 | 说明             | 示例值         |
| ------------ | ---- | ---------------- | -------------- |
| 单笔最小金额 | 金额 | 最小承兑金额     | 10 USD         |
| 单笔最大金额 | 金额 | 最大承兑金额     | 1,000,000 USD  |
| 单日最大金额 | 金额 | 每日承兑累计上限 | 5,000,000 USD  |
| 单月最大金额 | 金额 | 每月承兑累计上限 | 50,000,000 USD |

### 3.2 TP 产品配置

| 字段         | 类型         | 说明                        | 约束              | 默认值        |
| ------------ | ------------ | --------------------------- | ----------------- | ------------- |
| 允许的货币对 | 多选         | TP 允许商户使用的承兑货币对 | ⊆ SP支持         | = SP全部      |

| 单月最大金额 | 金额         | TP 限制的每月上限           | ≤ SP值           | = SP值        |
| 预约单有效期 | 数字（小时） | TP 调整的预约单超时         | ≤ SP值           | = SP值（72h） |
| 容差匹配阈值 | 百分比       | TP 调整的入账偏差           | ≤ SP值           | = SP值（5%）  |
| 过期自动处理 | 下拉         | 预约单过期后处理方式        | 自动取消 / 转人工 | 自动取消      |

#### 4.1.2 出款方式（继承自法币账户提现能力）

```
OffRamp 承兑后的出款能力不在 OffRamp 产品自身定义，
而是继承商户已开通的法币账户(FIAT_ACCOUNT)的提现能力：

承兑后出款到收款人：
  → 出款方式 = 商户法币账户的提现能力
  → 如果商户开了 BB 法币账户 → 可用 BB 的出款方式(SWIFT等)
  → 如果商户开了 IPL 法币账户 → 可用 IPL 的出款方式(本地转账等)
  → 如果都开了 → 两者都可用（受限于收款人在各SP的审核状态）

承兑后出款到数币地址：
  → 继承数币钱包(CRYPTO_WALLET)的提币能力

收款人管理：
  → 继承法币账户的收款人管理能力
  → 商户可添加的收款人类型 = 所有已开通SP的收款人能力并集
  → 详见 vaandpayees.md 4.3 节
```

#### 4.1.3 出款模式

| -------- | ---- | -------------------- | --------------------------------------------- |
| 出款模式 | 多选 | 承兑后的出款执行方式 | 先承兑到法币账户再出款 / 数币直接出款到收款人 |

## 5. 产品开通联动规则

```
商户开通 OnRamp 产品时：
  ├── 自动开通：数币钱包(CRYPTO_WALLET) — BB 提供
  ├── 自动开通：BB 法币账户(FIAT_ACCOUNT) — BB 提供
  ├── 可选开通：IPL 法币账户(FIAT_ACCOUNT) — IPL 提供
  └── 自动开通：承兑汇率能力

商户开通 OffRamp 产品时：
  ├── 自动开通：数币钱包(CRYPTO_WALLET) — BB 提供
  ├── 自动开通：BB 法币账户(FIAT_ACCOUNT) — BB 提供
  ├── 可选开通：IPL 法币账户(FIAT_ACCOUNT) — IPL 提供
  └── 自动开通：承兑汇率能力

各产品审核独立：
  BB 审核通过 → BB 数币钱包 + BB 法币账户可用
  IPL 审核通过 → IPL 法币账户可用
  BB 和 IPL 审核互不依赖
```

**OnRamp 可用的法币入金方式取决于法币账户开通情况：**

| 商户法币账户开通  | OnRamp 即时单       | OnRamp 预约单可用入金方式             |
| ----------------- | ------------------- | ------------------------------------- |
| 仅 BB 法币账户    | BB 法币余额扣款     | BB 代收付账户充值                     |
| 仅 IPL 法币账户   | IPL 法币余额扣款    | IPL VA 同名充值                       |
| BB + IPL 法币账户 | BB/IPL 法币余额扣款 | BB 代收付 + IPL VA（VA 排序 BB 优先） |

**OffRamp 可用的出款方式取决于法币账户开通情况：**

| 商户法币账户开通  | OffRamp 可用出款方式       | 收款人可用 SP              |
| ----------------- | -------------------------- | -------------------------- |
| 仅 BB 法币账户    | BB 的出款方式(SWIFT等)     | 仅 BB 审核通过的收款人     |
| 仅 IPL 法币账户   | IPL 的出款方式(本地转账等) | 仅 IPL 审核通过的收款人    |
| BB + IPL 法币账户 | BB + IPL 出款方式并集      | 各收款人按 SP 审核状态决定 |

> 详细入网推送规则见 `productionopening.md`。
