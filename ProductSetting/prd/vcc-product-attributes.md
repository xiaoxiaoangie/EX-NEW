## 产品定位

**VCC（Virtual Card）** 是 EX 平台的虚拟卡发卡产品：

- SP 向商户提供虚拟卡发卡能力，商户可为其用户/业务场景开立虚拟卡
- 用于线上消费、广告投放、订阅支付等场景
- **本期所有配置均在 Bin 维度**，不设项目（Programme）维度
- **本期只做预付卡（Prepaid）**，授信卡（Credit / Deferred Credit）下期再做

**配置层级（本期2层）：**

```
第1层：SP 能力定义（SP 上架时按 Bin 维度填写，不可被 TP/商户修改）
  → 卡 Bin 基本信息 / 卡账户能力 / 交易能力 / 卡限额配置 / 风控能力 / 持卡人要求

第2层：TP 产品配置（TP 签约后配置，可对商户做特殊配置；本期商户级由 TP 代配）
  → Bin 选择 / 转入限额收紧 / 3DS / 交易限额收紧

下期：第3层 商户自助配置
```

## 1. SP 能力定义（Bin 维度）

> SP 上架时定义卡 Bin，一个 SP 可上架多个 Bin，每个 Bin 独立配置以下所有属性。
> **本期不设项目（Programme）维度，所有配置直接挂在 Bin 上。**

### 1.1 卡 Bin 基本信息

| 字段       | 字段名（EN）   | 类型     | 说明                                          | 枚举值 / 示例                    |
| ---------- | -------------- | -------- | --------------------------------------------- | -------------------------------- |
| 卡 Bin     | bin            | 文本     | 6～8 位卡 Bin 号码（当前主流为 8 位）         | 如：53422618                     |
| 卡组织     | card-scheme    | 下拉     | 卡组织标识                                    | Visa / Mastercard                |
| 发卡地区   | region         | 下拉     | 发卡所在国家/地区                             | 美国 / 英国 / 香港 等            |
| 卡账户币种 | card-account ccy | 下拉   | 卡片账户币种                                  | USD / EUR / GBP 等               |
| 卡面 Logo  | logo           | 图片上传 | 卡组织 Logo / 卡面设计                        | —                               |
| 资金模式   | funding_model  | 下拉     | 卡片资金归属模式                              | Programme Managed / Cardholder Managed |
| 卡类别     | programme card type | 下拉 | 企业卡 / 个人卡                               | Corporate Cards / Consumer Cards |

> **卡 Bin 位数说明：** 行业已从 6 位 Bin 过渡到 8 位 Bin（IIN），当前主流为 8 位。系统同时兼容 6 位和 8 位，SP 上架时按实际 Bin 填写。

---

### 1.2 卡账户能力

> 每个 Bin 的卡账户级别属性，定义卡片的类型、有效期、充值能力等。

| 字段             | 字段名（EN）          | 类型 | 说明                                                   | 枚举值 / 示例                    |
| ---------------- | --------------------- | ---- | ------------------------------------------------------ | -------------------------------- |
| 卡有效期         | Card Validity Period  | 文本 | 卡片有效期限                                           | 如：3 years                      |
| 卡类型           | card type             | 下拉 | 卡片类型（**本期仅 Prepaid**）                         | Prepaid / Credit / Deferred Credit |
| 最大授信额度     | credit amount         | 金额 | 单卡最大额度（Prepaid 模式下为最大充值余额上限）       | 如：100,000 USD                  |
| 是否支持共享卡   | is shared             | 下拉 | 是否支持多卡共享一个账户                               | Shared / Not Shared              |
| 转入币种         | recharge ccy-default  | 下拉 | 支持转入/充值的币种                                    | USD / EUR / USDT / USDC 等       |
| 转入限额         | recharge limit-default | 区间 | 单次转入的最小/最大金额                               | 最小～最大                       |

> **本期只考虑 Prepaid 能力**，Credit / Deferred Credit 下期再做。Prepaid 模式下 `credit amount` 表示卡片最大可充值余额上限。

---

### 1.3 交易能力

> 定义卡片的授权模式、浮动规则和 3DS 安全验证。

| 字段             | 字段名（EN）          | 类型     | 说明                                 | 枚举值                                                        |
| ---------------- | --------------------- | -------- | ------------------------------------ | ------------------------------------------------------------- |
| 单商户卡数量限制 | card limit-per merchant | 数字   | 单商户可持有的最大卡数量             | —                                                            |
| 单卡交易次数限制 | card limit-per transaction | 下拉 | 卡片使用次数限制                     | 单次 / 多次 / 不限                                            |
| 使用次数         | allowed_transaction_count | 数字 | 允许的交易次数（单次/有限次时填写）  | —                                                            |
| 授权模式         | authorization_mode    | 下拉     | 交易授权方式                         | Real-time（实时授权）/ Standard（事后同步）                   |
| 授权浮动         | auth-padding enable   | 下拉     | 是否支持授权金额浮动                 | 允许 / 不允许                                                 |
| 授权浮动值       | auth-padding value    | 区间输入 | 浮动比例范围（开启授权浮动时可配置） | 如：5%、10%、15%                                              |
| 3DS 开关         | 3ds-switch            | 开关     | 3DS 验证开关                         | 开启 / 关闭                                                   |
| 3DS 模式         | 3ds model             | 下拉     | 3DS 验证模式（开启时可配置）         | OTP 验证 / Redirect Authentication / Decoupled Authentication |
| 3DS 通知方式     | —                    | 下拉     | 验证通知方式（开启时可配置）         | 直接发给终端用户 / Webhook 通知商户                           |

**联动规则：**

- 授权浮动 = 允许时，授权浮动值必填
- 3DS 开关 = 开启时，3DS 模式和 3DS 通知方式必填

### 1.4 卡限额配置

| 字段         | 字段名（EN）                  | 类型 | 说明                           | 枚举值           |
| ------------ | ----------------------------- | ---- | ------------------------------ | ---------------- |
| 交易受理渠道 | Transaction Acceptance Channel | 多选 | 支持的交易渠道                | ATM / POS / 线上 |
| 单笔限额     | transaction limits-PER_TRANSACTION | 金额 | 单笔交易限额               | —               |
| 单日限额     | transaction limits-daily      | 金额 | 每日交易限额                   | —               |
| 单周限额     | transaction limits-weekly     | 金额 | 每周交易限额                   | —               |
| 单月限额     | transaction limits-monthly    | 金额 | 每月交易限额                   | —               |
| 单年限额     | transaction limits-yearly     | 金额 | 每年交易限额                   | —               |
| 单卡限额     | transaction limits-percard    | 金额 | 单卡累计交易限额               | —               |

**校验规则：**

- 单笔 ≤ 单日 ≤ 单周 ≤ 单月 ≤ 单年 ≤ 单卡（各层级有值时才参与校验）

### 1.5 风控能力

> 按 Bin 维度配置 MCC 和地区限制规则。

| 字段           | 字段名（EN）    | 类型 | 说明                                 | 示例                          |
| -------------- | --------------- | ---- | ------------------------------------ | ----------------------------- |
| MCC 白名单     | MCC WHITELIST   | 多选 | 允许消费的商户类别码（为空=不限制）  | 5411, 5812, 7311              |
| MCC 黑名单     | MCC BLACKLIST   | 多选 | 禁止消费的商户类别码                 | 7995（赌博）, 6012（金融）    |
| 支持业务地区   | ACCEPTED REGION | 多选 | 允许交易的国家/地区（为空=不限制）   | US, GB, HK, SG                |
| 不支持业务地区 | RESTRICTED REGION | 多选 | 禁止交易的国家/地区                | IR, KP, CU                    |

**规则说明：**

- MCC 白名单和黑名单互斥：配置了白名单则只允许白名单内 MCC，配置了黑名单则排除黑名单内 MCC
- 地区同理：ACCEPTED REGION 和 RESTRICTED REGION 互斥
- 两者都为空时表示不做限制

---

### 1.6 持卡人要求

> 定义该 Bin 是否需要持卡人信息，以及持卡人的类型和地区限制。

| 字段         | 字段名（EN）      | 类型 | 说明                                   | 枚举值                     |
| ------------ | ----------------- | ---- | -------------------------------------- | -------------------------- |
| 是否需要持卡人 | cardholder needed | 下拉 | 开卡时是否必须绑定持卡人信息           | Yes / No                   |
| 持卡人类型   | cardholder 类型   | 下拉 | 持卡人身份类型（需要持卡人时可配置）   | Individual / Enterprise    |
| 持卡人地区   | holder regions    | 多选 | 允许的持卡人所在地区（需要持卡人时可配置） | US, GB, HK, SG 等         |

**联动规则：**

- cardholder needed = Yes 时，持卡人类型和持卡人地区必填
- cardholder needed = No 时，持卡人类型和持卡人地区不可配置

## 2. TP 产品配置

> TP 在 SP 能力范围内收紧，不可超出 SP 上限。

### 2.1 卡 Bin 选择

| 字段          | 类型 | 说明                     | 约束             | 默认值     |
| ------------- | ---- | ------------------------ | ---------------- | ---------- |
| 可用 Bin 列表 | 多选 | TP 允许商户使用的 Bin    | ⊆ SP 已上架 Bin | = SP 全部  |
| 默认 Bin      | 单选 | 商户开卡时默认使用的 Bin | ∈ 可用 Bin 列表 | 第一个 Bin |

### 2.2 资金与充值

| 字段             | 类型 | 说明                    | 约束       | 默认值    |
| ---------------- | ---- | ----------------------- | ---------- | --------- |
| 允许的转入币种   | 多选 | TP 允许商户转入的币种   | ⊆ SP 支持 | = SP 全部 |
| 最大转入限额     | 金额 | TP 限制的单次转入上限   | ≤ SP 值   | = SP 值   |
| 单商户卡数量限制 | 数字 | TP 限制的单商户最大卡数 | ≤ SP 值   | = SP 值   |

### 2.3 授权与 3DS

| 字段     | 类型 | 说明                | 约束              | 默认值  |
| -------- | ---- | ------------------- | ----------------- | ------- |
| 3DS 开关 | 开关 | TP 是否强制开启 3DS | SP 支持时才可开启 | 跟随 SP |
| 3DS 模式 | 下拉 | TP 指定的 3DS 模式  | ⊆ SP 支持        | = SP 值 |

### 2.4 交易限额

| 字段         | 类型 | 说明                  | 约束       | 默认值    |
| ------------ | ---- | --------------------- | ---------- | --------- |
| 交易受理渠道 | 多选 | TP 允许商户使用的渠道 | ⊆ SP 支持 | = SP 全部 |
| 最大单笔限额 | 金额 | TP 限制的单笔上限     | ≤ SP 值   | = SP 值   |
| 单日限额     | 金额 | TP 限制的每日上限     | ≤ SP 值   | = SP 值   |
| 单月限额     | 金额 | TP 限制的每月上限     | ≤ SP 值   | = SP 值   |

## 3. 资金与余额模型

> 本章说明两种资金模式的区别，本期仅支持 Prepaid（预付）。

### 3.1 资金模式对比

| 维度         | Programme Managed（项目管理）                  | Cardholder Managed（持卡人管理）               |
| ------------ | ---------------------------------------------- | ---------------------------------------------- |
| 资金归属     | 资金归属于 Programme（商户/TP）                 | 资金归属于持卡人个人                           |
| 充值方式     | 商户从法币账户向卡片充值                       | 持卡人自行充值（或商户代充）                   |
| 余额控制     | 商户统一管理所有卡片余额                       | 持卡人自行管理余额                             |
| 适用场景     | 企业费用管理、广告投放、采购卡                 | 个人消费卡、员工福利卡                         |
| 退款流向     | 退款回到商户法币账户                           | 退款回到持卡人卡片余额                         |
| 本期支持     | ✅                                             | ✅                                             |

### 3.2 本期资金流向（Prepaid 模式）

```
Programme Managed：
  商户法币账户余额 → 卡片余额（商户充值）
  卡片退款 → 商户法币账户余额

Cardholder Managed：
  持卡人充值 → 卡片余额
  卡片退款 → 卡片余额
```

### 3.3 下期扩展（Credit 模式）

```
下期规划：
  ☐ Credit（信用卡）：先消费后还款，有账单周期和还款日
  ☐ Deferred Credit（延期信用卡）：类似信用卡但无循环信用
  ☐ 授信额度管理：额度审批、额度调整、临时额度
```

## 4. 产品开通联动规则

```
商户开通 VCC 产品时：
  ├── 依赖：法币账户（FIAT_ACCOUNT）— 用于卡片充值/余额转入
  ├── 可选：数币钱包（CRYPTO_WALLET）— 支持 USDT/USDC 转入卡片时需要
  └── 自动开通：卡片管理能力（开卡/销卡/冻结/解冻）

VCC 与其他产品的资金流向：
  充值到卡：法币账户余额 → VCC 卡片余额（Programme Managed）
  数币充值：USDT/USDC → 自动换汇 → VCC 卡片余额（需 OnRamp 能力，下期）
  退款到账：VCC 卡片退款 → 法币账户余额 / 卡片余额（取决于资金模式）
```

## 6. 本期 vs 下期

### 本期实现（Phase 1）

```
✅ 所有配置在 Bin 维度，不设项目（Programme）维度
✅ 卡 Bin 基本信息：8 位 Bin 为主（兼容 6 位）、卡组织、地区、币种、资金模式、卡类别
✅ 卡类型：仅 Prepaid（预付卡）
✅ 资金模式：Programme Managed / Cardholder Managed
✅ 卡账户能力：有效期、共享卡、转入币种、转入限额
✅ 交易能力：授权模式、授权浮动、3DS
✅ 卡限额配置：单笔/单日/单周/单月/单年/单卡
✅ 风控能力：MCC 白/黑名单、地区白/黑名单
✅ 持卡人要求：是否需要、类型、地区
✅ TP 产品配置：Bin 选择、转入限额收紧、3DS、交易限额收紧
✅ TP 代配商户级：商户级限额由 TP 在 TP Portal 配置
✅ 与法币账户联动：充值/退款通过法币账户流转
```

### 下期规划（Phase 2）

```
☐ 授信卡类型：Credit / Deferred Credit，含授信额度管理
☐ 项目（Programme）维度：支持在 Bin 之上按项目分组管理
☐ 商户自助配置层（第3层）：商户在 MP Portal 自行配置卡片偏好
☐ 数币直充卡片：USDT/USDC 直接充值 VCC，无需先换汇
☐ 实体卡支持：卡形态扩展到实体卡，支持物流配送
☐ 卡片消费分析：商户端消费明细分析报表
☐ TP 可配 MCC / 地区限制：TP 在 SP 范围内收紧风控规则
☐ 动态 CVV：支持动态 CVV 安全机制
```
