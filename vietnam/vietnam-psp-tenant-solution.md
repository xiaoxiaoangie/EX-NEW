# 越南本地 PSP 租户跨境支付解决方案

> 文档类型：PSP 租户专项解决方案 / 讨论稿  
> 适用市场：越南  
> EX 客户：越南本地 PSP / 持牌 MSB / 支付机构 / 本地收付服务商，即 EX 租户  
> 租户客户：电商卖家、独立站商户、TikTok Shop / Shopee / Amazon 卖家、出口商、贸易商、广告代理、供应链商户  
> 暂不包含：沙盒环境 PSP / 加密沙盒机构 / 零售 On-Off Ramp 沙盒主体  
> 关联文档：`ProductArchetect/ABE-positioning-strategy.MD`、`ProductArchetect/ex-three-layer-solution.md`、`vietnam/ex-vietnam-sp-recruitment.md`

---

## 0. TL;DR

本方案中的 **EX 客户 = 租户 = 越南当地 PSP / 持牌 MSB / 持牌支付机构**。  
租户的客户不是 EX 直接客户，而是 PSP 自己服务的商户，例如电商卖家、独立站卖家、出口商、广告代理、供应链商户等。

越南 PSP 方案的产品主张：

> 越南当地 PSP / 持牌 MSB 保留本地品牌、本地商户、牌照主体和本地收付能力；EX/A/B 向其输出全球 VA、中国收付、跨境结算、发卡、风控、对账和白牌/API 系统，让其可以基于自身牌照和客户资源服务跨境商户。

本阶段暂不把沙盒环境 PSP、加密沙盒机构、零售稳定币支付沙盒主体作为主要目标对象。  
本阶段的重点是：**本地 PSP / 持牌 MSB 作为租户，服务其电商卖家和跨境贸易商户。**

## 1. 角色定义

### 1.1 四层角色

| 层级 | 角色 | 在本方案中的定义 | 是否 EX 直接客户 |
| --- | --- | --- | --- |
| L1 | 商户 | PSP 的客户，例如电商卖家、独立站商户、出口商、广告代理 | 否 |
| L2 | 租户 / TP | 越南当地 PSP、持牌 MSB、持牌支付机构，以自己的品牌和牌照能力服务商户 | 是 |
| L3 | SP / 资金服务主体 | A、B、发卡 SP、其他跨境能力方 | 视能力而定 |
| L4 | 渠道 | 银行、VA、卡 BIN、PSP 底层通道、中国收付渠道 | 否 |

### 1.2 关键口径

```text
EX 客户 = 租户 = 越南当地 PSP / 持牌 MSB / 持牌支付机构
租户客户 = PSP 的商户 = 电商卖家 / 出口商 / 独立站 / 广告代理等
EX 不直接抢 PSP 的本地商户
PSP/MSB 用自己的品牌和牌照能力向商户提供跨境收付、发卡、VA 等能力
EX/A/B 在后端提供系统和能力
```

### 1.3 与 OTC 方案的边界

持牌 MSB 不放在 OTC 方案中：

- 只要 MSB 或支付机构使用牌照对外展业、承接资金服务责任，就应作为 PSP/持牌租户处理；
- 这种客户的核心需求不是“托管给 A/B 判断能不能做”，而是把 EX/A/B 的跨境 VA、中国收付、发卡、对账和风控能力嵌入自己的持牌展业体系；
- 空有牌照但不使用牌照展业的“屯牌照”主体，不因为持牌就自动成为 PSP 租户，也不作为 OTC 核心客户；应按其真实业务模式另行判断。

### 1.4 暂不纳入的对象

本方案暂不把以下对象作为主要目标：

- 沙盒环境 PSP；
- 加密资产沙盒机构；
- 面向零售用户的稳定币支付试点主体；
- 以 C 端加密钱包为主的 Web3 钱包；
- 以零售 On/Off Ramp 为核心诉求的机构。

原因：

- 当前方案先聚焦可标准化、可商务化的 PSP 租户；
- PSP 租户的客户是商户，业务更接近跨境电商、出口收款、广告支付和贸易结算；
- 沙盒类机构涉及监管边界、试点范围和零售数币场景，后续可作为独立方案讨论。

## 2. 租户画像：越南当地 PSP / 持牌 MSB

### 2.1 PSP / MSB 租户典型特征

| 维度 | 特征 |
| --- | --- |
| 牌照/主体 | 有本地 PSP、MSB、支付机构或类似资金服务资质，并实际使用牌照展业 |
| 本地能力 | 有 VND 本地收款、付款、银行关系、商户服务经验，或有持牌资金服务能力 |
| 客户资源 | 掌握本地电商卖家、出口商、贸易商、代理商 |
| 品牌诉求 | 希望用自己的品牌服务客户，不希望客户被 EX 直接接管 |
| 能力缺口 | 缺海外 VA、中国收付、发卡、多币种结算和跨境风控 |
| 系统诉求 | 需要白牌门户、API、报表、对账、商户管理 |
| 商业诉求 | 希望通过跨境能力获得新增收入，而不是只做本地低毛利收付 |

### 2.2 PSP / MSB 租户的痛点

| 痛点 | 表现 | 方案切入 |
| --- | --- | --- |
| 缺海外收款账户 | 商户需要收 USD/EUR/GBP/HKD/SGD/CNY，但 PSP 本地账户覆盖不足 | 多国 VA |
| 缺中国收付能力 | 商户需要收中国买家款或付中国供应商 | 中国收款/付款通道 |
| 缺发卡能力 | 独立站卖家需要广告卡、SaaS 订阅卡、平台支付卡 | VCC 发卡 |
| 缺跨境风控 | 不熟悉跨境商户 KYB、贸易资料审核、卡交易风险 | 风控托管和规则 |
| 对账复杂 | 多币种、多国家、多渠道交易难对账 | 统一账本和报表 |
| 系统建设成本高 | 自建白牌和 API 成本高、周期长 | EX 白牌/API |
| 本地利润薄 | VND 本地收付费率低，增长空间有限 | 跨境 VA、FX、发卡、付款增值收入 |

- 收海外买家 USD/EUR/GBP；

## 4. 方案定位

### 4.1 对 PSP / MSB 租户的定位

EX 对越南 PSP / MSB 的定位应为：

> 跨境支付能力 SaaS + 多国资金能力 + 中国通道 + 发卡能力 + 统一运营后台。

PSP / MSB 获得：

- 可用自己品牌展业的白牌门户；
- 可嵌入自有系统的 API；
- 可向商户销售的多国 VA；
- 可向商户销售的中国收付能力；
- 可向电商商户销售的发卡能力；
- 可运营商户的风控、对账、报表和分润后台。

### 4.2 A/B/E 分工

| 角色 | 职责 |
| --- | --- |
| **EX** | 租户系统、白牌/API、商户管理、账本、路由、对账、报表、权限、分润 |
| **A / iPayLinks** | 多国 VA、法币收付、POBO、FX、中国通道、多币种结算 |
| **B / BlancBlock** | 暂作为数币能力储备和特定 B2B USDT 结算能力，不作为本方案主线 |
| **越南 PSP / MSB 租户** | 本地商户获取、基于自身牌照展业、本地 KYC/KYB 协同、本地 VND 收付、本地客服和运营 |
| **发卡 SP** | BIN、VCC 开卡、卡交易处理、卡组织规则、拒付处理 |

### 4.3 本阶段产品边界

本方案优先做：

- 海外 VA；
- 中国收款；
- 中国付款；
- 多币种结算；
- 发卡；
- 商户管理；
- 对账报表；
- 白牌门户/API。

本方案暂不主推：

- 零售稳定币支付；
- 沙盒 On/Off Ramp；
- C 端钱包；
- 个人用户加密消费；
- 主权稳定币叙事；
- 越南本地闭环支付替代。

## 6. 产品功能

### 6.1 租户门户

PSP 租户需要一个自己的管理后台：

- 租户信息；
- 品牌配置；
- 用户和权限；
- 商户列表；
- 产品开通状态；
- 交易查询；
- 余额查询；
- 结算管理；
- 对账报表；
- 费率配置；
- 分润报表；
- API Key 管理；
- Webhook 配置。

### 6.2 商户门户

PSP 的商户可以使用白牌商户门户：

- 提交 KYB；
- 查看 VA；
- 查看收款流水；
- 发起付款；
- 发起换汇；
- 申请发卡；
- 查看卡交易；
- 下载对账报表；
- 查看手续费；
- 管理收款人和供应商；
- 查看结算状态。

### 6.3 API 能力

PSP 若已有自己的系统，需要 API 接入：

| API | 说明 |
| --- | --- |
| 商户 API | 创建商户、提交 KYB、查询审核状态 |
| VA API | 申请 VA、查询 VA、关闭 VA |
| 收款 API | 查询入账、收款状态、流水 |
| 付款 API | 创建付款、查询付款状态、取消付款 |
| FX API | 查询汇率、锁价、换汇 |
| 发卡 API | 开卡、充值、冻结、销卡、查询交易 |
| 报表 API | 交易报表、余额报表、费用报表 |
| Webhook | 入账通知、付款状态、卡交易、审核状态 |

### 6.4 商户进件与 KYB

商户进件字段：

- 公司名称；
- 注册国家/地区；
- 注册证书；
- 董事/法人；
- UBO；
- 业务网址；
- 平台店铺链接；
- 行业；
- 商品/服务说明；
- 预计交易量；
- 预计收款国家；
- 预计付款国家；
- 用途说明；
- 银行账户；
- 联系人。

审核结果：

- 通过；
- 补充材料；
- 拒绝；
- 限制产品；
- 限制额度；
- 人工复核。

### 6.5 产品开通

商户不是一次性获得全部能力，而是按产品开通：

| 产品 | 开通条件 |
| --- | --- |
| 海外 VA | KYB 通过，业务国家和行业可接受 |
| 中国收款 | 贸易背景或电商背景可解释 |
| 中国付款 | 收款人和付款用途审核通过 |
| FX | 币种和交易用途符合要求 |
| 发卡 | 电商/广告/SaaS 等用途明确，风险行业排除 |
| 本地 VND 结算 | PSP 本地账户和商户银行账户校验通过 |

## 7. 渠道能力

### 7.1 A 提供的法币跨境能力

| 能力 | 说明 | PSP 可销售场景 |
| --- | --- | --- |
| 多国 VA | USD/EUR/GBP/HKD/SGD/CNY 等收款账户 | 出口收款、平台收款 |
| Collection | 接收海外买家或平台付款 | 电商、外贸 |
| Payout / POBO | 向供应商、服务商、平台付款 | 中国供应商付款、海外服务费 |
| FX | 多币种换汇 | USD/EUR 到 VND 或 CNY |
| 中国收付 | 中国本地或跨境收付款能力 | 中越贸易、电商采购 |
| 对账 | 交易、流水、费用、汇率报表 | PSP 财务和商户财务 |

### 7.2 PSP 自有本地能力

PSP 租户自己应提供或协同提供：

- VND 本地收款；
- VND 本地付款；
- 本地商户 KYC/KYB 协同；
- 本地客服；
- 本地银行关系；
- 本地结算；
- 本地税务/发票协同；
- 本地商户触达和运营。

### 7.3 发卡渠道能力

发卡能力需要：

- BIN 或发卡合作方；
- 开卡；
- 卡充值；
- 卡冻结/解冻；
- 销卡；
- 单卡限额；
- 商户 MCC 限制；
- 国家/地区限制；
- 卡交易流水；
- 拒付和退款；
- 账单报表；
- 风控规则。

### 7.4 渠道路由原则

```text
海外收款：
优先走 A 的多国 VA。

越南本地结算：
优先走 PSP 自有 VND 通道。

中国付款：
优先走 A 的中国收付/POBO 能力。

发卡：
走发卡 SP，EX 做产品和账本编排。

本方案暂不以沙盒 On/Off Ramp 为主路由。
```

## 8. 业务场景

### 8.1 越南电商卖家收海外平台款

```text
越南电商卖家
→ 在 PSP 白牌门户申请海外 VA
→ 平台或海外买家付款至 VA
→ A 入账
→ EX 通知 PSP 和商户
→ 商户选择保留外币或换汇
→ PSP 通过本地通道结算 VND 给商户
→ EX 生成对账报表和分润报表
```

### 8.2 越南商户付款给中国供应商

```text
越南商户提交付款申请
→ 上传供应商信息、合同/发票/订单资料
→ PSP 初审
→ EX/A 做付款用途和通道校验
→ 商户使用账户余额或本地 VND 入金
→ A 执行中国付款
→ 付款状态回传 PSP 和商户
→ 报表归档
```

### 8.3 独立站卖家发卡用于广告投放

```text
独立站卖家完成 KYB
→ 申请 VCC 产品
→ PSP/EX 审核用途
→ 商户充值或使用可用余额
→ 发卡 SP 开卡
→ 商户用于广告平台、SaaS、店铺工具支付
→ 卡交易回传 EX
→ PSP 和商户查看卡账单
```

### 8.4 出口商收欧美买家款并结算 VND

```text
出口商通过 PSP 申请 USD/EUR VA
→ 欧美买家付款
→ A 入账
→ 出口商提交结算申请
→ A 完成 FX
→ PSP 本地 VND 通道落地
→ 商户下载收款、换汇、结算报表
```

## 9. 账户与余额模型

### 9.1 账户结构

```text
PSP 租户账户
    ├─ 商户 A
    │   ├─ USD VA
    │   ├─ EUR VA
    │   ├─ VND 结算账户
    │   └─ VCC 卡账户
    ├─ 商户 B
    │   ├─ USD VA
    │   ├─ CNY 收款能力
    │   └─ 中国付款收款人
    └─ 商户 C
        ├─ 发卡账户
        └─ 广告卡列表
```

### 9.2 余额展示规则

- A 作为资金主体的海外 VA 余额，不与 PSP 本地 VND 余额混同；
- PSP 本地 VND 余额由 PSP 或其本地资金主体承接；
- 发卡余额按发卡 SP 的资金规则展示；
- 商户前台可以看到统一视图，但后台必须按资金主体分账；
- 不同资金主体之间的结算必须有交易记录和凭证。

### 9.3 结算方式

| 结算方式 | 说明 |
| --- | --- |
| T+N 结算 | A 入账后按周期与 PSP 或商户结算 |
| 按单结算 | 单笔收款后商户主动申请结算 |
| 批量结算 | 多笔收款合并结算至 VND |
| 余额留存 | 商户保留 USD/EUR 等外币余额，用于后续付款或发卡 |
| 换汇结算 | 商户选择汇率后换汇至 VND |

## 10. 风控与合规

### 10.1 PSP 与 EX/A 的责任分工

| 环节 | PSP 租户 | EX/A/B |
| --- | --- | --- |
| 本地商户触达 | 负责 | 不直接抢客 |
| 本地 KYC/KYB 初审 | 负责或协同 | 提供标准和复核 |
| 海外 VA 开通审核 | 协同 | A 负责最终准入 |
| 中国付款用途审核 | 提交材料和初审 | A 做通道和合规校验 |
| 发卡用途审核 | 初审商户用途 | 发卡 SP/EX 复核 |
| 本地 VND 结算 | PSP 负责 | EX 记录和对账 |
| 跨境交易监控 | 协同 | EX/A 规则和报表 |

### 10.2 风控能力

- 商户 KYB；
- UBO 穿透；
- 行业风险识别；
- 网站/店铺审核；
- 交易用途审核；
- 收款人校验；
- 付款用途材料留痕；
- 制裁名单筛查；
- 高风险国家和行业限制；
- 发卡交易风控；
- 异常交易监控；
- 拒付和退款监控。

### 10.3 不做的边界

- 不为 PSP 提供规避外汇管制的方案；
- 不服务无法解释贸易或电商背景的资金流；
- 不做无牌零售加密兑换；
- 不把沙盒稳定币支付作为本方案主线；
- 不把 EX 包装成本地持牌支付主体；
- 不把 PSP 的本地客户直接迁成 EX 直客。

### 12.2 PSP 可对商户收费

### 12.3 分润建议

| 产品 | 主要成本方 | PSP 收入逻辑 | EX/A/B 收入逻辑 |
| --- | --- | --- | --- |

| 报表/API | EX | SaaS 转售 | SaaS 月费 |

## 13. 分期落地

### 一期：PSP 租户基础版

目标：让 PSP 可以用白牌/API 向商户提供海外 VA 和基础收付款。

范围：

- PSP 租户后台；
- 商户进件；
- 多国 VA；
- 入账通知；
- 商户收款报表；
- 基础结算报表；
- PSP 分润报表；
- API Key 和 Webhook。

里程碑：

- 接入 1 家越南 PSP 租户；
- 跑通商户申请 VA；
- 跑通海外收款入账；
- 跑通 PSP/商户报表。

### 二期：中国收付 + FX

目标：服务越南商户与中国供应链之间的收付需求。

范围：

- 中国收款；
- 中国付款；
- 收款人管理；
- 付款用途审核；
- 贸易资料上传；
- FX 报价和锁价；
- VND 结算协同。

里程碑：

- 跑通越南商户付款给中国供应商；
- 跑通中国买家付款给越南商户；
- 形成付款用途审核 SOP。

### 三期：发卡

目标：服务电商独立站和广告投放客户。

范围：

- VCC 申请；
- 开卡；
- 充值；
- 冻结/解冻；
- 销卡；
- 卡交易查询；
- 卡交易报表；
- MCC/国家/额度控制；
- 拒付和退款处理。

里程碑：

- PSP 可向电商商户销售发卡产品；
- 商户可在 PSP 门户管理卡和账单；
- 发卡交易进入 EX 对账体系。

### 四期：资金运营和增值服务

目标：从通道能力升级为商户资金运营平台。

范围：

- 商户资金 ERP；
- 多产品余额视图；
- 结算周期配置；
- 资金预测；
- 提前结算试点；
- 风控报表产品化；
- 商户经营画像。

里程碑：

- PSP 可向商户提供收款 + 付款 + 发卡 + 对账的一体化服务；
- EX 沉淀可用于风控和金融产品的商户数据。
