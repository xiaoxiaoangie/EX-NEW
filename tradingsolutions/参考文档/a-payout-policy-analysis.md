# A 出金调单政策（2025-12-04 版）特点分析

> 基于 A 实际出金政策文件提炼的核心规律，用于指导 B-T-A 架构下出金链路设计、客户分流、材料前置准备。
>
> 不复述政策细节（细节看原文档），只提炼"A 的出金调单逻辑特征 + 对我方的影响"。

---

## 一、A 出金调单的 9 个核心特征

### 特征 1：**收款人类型分类系统**（出金调单的根逻辑）

A 把所有出金按"收款人是谁"分成 7 大类，每类不同材料要求：

| 类型 | 关系 | 材料强度 |
|----|----|-----|
| 提现 | 商户自身（对公 / 法人 / 董事 / ≥25% UBO 对私）| 极低 |
| RELATE | 关联公司 | 中（股权 / 亲属证明）|
| GOODS / CARGO / WARE / LOGS / ADVES | 真实供应商 / 货代 / 仓储 / 物流 / 广告 | **极高**（合同+发票+三流）|
| EMPL | 员工 | 中（劳动合同 / 社保）|
| PAYEE | 关联个人（亲属 / 小股东 / 关联公司股东）| 低（关系证明）|
| OTHER | 其他服务商 | 高（合同 + 资质 + 白名单场景）|

> **实战启示**：客户出金前先帮其"分类"，再准备对应材料包。错分类 = 材料不匹配 = 必被退回。

---

### 特征 2：**OTHER 是白名单制不是黑名单制**（容易踩的坑）

OTHER 其他服务商**不是任何场景都接**，A 明确列出支持场景：

- **外币**：佣金、税务申报、保险费、代运营、技术服务、律师服务、产地证 / 商品认证 / 检验检疫、经营相关的房租 / 物业 / 水电煤、亚马逊官方收款等
- **CNY 更窄**：仅佣金 / 代运营 / 研发设计 / 软件应用 / 国内企业派遣海外员工劳务费，**其他场景暂不支持**

> **实战启示**：业务类型必须能映射到白名单。映射不上的 **不是补材料能解决的**，必须 ① 换收款人类型 ② 换币种 ③ 拆解业务 ④ 走 T 桥接出款。

---

### 特征 3：**反"过桥"硬约束**（最容易被忽略的红线）

提现场景明确写："**不支持同名外币账户划转入账后又出金到同名外币账户**"。

> **实战启示**：同名进同名出的"过桥模式"直接禁。A 知道这是常见的过桥手法，规则就是防这个。
> - 客户 001 在 A 同名 VA 收款 → 不能再同名外币转出去
> - 必须是"实质业务出金"（付供应商 / 付员工 / 付服务商）

这条对 B-T-A 架构关键：**T → 客户 A 自名账户 → 客户同名外币转出去** 这条路是被禁的。客户 A 自名账户的钱只能付向真实业务方。

---

### 特征 4：**GOODS 类材料深度最深**（贸易出金的"标配"）

GOODS 供应商出金要求三层材料：

1. **采购合同**（含双方主体 / 产品 / 金额数量）
2. **行业证明**（外销网址 / 报关单 / 物流单据 / 增值税发票 / 出口许可证）
3. **报关单 / 增值税发票 / 出口许可证**（HK 企业可用大陆关联公司的材料）

且报关单要 **号可查 + 产品匹配 + 收发货人至少一方匹配供应商或其关联公司**。

> **实战启示**：
> - **HK 客户没大陆关联公司 = GOODS 出金过不了**，需要走 T 介入提供贸易链
> - 客户的"假供应商"必须经得起报关单查证 = T 协助伪造材料是高风险（穿透即破）
> - **GOODS 类是 A 调单最深、最难造材料的一类** → 能避开就避开（用 OTHER 服务费等替代）

---

### 特征 5：**佣金有硬比例约束**（唯一定量规则）

OTHER-佣金：**返佣金额 ≤ 订单金额的 15%**。

且要求：佣金协议 + 该笔佣金对应的订单合同 + 关于佣金详情的沟通截图（时间需前置 / 内容涉及佣金比例 / 费用构成）。

> **实战启示**：
> - 用"佣金"作为出金理由的客户，单笔金额受订单合同金额硬约束
> - 沟通截图要求 = A 在查"是否是事后伪造的佣金安排"
> - **大额出金不能用"纯佣金"包装**，必须有真实订单做底

---

### 特征 6：**国家分层管理**（地理风控）

| 类型 | 地区 | 处理 |
|---|---|---|
| 出金禁止国 | 全部出金禁止 | 拒 |
| 不支持出金 | 科特迪瓦、伊拉克、埃塞俄比亚、黎巴嫩（即使银行允许，A 也拒）| 拒 |
| 极高风险国家 | 收款人为个人 | 需个人商业背景材料；**佣金不支持** |

> **实战启示**：
> - 客户 BD 阶段就要先筛收款国 / 收款人类型
> - 极高风险地区 + 个人 + 佣金 = 三重叠加直接拒，无解
> - **B 客户群里如果有这些地区收款需求，A 通道完全不可用，必须 B 端自己解决**

---

### 特征 7：**事后调单常态化**

明文写："后续出金调单：不论出金的角色类型，**必要时**配合提供商户与收款人之间的物流单据 + 合同，或其他可证明双方存在实际贸易关系的材料。"

> **实战启示**：
> - **出金审核通过 ≠ 永久过关**，A 随时可回调单
> - 客户必须 **持续留存** 与收款人的真实业务凭证（合同 / 物流 / 发票），最少 6 年
> - T 协助包装的，留底必须长期可调取

---

### 特征 8：**收款人形态硬约束**

| 收款人形态 | 允许 |
|----|----|
| 商户自身对公 | ✓（提现）|
| 法人 / 董事 / ≥25% UBO 对私 | ✓（提现）|
| 关联个人（亲属 / 小股东）| ✓（PAYEE）|
| **本公司 < 25% 股东** | ✗（无法用 PAYEE 出金）|
| 无关联第三方个人 | ✗ |
| 无关联第三方对公 | ✓（但必须落到 GOODS/OTHER 等场景，且材料齐）|

> **实战启示**：
> - 客户的 UBO / 股权结构会影响"可出金范围"
> - 小股东出金路径窄
> - **个人出金天然受限**，主要靠对公出金 + 关联人转

---

### 特征 9：**合同形式 + 签章 + 主体匹配的"双方"原则**

- **形式**：合同 / 协议 / 对账单均可，但必须含双方主体 + 产品 / 服务 + 费用构成
- **签章**：企业必须公章 + 收款方个人必须签字
- **主体**：收款方为个人时，可个人签合同；但收款方为供应商 / 物流 / 仓储 / 货代 / 广告 / 其他服务商时，**必须配合提供合同**

> **实战启示**：
> - 客户准备"贸易合同"时不能光有合同正文，签章必须实化
> - "对账单"是合规替代品（当没有正式合同时）
> - 收款人是个人 → 个人签字版合同要预先准备

## 二、对 B-T-A 架构的具体启示

### 2.1 客户分类引擎（出金前置）

每个 B 客户的出金需求做这个分类决策树：

```
[客户的出金需求]
        │
        ▼
   收款人是谁？
        │
   ┌────┴────┐
   ▼         ▼
[自己]    [非自己]
   │         │
   │   关联人？─ 是 ─► PAYEE 路径
   │         │
   │      关系类型？
   │      ┌────┬────┬────┬────┐
   │      ▼    ▼    ▼    ▼    ▼
   │   供应商 物流  员工 服务商 其他
   │   GOODS  C/W/L EMPL OTHER  
   │                       │
   │                  能映射白名单？
   │                  ┌────┴────┐
   │                  ▼         ▼
   │                 是→走OTHER  否→走桥接 T
   ▼
[提现]
   │ 同名提现 vs 关联人对私
   │ 检查"反过桥"规则
   ▼
   过 / 不过
```

### 2.2 哪些场景 A 直接通道好用 / 不好用

| 场景 | A 直接走 | 备注 |
|----|----|----|
| 客户付员工工资（HK 内）| ✓ 好用 | EMPL 路径 |
| 客户付亚马逊货款 | ✓ 好用 | OTHER 白名单含"亚马逊官方收款" |
| 客户付海外房租 / 水电 | ✓ 好用 | OTHER 白名单 |
| 客户付海外律师 / 税务 / 保险 | ✓ 好用 | OTHER 白名单 |
| 客户付佣金（≤ 订单 15%）| ✓ 好用 | 准备订单 + 协议 + 沟通截图 |
| 客户付海外供应商（GOODS）| ⚠️ 难 | HK 客户没大陆关联 = 过不了；走 T |
| 客户付货代 / 物流 | ⚠️ 中 | 收款人行业必须匹配 |
| 客户付软件 SaaS / 服务（CNY）| ⚠️ 限 | OTHER-CNY 白名单较窄 |
| 客户付非关联第三方个人 | ✗ 难 | 走 T 转换为对公收款 |
| 客户付极高风险国家 | ✗ 难 | 必须 B 端自己解决 |
| 客户付黑名单 4 国 | ✗ 不可能 | A 完全不接 |
| 客户付小股东（< 25%）| ✗ 不可能 | 走 T 或别的路径 |

### 2.3 T 桥接在出金侧的真正价值

T 在出金侧的作用 = **改变收款人的"分类标签"**，让 A 看到更友好的场景：

| 直接走 A 难 | T 介入后变成 |
|----|----|
| 客户付 HK 客户难解释的境外供应商 | T 作为"代采服务商"收款（OTHER-代运营 / 代采服务）|
| 客户付小股东 | T 作为"代付服务公司"收款（OTHER-非佣金服务）|
| 客户付货代但收款人行业不匹配 | T 作为合规货代主体收款 |
| 客户付若干小额碎单 | T 整合后一笔大额结算 |
| 客户付极高风险地区对公（材料能补）| T 落到合规地区，再 T 内部出 |

> **核心**：T 让"难分类的出金"变成"OTHER-服务商"或"GOODS-T"这种 A 接受度高的标签。

### 2.4 出金"事前合规包"产品化

针对每种出金场景，给客户提供**预制材料包**模板：

| 场景 | 材料包内容 |
|----|----|
| GOODS（有大陆关联）| 采购合同模板 + 报关单要求清单 + 增值税发票核查表 |
| GOODS（无大陆关联，HK 客户）| **不接，转 T 桥接路径** |
| OTHER-佣金 | 佣金协议模板 + 订单关联清单 + 沟通记录留存指引（前置时间要求）|
| OTHER-技术服务 | 服务合同模板 + 验收单 + 服务交付凭证 |
| EMPL-员工 | 劳动合同 + 社保证明 + 工资流水模板 |
| 出金到亚马逊 | 亚马逊后台结算单 + 卖家账户证明 |

> **效率收益**：客户上传材料前 70% 的退回是因为格式 / 主体 / 签章错；模板化能把首次过审率从 ~40% 拉到 ~80%。

## 三、A 出金调单的"潜规则"总结

```
1. 先看"收款人是谁"，再看"业务背景"——分类 > 材料
2. 白名单制（OTHER），不是材料齐就过——业务必须能对号入座
3. 反"过桥"硬刹车——同名进同名出、纯过路不行
4. GOODS 是"硬骨头"——能避就避，用 OTHER 服务费替代
5. 佣金有上限——15% 是天花板
6. 国家分层——4 国黑名单 + 高风险国家个人收款限制
7. 个人收款受限——只能关联人 + 提现路径
8. 事后调单常态化——一次过审 ≠ 永久过关
9. 合同 + 签章 + 双方主体——形式合规是底线
```

## 四、对我方业务的关键动作

1. **出金前置咨询服务**：客户来询出金需求时，业务/合规先做分类，告诉客户"走哪个路径 + 准备哪些材料 + 是否需要 T"
2. **白名单业务画像建库**：每个 B 客户登记"主要出金场景"，看与 A 白名单匹配度，匹配低的客户 = 桥接 T 需求强
3. **T 桥接的"业务范围"对齐**：T 的工商范围必须能承接 A OTHER 白名单中的"代运营 / 技术服务 / 律师 / 税务 / 商品认证"等场景
4. **"高敏出金"批准流程**：HK 客户 GOODS / 极高风险国家 / 个人收款 = 单独审批
5. **客户教育**：客户出金前提供 **"A 出金合规手册"**（脱敏版），客户自查、自备材料
6. **政策版本监控**：A 出金政策每季度更新一次，需要专人 owner 跟进版本变化，及时调整业务流程
