TP/SP客户各中心 — 职责边界与权限设计

TP（租户）与 SP（服务商）视角下的中心划分、菜单结构与功能权限

角色定义：SP、TP、商户

🔌SP（服务商 / Service Provider）

将能力集成到 EX 平台的机构。

  • 本质：能力提供方，通过 API 或系统对接集成到 EX
  • 类别：支付服务商、物流服务商、风控服务商等（当前仅考虑支付服务商）
  • 牌照：支付服务商一定持牌
  • 后台：EX 为每个 SP 提供 SP 后台；也可选择纯 API 对接
  • 与 TP 无关：SP 身份独立于 TP 身份，SP 不一定是 TP
SP 的两种使用模式

基础模式：SP 后台（商户/交易/计费数据同步），无清算/合规/工单中心
付费购买 EX 整套系统：可包含清算中心、合规风控中心、渠道中心、财资中心等

🏢TP（租户 / Tenant）

购买 EX 能力的机构。TP 是 EX 对使用其他 SP 能力的客户的统一称呼。

  • 本质：EX 的一类客户，使用 EX 平台的能力来服务自己的客户
  • 与 SP 的关系：TP 通过签约 SP 的产品来获得支付能力
  • 可选 SP 身份：TP 可以同时是 SP（将自己的能力也集成到 EX）
TP 与 SP 的两种合作模式

代理模式：TP 代理 SP 产品 → SP 视角 TP 是"机构代理商" → 推送商户给 SP → SP 清算
直接合作模式：TP 用自己牌照 → SP 视角 TP 是"机构商户" → TP 自己管商户 → TP 自己清算
同一个 TP 可以同时使用两种模式（跟部分 SP 代理，跟部分 SP 直接合作）

💡 费率配置归属核心规则

判断依据：机构是否有清算中心（与产品的合作模式无关）

机构有清算中心？费率配在哪说明
IPL（SP 完整版）✅清算中心 → 收费配置所有商户费率统一在清算中心
BB（SP 完整版）✅清算中心 → 收费配置同上
MLG（SP 基础版）❌不配费率费率由 TP（Bonbil）在自己的清算中心配，或由 SA 代配
鲲鹏（TP 基础版）❌客户中心 → 会员配置标准费率/返点，无逐商户定制
Bonbil（TP 专业版）✅清算中心 → 收费配置所有会员费率统一在清算中心，包括代理产品和自有牌照产品

⛔ 红线：产品中心不配置具体费率数值，只引用费率方案上架。费率的具体配置只在两个地方：有清算中心 → 清算中心；无清算中心 → 客户中心会员配置。

🔗 套餐绑定关系
  • 清算中心 + 渠道中心 + 财资中心：三者绑定售卖。有清算 → 需要管渠道（上游通道）→ 需要看渠道余额（财资）
  • 合规风控中心：独立附加服务，单独售卖，不绑定在任何套餐内
  • 设置中心：所有套餐均包含（用户权限、协议、开发者中心）
  • 产品中心 + 客户中心：所有套餐均包含（基础能力）

核心概念：会员 vs 商户

👤会员（Member）

会员是一种关系维护。指由机构作为 TP 直接获客并维护的客户关系。

  • 机构作为 TP 获取的客户 = 该机构的会员
  • 会员关系归属于获客方（TP 身份）
  • 会员管理菜单仅在 TP 身份下可见

🏪商户（Merchant）

商户是业务实体。机构作为 SP 时，所有名下客户统一称为商户：

  • 自己作为 TP 获客的会员（同时也是自己 SP 的商户）
  • 其他 TP 推送过来的客户
  • 商户查询菜单在 SP 身份下可见
💡 关键区别（同时是 TP + SP 的机构）

会员 = 关系归属（作为 TP 获客）；商户 = 业务实体（作为 SP 提供服务）。
同一个客户，同一波运营人员：从 TP 角度看是“会员”（管关系、发起配置申请），从 SP 角度看是“商户”（管订单、余额、清算）。
系统不需要切换视角，两个菜单同时存在，运营按业务场景自然进入对应菜单即可。

各中心职责与边界

📦产品中心

职责：管理产品的上架、展示与签约。

  • SP 视角：上架自有产品、管理产品状态与定价
  • TP 视角：浏览产品市场、查看已代理的产品
⛔ 边界：产品中心不涉及商户级别的产品开通，商户产品开通在客户中心

👥客户中心

职责：管理所有商户信息与商户级别的业务配置。

  • 商户查询：查看名下所有商户的信息、产品、余额、费率/汇率/返点、收款工具、卡片、收款人、支付订单、贸易背景资料
  • 会员管理：产品开通、收费/汇率/分佣申请、用户权限管理、返点分佣查询
⛔ 边界：客户中心不管产品上架，只管商户维度的产品开通与配置；返点分佣查询已纳入会员管理

⚙️清算中心

职责：管理商户订单处理、商户账户管理、对账、费率/汇率/返点配置与汇率源管理。

  • 持牌 TP：管理自己会员的清算（用自己牌照服务客户时）
  • SP（付费整套）：管理被推送商户的清算
  • 商户订单管理（收款/付款/承兑订单 + 交易单 + 渠道单）
  • 商户账户与钱包（余额查询/流水/管理/冻结解冻）
  • 对账管理（商户对账/渠道对账/差错处理）
  • 产品费率/汇率/返点配置
  • 汇率源管理
  • 审批工单（费率/汇率/返点变更审批）
⛔ 边界：清算中心 usesOwnLicense=true 或 SP 付费整套；纯代理 TP 无此中心

🛡️合规风控中心

职责：基于风险处置对所有商户进行合规运营管理。

  • 持牌 TP：客户合规运营（用自己牌照服务客户时）
  • SP（付费整套）：管理被推送商户的合规
  • 商户基本信息管理（风险处置 + RFI）
  • 商户余额管理（冻结/止入止出/监控）
  • 商户产品管理（关闭/限制产品能力）
  • 商户支付订单管理（收款工具/卡片/店铺/收款人）
  • 贸易背景资料管理（贸易订单/文件/店铺/持有人）
  • RFI 管理（商户信息/VA/支付订单/收款人/发卡）
  • 渠道运营（渠道订单查询 + 渠道调单管理）
  • 审批工单（商户发起 → 合规初审 → 合规复审）
⛔ 边界：合规风控中心 usesOwnLicense=true 或 SP 付费整套；纯代理 TP 无此中心；合规侧余额管理与清算中心独立

🔌渠道中心

职责：管理渠道接入、渠道能力、路由策略与渠道订单查询。

  • 持牌 TP：各 SP 就是渠道（EX 内部 SP 模式，能力自动同步）
  • SP（付费整套）：外部 Channel + 其他 EX 内部 SP
  • 渠道管理：管理渠道的基本信息、状态、对接配置
  • 渠道能力：EX 内部 SP 能力（自动同步，只读）+ 外部 Channel 能力（手动配置）
  • 渠道路由：配置路由规则（优先级、权重、条件路由、降级策略）
  • 渠道订单查询：运营视角查询渠道侧订单（含其他 SP 和 Channel 的订单）
⛔ 边界：渠道中心 usesOwnLicense=true 或 SP 付费整套；渠道中心管"通道接入与路由"，清算中心管"订单处理与资金对账"

💰财资中心

职责：管理自有资金（非商户资金），包括头寸、调拨、备付金与外汇。

  • 持牌 TP：管理自有资金（用自己牌照服务客户时）
  • SP（付费整套）：管理 SP 自有资金
  • 头寸管理：查看各渠道/银行的资金头寸、余额监控、预警
  • 资金调拨：发起资金调拨（渠道间/银行间）、审批、查看调拨记录
  • 备付金管理：备付金账户余额、充值/提现记录、监管报表
  • 外汇管理：外汇头寸、购汇/结汇操作、汇率锁定
⛔ 边界：财资中心 usesOwnLicense=true 或 SP 付费整套；财资中心管"自有资金"，清算中心管"商户资金"

🔄工作流中心（Workflow）

职责：定义和管理审批流程模板。不存在统一的工单中心，审批工单分散在各业务领域中心处理。

  • 流程定义：制定审批流程模板（如费率变更审批流、产品开通审批流、合规审核流）
  • 节点配置：配置每个节点的审批人、条件、超时策略
  • 触发规则：业务操作触发流程 → 生成 Ticket（工单）→ 当前节点生成 Task（待办）→ 推送到对应领域
  • 流转机制：Task 完成 → Workflow Engine 推进下一节点 → 生成新 Task → 直到流程结束
📌 Workflow / Ticket / Task 关系

Workflow（工作流）= 流程定义（剧本），决定怎么走
Ticket（工单）= 业务单据（载体），从发起到结束是同一张单
Task（待办）= 节点任务（通知），某人要做的事，做完即消失

⛔ 边界：工作流中心只负责定义流程，不处理具体业务；审批工单在各领域中心内处理（清算中心有审批工单，合规风控中心有审批工单）

🚫 中心间边界红线

  • SP vs TP：SP 是能力提供方，TP 是能力使用方；两者独立，可组合
  • 代理 vs 直接合作：代理 = 推送商户给 SP；直接合作 = TP 自己管商户，SP 是渠道
  • 会员 vs 商户：同一客户，TP 视角叫会员，SP 视角叫商户
  • usesOwnLicense 控制：通过自己牌照服务客户的 TP 有清算/合规/渠道/财资中心；纯代理 TP 没有（即使持牌）
  • SP 基础 vs 付费：SP 基础模式只有 SP 后台；付费后可获得整套中心能力
  • 产品中心 vs 客户中心：产品中心管"产品上架与市场"，客户中心管"商户的产品开通与配置"
  • 产品中心 vs 渠道中心：产品中心管"卖什么给商户"（前端），渠道中心管"用什么能力交付"（后端）
  • 客户中心内部分工：商户查询管"只读查询（含订单/贸易背景）"，会员管理管"配置申请 + 返点分佣查询"
  • 工作流中心 vs 各领域：工作流中心只定义流程模板；审批工单在各领域中心内处理（清算审批在清算中心，合规审批在合规中心）
  • 会员管理仅限 TP：SP 不维护会员关系，SP 侧只有商户查询（只读 + 授权数据）
  • 客户中心 vs 清算中心：客户中心管"只读查询 + 配置申请"，清算中心管"订单管理 + 余额操作 + 对账 + 费率汇率配置"
  • 清算中心 vs 财资中心：清算中心管"商户资金"（余额/流水/对账），财资中心管"自有资金"（头寸/调拨/备付金/外汇）
  • 合规风控 vs 清算中心：合规侧余额管理与清算侧独立，合规侧基于风险处置，清算侧基于业务运营
  • 渠道中心 vs 清算中心：渠道中心管"通道接入、能力配置、路由策略"，清算中心管"商户订单处理与资金对账"
  • 渠道中心 vs 合规风控：渠道中心的渠道订单查询是运营视角（日常查询），合规的渠道运营是风险视角（调单处理）
