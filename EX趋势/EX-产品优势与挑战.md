## 0. 结论先行

EX 在**支付能力广度、Web2 × Web3 融合、东南亚客户契合度**上具备短期窗口期优势；但在**承兑落地、白牌/API 成熟度、对外叙事统一、长期差异化护城河**四个维度均存在结构性挑战。**未来 6–12 个月**的竞争分水岭是：能否在能力同质化窗口关闭前，从「列能力」切换到「场景化交付」与「持牌主体间资金闭环编排」。

## 1. 市场背景与外部约束

| --- | --- | --- |
| **大会主题** | "From Infrastructure to Impact" | 行业从拼基础设施转向拼真实落地，单点能力故事不再值钱 |
| **能力趋同** | 合规全场景 + 稳定币 + 本地通路已成"人人都讲"的标配 | 同质化窗口在加速关闭，差异化必须在 6–12 个月内立起来 |
| **Web3 渗透** | 纯 Web2 玩家越来越少；客户默认期待「也能接 crypto」 | EX/BB 的双轨能力是基础门票，不是亮点 |

## 2. EX 产品优势（Strengths）

### 2.1 能力广度：支付全栈 + Web2/Web3 双轨

- **覆盖法币 + 加密 + 卡 + 收单全链路**：法币收付（VA / POBO / COBO）、On/Off-Ramp 承兑、Crypto Checkout、VCC 一站打齐。
- **Web2 × Web3 融合是当前行业最稀缺的真落地能力**：现场对接的 9Pay、Coins.ph、QCP、OSL 等机构均一次问完两侧能力，EX/BB 是少数能整套接住的供应商。
- **行业数据印证**：72% 拥抱区块链、87% 愿意与非传统玩家合作 → EX 双轨叙事天然契合。

### 2.2 持牌主体矩阵在建（EX 编排 × BB / IPL 持牌）

- **BB（BlancBlok）**：波兰 VASP（已持）+ 瑞士 SRO（已持）+ 加拿大 MSB（申请中）+ 加拿大 MPI（申请中）+ 香港 VASP（准备中）+ 欧盟 CASP（申请中）。
- **IPL**：法币收付通路侧持牌主体。
- **EX**：科技 / 合规运营平台（**不持牌、不碰资金**），做编排、风控、对账、对外叙事。
- **意义**：相比 Remi 这类纯 API 公司，EX/BB 自有持牌底座，**法币 ↔ 稳定币的资金可以在自有持牌主体间对敲**，长期具备 Bison 类似形态的潜力。

### 2.3 多模式交付：直客 + 白牌 + API 三层并行

- **直客**：直接服务最终商户（VCC / Crypto Checkout / 收付）。
- **白牌**：UI / 域名 / 品牌可替换给租户。
- **API**：纯接口输出给已持牌或自有客户运营能力的机构。
- 一份能力栈对应三种买家，**单客户 LTV 可以拉得更长**。

### 2.4 拓展验证：数据已经跑出来

| 指标 | 目标 | 实际 | 达成率 |
| --- | --- | --- | --- |
| 展会约谈 | 25 | 68 | **272%** |
| 现场意向签约 | 15 | 23 | **153%** |
| B 渠道建联（U卡/OTC/银行/钱包） | 12+ | 32 | **267%** |
| 高质量 FIBD 渠道（≥4分） | — | 25 | — |

- **东南亚客户密度高、银行渠道意向旺、Web3 机构主动找上门**——产品方向被市场用脚投票验证。
- 已建联标杆渠道：OSL Group、Coins.ph、QCP Group、Krungsri Bank、Clear.Bank、Crown Agents Bank 等。

### 2.5 EX × BB 合并叙事：对外口径有了简洁版本

展会上把"科技平台 + 持牌"合并讲，比拆开介绍效率高得多。对外收敛为 4 条：

| 序号 | 能力 | 计费模式 |
| --- | --- | --- |
| 1 | **支付能力**（法币 + 加密的收付 / 承兑） | 标准计费 |
| 2 | **直客服务能力** | 标准计费 |
| 3 | **白牌 / API 输出** | 接入费 |
| 4 | **风控能力** | 单独计费 |

这套叙事**同时覆盖直客与渠道**，不再被迫去解释"平台 / SP / 租户"的内部结构。

### 2.6 软资产潜力：多 SP 编排是客户复制不来的

- **跨 SP 统一清结算网络、合规规则引擎、资金调拨编排、统一风控决策中台**——这些都是 EX 不需要持牌就能沉淀的"软基础设施"。
- 一旦成型，客户和竞争对手要复刻代价极高，是**长期差异化的最大候选**。

## 3. EX 产品挑战（Challenges）

### 3.1 承兑能力是"基础门票"，但还没真正跑通

- 展位上**几乎所有客户的第一句话**都是 *"你们有没有收付 / 承兑能力？"*——这是入场券，不是加分项。
- 当前 OnRamp / OffRamp 仍处一期阶段（BB 单中心垫资、A1 + B1 范围），跨 SP 调拨与多币种 / 多通道路由都尚未上线。
- **风险**：如果半年内承兑稳定性、限额、覆盖币种 / 通道没跑稳，前面所有意向客户都会复用上 *"再看看"*。

### 3.2 能力同质化窗口正在快速关闭

- 合规全场景 + 稳定币 + 本地通路被越来越多的机构同步在讲，**短期内变成"人人都有"**。
- 仅靠"我们也有这些能力"已经站不住，**1–2 年内不立起独有差异化，溢价空间就会被压平**。
- 长期出路必须是**两种之一或组合**：
  1. **持牌矩阵 + 持牌主体间资金闭环**（参考 Bison 的 off-ramp 跑出来路径）；
  2. **场景化交付 + 行业 know-how**（不是把能力卖给客户，是把"行业默认配置 + 合规模板"卖给客户）。

### 3.3 三公司（EX / BB / IPL）并行叙事，渠道理解成本高

- 同一渠道方在展会上需要花精力梳理三者关系——**有时候连我们自己人都先要解释一遍**。
- 这与 Bison 案例对比尤其明显：Bison Bank（持牌） + BiG（VASP 子公司） + Remi（独立 API 公司）三层结构清晰、各司其职，对外有**单一品牌出口**（Remi）。
- **思考题**：是否需要一个**统一对外品牌出口**（类比 Remi）来收口？短期解决叙事问题，长期作为牌照矩阵的对外阵地。

### 3.4 客户认知门槛高："科技平台 / SP / 租户"难以一次讲清

- 现场客户对结构化概念缺乏耐心，注意力 100% 集中在"承兑 / 支付能力"本身。
- 即便有 §2.5 那套四条合并叙事，**对外销售物料、Pitch deck、官网首页**仍未完全转向"客户视角"。
- **风险**：销售素材停留在"我们是什么"，没有切到"你能用我做什么"。

### 3.5 白牌 / API 成熟度跟不上签约客户预期

- 与潜在客户不同，**已签约客户**的关注点 100% 落在落地：
  - API 完整度（收付、承兑、账户、风控、对账闭环是否全覆盖）；
  - 文档清晰度与 Sandbox 自助走通能力；
  - 稳定性、上线周期、问题响应速度（SLA）。
- 现状差距：API 文档 / Sandbox / 白牌门户 / 监控告警 / 工单 SLA 体系都在搭，但还**不到"客户能自助跑完一个完整业务闭环"的水平**。
- **风险**：口碑会**先在签约客户圈坏掉**，而签约客户互相打听是销售线索的重要来源——**这比拓新失败更危险**。

### 3.6 海外 BD 与总部 / 产品侧协同链路偏长

- 现场带去的两位海外 BD 基础合格，但**深度问题**（合规、资金路径、牌照链路、KYB 政策）**仍依赖后方远程支援**。
- 时区差 + 远程沟通 → 客户当场拿不到答案，二次跟进成功率明显下降。
- **改进方向**：
  - 系统化 BD 知识库（FAQ / 业务模板 / 合规边界 / 牌照矩阵速查表）；
  - 关键展会前置培训 + 现场技术后援值班；
  - 高潜客户由产品 / 合规直接现场接管。

### 3.7 展会现场暴露的产品/物料硬伤

- **缺交互式 Demo 系统**：客户没办法现场跑一个 VA 收款 / 承兑 / 卡发行的真实链路，只能听讲解。
- **沟通话术非标**：每条业务线缺"初次见面 1–2 段标准话术"，新人 BD 现场表达效果差异大。
- **品牌引流不足**：现场扫码关注 LinkedIn / 留资的引导力度偏弱，私域沉淀不足。

> 详见 `money2020.md` 二、筹备不足之处。

### 3.8 目标客户画像尚未完全收敛

不同画像需要**完全不同**的能力组合，但资源有限，铺得过广反而每条都不深：

| 画像 | 核心诉求 | EX 当前匹配度 |
| --- | --- | --- |
| **Web3 机构**（OTC / CEX / 钱包） | 法币 On/Off-Ramp + VA + 大额结算 + 风控 | 中（承兑刚开始跑） |
| **持牌支付机构 / 银行** | 多 SP 编排 + 白牌 + 合规模板 | 中（白牌仍在建） |
| **跨境贸易直客** | 收付 + FX + KYB + 简易接入 | 中高（IPL 通路较成熟） |
| **平台型商户**（SaaS / 市场） | 收单 + 分账 + 卡 + Webhook | 中（Crypto Checkout 刚启动） |

**待回答**：未来 6 个月以哪个画像为绝对优先级？打透一个画像比泛泛覆盖四个更值得。

### 3.9 长期路径选择尚未拍板

留给战略 / 老板层后续讨论，不在此结论：

- **路径 A**：做"**多 SP 编排平台**"——EX 不持牌，靠软资产（编排 / 规则引擎 / 跨主体清算）赚平台费。
- **路径 B**：做"**自有持牌网络**"——BB + 收购 / 合作的持牌主体集群，做闭环资金网络，对标 Bison。
- **路径 C**：A + B 组合，对外用统一独立品牌出口（类比 Remi）。

> 三条路径所需投入、回报周期、合规风险差异极大，必须**先选再投**，否则很容易"啥都做、啥都浅"。

### 3.10 不像 Bison 自带"私人银行 + 高净值托管"传统底盘

- Bison 之所以推 Off-Ramp 顺滑，是因为**法币沉淀在自家银行 = 低成本存款 + 手续费 + 汇差**，是它原生擅长的赚钱方式。
- BB / IPL **没有传统私人银行 / 贸易融资业务**作为底盘，做承兑出金时缺少"原生赚钱场景"——必须靠纯通道费 / FX 价差，**单笔经济性偏薄**。
- **应对**：长期需要思考是否补上一类**高粘性、高 LTV 的资金沉淀业务**（如机构托管、贸易融资、稳定币发行），让承兑业务不只赚通道费。

## 6. 核心判断

1. **短期看承兑、白牌、API 的"能不能用"**——这是基础门票，不及格其他都谈不上。
2. **中期看场景化与画像聚焦**——把能力栈卖给"行业 + 场景"，不是卖给"想要 API 的人"。
3. **长期看持牌主体间资金闭环 + 软基础设施沉淀**——Bison 已经把这条路跑出来，EX/BB 的窗口期还在，但不会持续太久。
4. **三主体叙事问题不能拖**——客户和渠道理解成本是真实的销售损失，需要尽早决策是否引入统一独立品牌出口。
