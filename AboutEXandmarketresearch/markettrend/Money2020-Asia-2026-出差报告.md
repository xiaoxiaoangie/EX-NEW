## 2. 展会总体观察

### 2.1 规模与参展机构构成

展会**整体人流与机构数量中等**（并非非常火爆），但**参展人员密度高、专业度高**，现场几乎全是支付产业链内的从业者，少有无关观众。大致可分为 **4 类**：

1. **Web2 支付机构 & 银行**
2. **Web3 机构**（交易所、钱包、稳定币发行方、链上清结算）
3. **合规科技**（KYC / KYB / AML / 交易监控）
4. **AI 提效工具**（客服、coding/testing、风控建模）

### 2.2 行业趋势

- **Web3 能力已是"标配"**：纯 Web2 的支付机构很少很少；只找web2 能力的商家等也非常少，即便是，了解到目前可以提供web3 能力，也觉得非常感兴趣
- **共识从"纯 Web3"回归到"Web2 × Web3 融合"**：公链、DeFi、GameFi 等场景普遍缺乏真实落地；**支付是 Web3 目前少数跑通的落地场景**。
- 各方都在探索**"Web3 资产能力 + Web2 合规 / 法币通路"** 的叠加路径。

### 2.3 合规 + 稳定币 + 本地支付轨：多数展商的标准组合

除 Bison / Remi 外，现场合规类参展方占比相当高，而且**已经不是单点工具了**，普遍往全场景方向做：

- **合规机构普遍已做到"全场景"**：KYC / KYB、交易风控、KYT（链上追踪）、Travel Rule、制裁名单命中等能力一家全包，已经不太有只做某一环的单点玩家；
- **在全场景合规之上，普遍叠加两件事**：
  - **稳定币**（USDC / USDT 的入金 / 出金 / 清算）；
  - **支付能力：**va+payout

**但展会同时暴露出一个问题**：**大家的能力都差不多，或者正在快速趋同**。合规全场景 + 稳定币 + 本地通路，这套组合被越来越多的机构一起讲，短期内会变成"人人都有"的标配。

**这对 EX / BB 的真实启示**：

- 仅靠"也有这些能力"是站不住的，同质化窗口会很快会关闭；
- 更重要的是**回过头来想清楚两件事**：
  1. **我们的目标客户画像是谁？**（是 Web3 机构 / 是传统持牌支付机构 / 是直客商户，不同画像需要的能力组合完全不同）
  2. **长期差异化靠什么？** 是牌照组合、还是持牌主体间的闭环资金网络、还是特定地区的本地通路深度、还是直客服务能力——**不能等到大家都一样了再想**。

## 3. EX 展台观察

现场客户诉求高度一致、直接，几乎都是开门见山：

> **"你们有没有收付 / 承兑能力？"**

由此观察到几个问题：

1. **客户注意力聚焦在"承兑 / 支付能力"本身**，对 EX 作为"科技平台"的定位缺乏耐心听完；
2. **三公司（BB / IPL / EX）并行对接**同一渠道时，渠道方要额外花精力梳理三者关系，造成理解成本；
3. **术语门槛太高**：客户与渠道难以一次性理解"科技平台 / SP / 租户"这类结构化概念。

### 3.1 海外 BD 观察

- 现场带去的 **2 位海外 BD 整体表现可以**：语言、客户沟通节奏、对产品的基本理解都到位，作为展会一线接触是合格的；
- **但与总部 / 产品侧的协同还有空间**：客户现场提出的深度问题（合规、资金路径、牌照链路等）仍依赖后方远程支援，回复链路略长；
- **后续需要系统性思考如何更好地跟海外 BD 交互；**

## 4. 核心启示：EX + BB 合并叙事

在展会上**把 EX 和 BB 合并一起介绍**，比单独讲解每一侧**更简洁、客户更能立刻 get 到**。建议对外口径统一收敛为 4 条：

| 序号 | 能力                      | 计费模式                               |
| ---- | ------------------------- | -------------------------------------- |
| 1    | **支付能力**        | 基础的收付 / 承兑能力，覆盖法币 + 加密 |
| 2    | **直客服务能力**    | 可直接服务最终商户                     |
| 3    | **白牌 / API 输出** | 需要白牌接入？**有接入费**       |
| 4    | **风控能力**        | 需要风控？**单独计费**           |

这套叙事同时覆盖直客与渠道两类合作方，**不用再去强行区分"平台 / SP / 租户"**，节省沟通成本。

## 5. 重点竞品观察：Bison Bank + Bison Digital Assets + Remi

现场**唯一一家业务结构与我们高度相似**的是葡萄牙 **Bison Bank（贝森银行）**。展位同时出现 Web2 银行团队、Web3（Bison Digital Assets）团队，且**背后还有一个战略合作方 Remi**（独立公司），三者角色一开始并不好区分——连我们自己看完都一度混乱。理清后发现，这**恰恰是他们能跑通"持牌银行 + Web3 网络"的关键**。

### 5.2 三方各自能力与定位

| ------------------------------------ | -------------------------------------------------------- | ------------------------------------------------------- | ----------------------------------------------------- |
| **Bison Bank**                 | 葡萄牙私人银行（2018 年被香港 Bison Capital 收购后重组） | 欧盟银行牌照 + MiCA（稳定币）                           | 法币账户 / 托管 / 清结算 / 合规背书                   |
| **Bison Digital Assets (BiG)** | Bison Bank 的**全资 Web3 子公司**                  | 葡萄牙央行**首张 VASP 牌照**（2022）              | crypto 托管、OTC、on/off-ramp、机构入网               |
| **Remi**                       | **独立新加坡金融科技公司**                         | 自身无银行牌照，**依托 Bison 的 MiCA 做合规载体** | API 基础设施、稳定币结算网络、Programmable Compliance |

**关键澄清**：Remi 在新闻稿中明确自我定位为 Bison Bank 的"**core settlement network**"——即 Remi 是 Bison 合规稳定币商业化的**技术与网络出口**。

### 5.3 On/Off-Ramp 的真实模式（更正以往理解）

现场聊下来，Bison 的 on-ramp 和 off-ramp **都只做机构 / 大客户，不做零售散户**，且两端的资金角色完全不对称：

**On-ramp（法币 → U）：Bison 只是"过路收款行"**

```
① 终端客户 ─电汇USD→ Bison 机构账户/FBO客资账户
② Bison 通知机构「收到 X USD」
③ 机构指令 OTC「放 X USDT 给客户钱包」
④ OTC 从【机构预先在 OTC 处存入的法币头寸】扣款，T+0 放 U
⑤ 后续 Bison 把法币调拨给机构 → 机构补回 OTC 头寸
```

- **OTC 不承担资金敞口**，敞口由**做 ramp 的机构/大客户**承担（法币在途风险 + OTC 头寸占用成本）；
- Bison 赚的是**通道费 + 合规审查费**，法币只是过路，**沉淀在 OTC 那头**。

**Off-ramp（U → 法币）：Bison 是"持牌付款行 + 存款银行"**

```
机构/大客户提前把法币沉淀在 Bison 账户
                ↓
客户发起下发指令 → Bison 用行内余额走 SEPA/SWIFT 出款到受益人
```

- 下发前**资金已在行内**，不存在"收加密钱款"的动作；
- 对 Bison 而言 = **低成本存款 + 手续费 + 汇差**，是传统银行最熟悉的赚钱方式。

### 5.4 为什么 Bison 推 off-ramp、冷淡 on-ramp

不是合规做不了，而是**商业利益和风险结构不对等**：

| 维度           | On-Ramp                                                | Off-Ramp                                                 |
| -------------- | ------------------------------------------------------ | -------------------------------------------------------- |
| 法币沉淀地     | OTC（对 Bison 无益）                                   | **Bison（银行负债端存款）**                        |
| Bison 角色     | 过路通道                                               | 付款行 + 存款银行                                        |
| 代理行审查     | USD 入金经代理行，crypto 关联资金易被 RFI / 退汇       | SEPA/SWIFT 标准对公付款，代理行不敏感                    |
| 客户门槛       | 必须能在 OTC 备几百万~几千万法币头寸 → 天然只服务机构 | 已 KYB 的机构客户沉淀资金，合规链路短                    |
| 单笔经济性     | 薄利通道费                                             | 存款 + 手续费 + 汇差                                     |
| Bison 历史基因 | 与传统私人银行业务关系弱                               | **延续私人银行 + 贸易融资 + 高净值托管**，业务顺滑 |

**一句话**：**off-ramp 让 Bison 挣"银行该挣的钱"，on-ramp 是替别人做嫁衣**。这是在欧盟 MiCA 合规 + 代理行 de-banking + 中小型持牌银行脆弱代理关系这三条硬约束下的**最优商业选择**。

## 6. 待办

### 6.1 短期：持续建设收付 / 承兑能力

- 把 **VA 收款 + Payout + On/Off-Ramp 承兑** 作为本期主线产品继续打磨；
- 这是现场客户**最关心、第一句话就问的能力**，没有这个底盘，其他叙事都站不住；
- 

### 6.2 中期：EX 白牌 / API 方案持续建设

会议上**已经有签约客户也到场**，他们和潜在客户不一样——

- **潜在客户**：听故事、看 Pitch；
- **签约客户**：直接关注**实际能落地的能力**——API 是否齐全、文档是否清晰、稳定性、上线周期、问题响应速度。

如果 EX 白牌 / API 跟不上，**口碑会先在签约客户圈里坏掉**，比拓新更危险。需要重点推进：

1. **API 完整度对齐**：收付、承兑、账户、风控、对账闭环；
2. **白牌门户**：UI 可定制、品牌可替换、权限可分层；
3. **接入文档与 Sandbox**：客户能自助走通 demo，不依赖人；
4. **SLA 与运维响应**：签约客户要的是"出问题有人接"，不是"功能多"。

### 6.3 中长期：开放讨论

能力同质化窗口正在快速关闭（合规全场景 + 稳定币 + 本地通路已经是"人人都有"），单点能力堆料不再有竞争力。中长期需要回答的不再是"做什么功能"，而是"**靠什么活下去、靠什么变贵**"。以下方向都值得拉开讨论，不下结论：

**中期方向：场景化落地**

- 从"我们有什么能力"切换到"客户在什么场景下要解决什么问题"；
- 按客户画像拆（Web3 机构 / 持牌支付机构 / 跨境贸易直客 / 平台型商户），不同画像组合不同能力；
- 按场景拆（跨境收单结汇出金一条龙 / 薪酬代发 / 平台分账 / 加密发卡 / 做市商法币通路 …）；
- 输出 = 能力组合 + 行业 know-how + 合规模板 + 默认配置，让客户不用自己拼积木。

**长期方向：基础设施 + 牌照矩阵**

> 前提澄清：**EX 是科技 / 合规运营平台，不持牌、不碰资金**；BB / IPL / 其他合作 SP 是持牌主体，所有法币与加密资金都运行在持牌主体账本上。下面说的 “闭环” 指的是**持牌主体之间的资金闭环**，EX 做编排、合规、清算、对外叙事。

- **基础设施层（EX 负责建）**：多 SP 统一清结算网络、合规规则引擎、跨 SP 资金调拨编排、统一风控决策中台——这些是客户复制不来的软资产，不需要持牌；
- **牌照矩阵（BB 及合作持牌 SP 负责拿）**：评估**多地多牌**组合策略——
  - 美国（MSB / 各州 MTL / sponsor bank 关系）
  - 欧盟（EMI / MiCA CASP）
  - 香港（MSO / 稳定币发行人）
  - 新加坡（MPI / DPT）
  - 中东（VARA / ADGM）
- 牌照不是终点，**牌照 + 合规运营 know-how 才是护城河**（参考贝森：拿到 VASP + MiCA 之后才真正跑出 off-ramp 业务）；
- **持牌主体间的资金闭环**：BB 法币 ↔ BB / 合作 SP 的稳定币能力在持牌体系内对敲（类似贝森银行 Web2 存款 ↔ BiG 的 Web3 出金两端结构，贝森两者都是持牌主体），EX 做跨主体的资金调拨编排与合规贯穿，不持有资金。要跑出这个形态，前提是先把基础能力和牌照打齐。

**待思考的问题**

- 我们是想做"**多 SP 编排平台**"还是"**自有持牌网络**"？两条路投入和回报差别极大；
- **独立品牌**承接对外（类比贝森的 Remi）是否值得做？短期能解决"三家并行对渠道"的叙事问题，长期可作为牌照矩阵的统一出口；
- 哪些牌照**自己拿**、哪些通过**合作 / 入股 / 收购**拿？周期、资本、风险全不一样。

> 这一节不给答案，留作老板 / 战略团队后续专题讨论。
